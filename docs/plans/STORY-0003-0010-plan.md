#### 3.1.2 Sections Modified

**Step 2 -- Extension for TDD Rules Extraction (Lines ~50-68)**

Add a new subsection after the existing "What stays in individual stories" block. This subsection instructs the skill to identify and extract TDD-related cross-cutting rules when the spec describes a system that adopts TDD practices.

New content to append to Step 2:

```markdown
**TDD cross-cutting rules:**
When the spec describes a system that follows TDD practices, extract these as cross-cutting rules:
- **Red-Green-Refactor**: Mandatory cycle for all production code — write failing test, make it pass, refactor
- **Atomic TDD Commits**: Each Red-Green-Refactor cycle produces atomic commits with Conventional Commits format
- **Gherkin Completeness**: Every acceptance criterion must have corresponding Gherkin scenarios
```

**Step 4 -- DoD Extension for TDD Items (Lines ~88-103)**

Add TDD-specific items to the "Global Definition of Done (DoD)" bullet list. These are **additional** items -- existing items (coverage, test types, documentation, SLOs, persistence) are preserved.

New content to append to the DoD bullet list:

```markdown
- **TDD Compliance**: Commits show test-first pattern (test precedes implementation in git log).
  Explicit refactoring step after green. Tests are incremental, progressing from simple to complex
  following the Transformation Priority Premise (TPP).
- **Double-Loop TDD**: Acceptance tests derived from Gherkin scenarios (outer loop). Unit tests
  guided by the Transformation Priority Premise (inner loop).
```

#### 3.1.3 Sections Added

| New Content | Location | Purpose |
|-------------|----------|---------|
| TDD rules extraction guidance | End of Step 2 | Instruct skill to extract TDD cross-cutting rules when applicable |
| TDD Compliance DoD item | Step 4, DoD bullet list | Ensure generated epics include TDD compliance in DoD |
| Double-Loop TDD DoD item | Step 4, DoD bullet list | Ensure generated epics include Double-Loop TDD in DoD |

#### 3.1.4 Sections Removed

**None.** All changes are additive (RULE-003: Backward Compatibility). Existing DoD items and rule extraction guidance are preserved.
