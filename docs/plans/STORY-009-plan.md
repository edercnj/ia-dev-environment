### 2.1 PipelineResult Dataclass (`claude_setup/models.py` -- extend existing)

| Field | Type | Description |
|-------|------|-------------|
| `success` | `bool` | Pipeline completed without errors |
| `output_dir` | `Path` | Final output directory |
| `files_generated` | `List[Path]` | All files generated by all assemblers |
| `warnings` | `List[str]` | Warnings accumulated during generation |
| `duration_ms` | `int` | Execution time in milliseconds |

```python
@dataclass
class PipelineResult:
    success: bool
    output_dir: Path
    files_generated: List[Path]
    warnings: List[str]
    duration_ms: int
```

This is a pure domain model with zero framework dependencies. Belongs in `models.py` alongside `ProjectConfig`.

### 2.2 Pipeline Orchestration (`claude_setup/assembler/__init__.py` -- extend existing)

New top-level function:

| Function | Signature | Description |
|----------|-----------|-------------|
| `run_pipeline` | `(config: ProjectConfig, src_dir: Path, output_dir: Path, dry_run: bool = False) -> PipelineResult` | Orchestrates all assemblers in sequence with atomic output. |

**Algorithm:**

1. Record start time.
2. If `dry_run`, call `_plan_pipeline()` to list what would be generated without writing.
3. Otherwise, enter `atomic_output(output_dir)` context manager:
   a. Create `TemplateEngine(src_dir, config)`.
   b. Instantiate all assemblers: `RulesAssembler()`, `PatternsAssembler(src_dir)`, `ProtocolsAssembler(src_dir)`, `SkillsAssembler()`, `AgentsAssembler()`, `HooksAssembler(src_dir)`, `SettingsAssembler(src_dir)`, `ReadmeAssembler(src_dir)`.
   c. Call each `assembler.assemble(config, temp_dir, engine)` in order, accumulating `files_generated` and `warnings`.
   d. On success, atomic move from temp to output.
   e. On failure, cleanup temp dir, return `PipelineResult(success=False, ...)`.
4. Record end time, compute `duration_ms`.
5. Return `PipelineResult`.

**Assembler execution order (critical -- ReadmeAssembler must be last):**

1. `RulesAssembler.assemble()` -- generates `rules/`
2. `SkillsAssembler.assemble()` -- generates `skills/`
3. `AgentsAssembler.assemble()` -- generates `agents/`
4. `PatternsAssembler.assemble()` -- generates `rules/patterns/`
5. `ProtocolsAssembler.assemble()` -- generates `rules/protocols/`
6. `HooksAssembler.assemble()` -- generates `hooks/`
7. `SettingsAssembler.assemble()` -- generates `settings.json`, `settings.local.json`
8. `ReadmeAssembler.assemble()` -- generates `README.md` (LAST, counts other artifacts)

### 2.3 Utilities Module (`claude_setup/utils.py` -- new file)

| Function | Signature | Description |
|----------|-----------|-------------|
| `atomic_output` | `(dest_dir: Path) -> ContextManager[Path]` | Context manager: creates temp dir, yields it, on success copies to dest, on failure cleans up temp. |
| `setup_logging` | `(verbose: bool) -> None` | Configures root logger level (DEBUG if verbose, INFO otherwise). |
| `find_src_dir` | `() -> Path` | Auto-detects the `src/` directory relative to the package installation. |

**`atomic_output` contract:**

- Creates a `tempfile.mkdtemp()` directory.
- Yields the temp `Path`.
- On `__exit__` with no exception: uses `shutil.copytree` to copy temp contents to `dest_dir`, then removes temp.
- On `__exit__` with exception: removes temp dir via `shutil.rmtree`, re-raises exception.
- Guarantees: no partial files in `dest_dir` on failure (RULE-003).

**`find_src_dir` contract:**

- Resolves `Path(__file__).resolve().parent.parent / "src"`.
- Raises `FileNotFoundError` if the directory does not exist.

### 2.4 CLI Rework (`claude_setup/__main__.py` -- rewrite)

The current CLI has a single `init` subcommand. STORY-009 requires:

**New command structure:**

```
claude-setup generate --config <path> [--output-dir DIR] [--src-dir DIR] [--verbose] [--dry-run]
claude-setup generate --interactive [--output-dir DIR] [--src-dir DIR] [--verbose] [--dry-run]
claude-setup validate --config <path> [--verbose]
```

**`generate` command:**

1. If `--config` provided: `load_config(path)` to get `ProjectConfig`.
2. If `--interactive` provided: `run_interactive()` to get `ProjectConfig`.
3. If neither: raise `click.UsageError`.
4. If both: raise `click.UsageError`.
5. Resolve `src_dir` (from `--src-dir` or `find_src_dir()`).
6. Resolve `output_dir` (from `--output-dir` or default `.`).
7. If `--verbose`: `setup_logging(verbose=True)`.
8. Call `run_pipeline(config, src_dir, output_dir, dry_run=dry_run)`.
9. Display `PipelineResult` summary.
10. Exit with code 0 on success, 1 on failure.

**`validate` command:**

1. `load_config(path)` to get `ProjectConfig`.
2. Call `validate_stack(config)` from `domain/validator.py`.
3. Display validation results.
4. Exit with code 0 if valid, 1 if errors.

**Backward compatibility:** The existing `init` command is replaced by `generate`. No backward compat required since the tool is pre-1.0.

### 2.5 Dry Run Support

Dry-run is implemented at the pipeline level, not per-assembler. When `dry_run=True`:

1. Pipeline does NOT call assemblers.
2. Pipeline returns a `PipelineResult` with `success=True`, `files_generated=[]` (or a predicted list), `warnings=["Dry run -- no files written"]`.
3. CLI prints a summary of what would be generated based on config analysis.

Alternative approach (preferred for accuracy): run the full pipeline in a temp dir, collect results, then discard the temp dir instead of moving to destination. This gives an accurate `files_generated` list.

### Assembler Interface Normalization

Current assembler constructors are inconsistent:

| Assembler | Constructor | `src_dir` pattern |
|-----------|-------------|-------------------|
| `RulesAssembler` | `__init__(self)` -- no args | Resolves `src_dir` internally via `Path(__file__)` |
| `SkillsAssembler` | `__init__(self)` -- no args | Resolves `src_dir` internally |
| `AgentsAssembler` | `__init__(self)` -- no args | Resolves `src_dir` internally |
| `PatternsAssembler` | `__init__(self, src_dir: Path)` | Receives `src_dir` |
| `ProtocolsAssembler` | `__init__(self, src_dir: Path)` | Receives `src_dir` |
| `HooksAssembler` | `__init__(self, src_dir: Path)` | Receives `src_dir` |
| `SettingsAssembler` | `__init__(self, src_dir: Path)` | Receives `src_dir` |
| `ReadmeAssembler` | `__init__(self, src_dir: Path)` | Receives `src_dir` |

The pipeline must handle both constructor patterns. No normalization needed for STORY-009 -- the pipeline just instantiates each assembler with the appropriate signature. A future story can normalize if desired.

```
__main__.py (CLI adapter/inbound)
    --> assembler/__init__.py (run_pipeline -- application layer)
        --> assembler/*.py (all assemblers)
            --> domain/stack_mapping.py
            --> domain/resolver.py
            --> models.py (ProjectConfig, PipelineResult)
            --> template_engine.py
    --> config.py (load_config)
    --> interactive.py (run_interactive)
    --> domain/validator.py (validate_stack)
    --> utils.py (atomic_output, setup_logging, find_src_dir)

utils.py
    --> stdlib only (tempfile, shutil, pathlib, logging)

models.py (PipelineResult)
    --> stdlib only (dataclasses, pathlib)
```

All dependencies point inward:
- CLI (`__main__.py`) depends on application (`run_pipeline`) and domain (`models`, `validator`)
- Application (`run_pipeline`) depends on domain (`models`, assemblers)
- Domain (`models.py`, `stack_mapping.py`) depends only on stdlib
- `utils.py` depends only on stdlib

No circular dependencies. No domain -> adapter dependency.

## 10. Risk Assessment

| Risk | Impact | Likelihood | Mitigation |
|------|--------|------------|------------|
| Atomic output race condition (temp dir cleanup on interrupt) | HIGH | LOW | Use `try/finally` in context manager. Register `atexit` handler for temp dir cleanup. Add signal handling for SIGINT/SIGTERM. |
| Assembler constructor inconsistency (some take src_dir, some don't) | MEDIUM | CERTAIN | Pipeline handles both patterns explicitly. Document the inconsistency for future normalization. |
| `--dry-run` accuracy vs. performance tradeoff | MEDIUM | MEDIUM | Preferred approach: run full pipeline in temp dir, then discard. Accurate but slower. Alternative: config-based prediction (faster but may miss edge cases). |
| CLI backward compatibility (removing `init` command) | LOW | LOW | Pre-1.0 tool, no external consumers. Add deprecation warning if `init` is invoked. |
| Cross-platform temp dir behavior (Windows vs. Unix) | MEDIUM | LOW | Use `tempfile.mkdtemp()` which is cross-platform. Avoid Unix-specific operations in `atomic_output`. |
| Large file tree performance during atomic copy | LOW | LOW | Use `shutil.copytree` with `dirs_exist_ok=True` (Python 3.8+). For very large outputs, consider `os.rename` if same filesystem. |
| Assembler failure midway through pipeline | HIGH | MEDIUM | Atomic output guarantees cleanup. Pipeline catches exceptions per-assembler and records in warnings. On critical failure, entire pipeline aborts cleanly. |
| Existing tests for `init` command break | MEDIUM | CERTAIN | Update `tests/test_cli_init.py` and `tests/test_cli.py` to test new `generate` and `validate` commands. |
