# Epico: Skills Evolution — Novas Skills para ia-dev-environment

**Autor:** Tech Lead
**Data:** 2026-03-19
**Versao:** 1.0.0
**Status:** Concluido

## 1. Visao Geral

**Chave Jira:** EPIC-0007

Adicionar 5 novas skills ao gerador `ia-dev-environment`, expandindo as capacidades de automacao disponíveis para projetos gerados. Cada skill e gerada como template para **ambos** Claude Code (`.claude/skills/`) e GitHub Copilot (`.github/skills/`), mantendo paridade entre as duas plataformas.

**Skills incluidas:**

| # | Skill | Grupo | Descricao |
|---|-------|-------|-----------|
| 1 | `x-fix-pr-comments` | git-troubleshooting | Le comentarios de review de PR e corrige automaticamente os actionable |
| 2 | `x-codebase-audit` | review | Auditoria completa do codebase contra todos os padroes do projeto |
| 3 | `x-changelog` | git-troubleshooting | Gera CHANGELOG.md a partir de Conventional Commits |
| 4 | `x-dependency-audit` | review | Verifica dependencias quanto a vulnerabilidades, versoes desatualizadas e licencas |
| 5 | `x-mcp-recommend` | dev | Analisa tech stack e recomenda MCP servers relevantes |

**Escopo incluido:**
- Templates SKILL.md para Claude Code (5 skills em `skills-templates/core/`)
- Templates .md para GitHub Copilot (5 skills em `github-skills-templates/`)
- Registro no `GithubSkillsAssembler` (mapa SKILL_GROUPS)
- Atualizacao de testes unitarios (tamanhos de grupo)
- Regeneracao de golden files para os 8 perfis bundled

**Escopo excluido:**
- Alteracoes no `SkillsAssembler` (auto-discovery de `core/` e suficiente)
- Novas knowledge packs ou agents
- Novos assemblers ou mudancas no pipeline
- Skills condicionais (todas as 5 sao incondicionais)

## 3. Definicoes de Qualidade Globais

### Global Definition of Ready (DoR)

- Versao Java do ia-dev-env funcional (EPIC-0006 concluido ou em andamento)
- GithubSkillsAssembler implementado e testado
- SkillsAssembler com auto-discovery funcional
- Golden files de referencia existentes para os 8 perfis
- Templates existentes como referencia de formato e placeholders

### Global Definition of Done (DoD)

- **Cobertura:** >= 95% Line Coverage, >= 90% Branch Coverage (JaCoCo)
- **Testes Automatizados:** Unitarios (JUnit 5 + AssertJ), golden file (paridade byte-a-byte)
- **Paridade Dual:** Cada skill existe em AMBOS `skills-templates/core/` E `github-skills-templates/`
- **Autonomia:** Cada SKILL.md e auto-contido — um agente IA lendo apenas o SKILL.md consegue executar a skill
- **Placeholders:** Apenas tokens estabelecidos (`{{PROJECT_NAME}}`, `{{LANGUAGE}}`, `{{BUILD_TOOL}}`, etc.)
- **TDD Compliance:** Test-first, refactoring explicito, TPP incremental
- **Backward Compatibility:** Nenhuma skill existente removida ou renomeada

## 4. Regras de Negocio Transversais (Source of Truth)

> Regras que se aplicam a multiplas historias. Cada historia referencia as regras pelo ID.

| ID | Titulo | Descricao |
| :--- | :--- | :--- |
| **[RULE-001]** | Dual Copy Consistency | Toda skill existe em AMBOS `skills-templates/core/` E `github-skills-templates/`. Ambos atualizados juntos. Template Claude Code usa frontmatter com `allowed-tools` e `argument-hint`. Template GitHub Copilot usa frontmatter com `name` e `description` apenas. |
| **[RULE-002]** | Source of Truth e resources/ | Todas as alteracoes em `java/src/main/resources/`. `.claude/` e `.github/` sao outputs gerados. Nunca editar outputs diretamente. |
| **[RULE-003]** | Backward Compatibility | Novas skills sao aditivas. Nenhuma remocao ou renomeacao de artefatos existentes. Contadores de grupo incrementam mas nunca decrementam. |
| **[RULE-004]** | Skill Autonomy | Cada SKILL.md DEVE ser auto-contido — um agente IA lendo apenas o SKILL.md consegue executar a skill sem consultar documentacao externa. Incluir workflow completo, comandos, formatos de output e tratamento de erros. |
| **[RULE-005]** | Placeholder Tokens | Usar placeholders estabelecidos (`{{PROJECT_NAME}}`, `{{LANGUAGE}}`, `{{LANGUAGE_VERSION}}`, `{{BUILD_TOOL}}`, `{{FRAMEWORK}}`, `{{COMPILE_COMMAND}}`, `{{TEST_COMMAND}}`). Novos placeholders proibidos sem atualizar ContextBuilder. |
| **[RULE-006]** | Language Convention | Body do epico/historia em pt-BR. Termos tecnicos, IDs, codigo em English. Gherkin com keywords PT-BR (Cenario, DADO, QUANDO, ENTAO). |
