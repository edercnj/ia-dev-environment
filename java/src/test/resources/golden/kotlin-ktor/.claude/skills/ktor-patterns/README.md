# ktor-patterns

> Ktor-specific patterns: plugin installation, routing DSL, Exposed ORM, Koin DI, HOCON config, testApplication testing, StatusPages error handling.

| | |
|---|---|
| **Category** | Stack Pattern |

> **Full content**: See [SKILL.md](./SKILL.md) for the complete pattern reference.

## Key Patterns

- Ktor plugins (ContentNegotiation, CallLogging, StatusPages, CORS)
- Routing DSL with nested routes and route organization
- Data access with Exposed ORM (DSL and DAO patterns)
- Dependency injection with Koin modules and injection
- Configuration with HOCON and application.conf
- Testing with testApplication and HttpClient assertions
