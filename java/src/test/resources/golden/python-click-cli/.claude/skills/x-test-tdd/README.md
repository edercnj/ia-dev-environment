# x-test-tdd

> Executes systematic Red-Green-Refactor TDD cycles for a task. Reads the task plan generated by x-task-plan, runs each cycle in TPP order, validates RED/GREEN/REFACTOR phases, delegates atomic commits to x-git-commit with TDD tags, and supports resume and dry-run.

| | |
|---|---|
| **Category** | Development / TDD |
| **Invocation** | `/x-test-tdd TASK-XXXX-YYYY-NNN [--from-cycle N] [--dry-run]` |
| **Reads** | testing, coding-standards |

> **Spec**: See [SKILL.md](./SKILL.md) for the complete execution specification.

## What It Does

Drives the inner loop of Double-Loop TDD by executing systematic Red-Green-Refactor cycles for an individual task. Each cycle reads the task plan (generated by `x-task-plan`), writes a failing test (RED), implements the minimum code to pass (GREEN), optionally refactors (REFACTOR), and delegates each phase to `x-git-commit` with TDD tags. Cycles follow strict Transformation Priority Premise (TPP) ordering from degenerate to complex.

## Usage

```
/x-test-tdd TASK-0029-0001-001
/x-test-tdd TASK-0029-0001-001 --from-cycle 3
/x-test-tdd TASK-0029-0001-001 --dry-run
```

## Flags

| Flag | Default | Effect |
|------|---------|--------|
| `--from-cycle` | 1 | Resume execution from a specific cycle number (skip earlier cycles) |
| `--dry-run` | off | List all cycles with descriptions and TPP levels without executing |

## TDD Cycle Phases

| Phase | What Happens | Validation | Commit Tag |
|-------|-------------|------------|------------|
| RED | Write a failing test | Test MUST fail; if it passes, ABORT | `[TDD:RED]` |
| GREEN | Implement minimum code | Must compile and pass all tests | `[TDD:GREEN]` |
| REFACTOR | Improve design | Tests MUST still pass; revert if they fail | `[TDD:REFACTOR]` |

## Transformation Priority Premise (TPP)

Cycles are ordered from simple to complex transformations:

| Priority | Transformation | Description | Example |
|----------|---------------|-------------|---------|
| 1 | degenerate | {} -> nil | Method returns null/void/empty |
| 2 | constant | nil -> constant | Returns a fixed value |
| 3 | constant+ | constant -> constant+ | Returns another fixed value |
| 4 | scalar | constant -> scalar | Returns a variable |
| 5 | statements | statement -> statements | Adds instructions |
| 6 | conditional | unconditional -> conditional | Adds if/switch |
| 7 | collection | scalar -> collection | List/Map/Set |
| 8 | recursion | collection -> recursion | Iteration/recursion |

## Commit Format

Each phase produces an atomic commit via `/x-git-commit`:

| Phase | Commit Format |
|-------|--------------|
| RED | `test(TASK-XXXX-YYYY-NNN): add test for [description] [TDD:RED]` |
| GREEN | `feat(TASK-XXXX-YYYY-NNN): implement [description] [TDD:GREEN]` |
| REFACTOR | `refactor(TASK-XXXX-YYYY-NNN): [refactoring description] [TDD:REFACTOR]` |

## Validation Rules

- **RED that passes** = invalid cycle (ABORT) -- the test must define NEW behavior
- **GREEN with gold-plating** = violation -- implement only what the test requires
- **REFACTOR that fails** = automatic revert -- refactoring must preserve behavior
- **Non-TPP order** = warning -- cycles should progress from simple to complex

## Workflow

1. **Validate** -- Parse task ID, resolve task plan path
2. **Load** -- Read task plan, extract TDD Cycles section
3. **Execute** -- For each cycle: RED -> GREEN -> REFACTOR -> commit
4. **Report** -- Summary of cycles executed and commits created

## Outputs

| Artifact | Path |
|----------|------|
| Atomic commits | Git history with `[TDD:RED]`, `[TDD:GREEN]`, `[TDD:REFACTOR]` tags |
