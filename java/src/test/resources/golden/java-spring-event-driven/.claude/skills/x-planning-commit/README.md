# x-planning-commit

> Batch-commits planning artifacts under `plans/**` without triggering the code pre-commit chain (format/lint/compile). Sibling of `x-git-commit` for docs/markdown/json.

| | |
|---|---|
| **Category** | Git/Planning |
| **Invocation** | `/x-planning-commit --epic-id <NNNN> --paths <p1,p2,...> --subject <msg> [--scope chore\|docs] [--body <text>] [--dry-run]` |

> **Spec**: See [SKILL.md](./SKILL.md) for the complete execution specification.

## What It Does

Batch-commits the planning artifacts a planning skill just wrote (under `plans/epic-XXXX/**` or `.claude/templates/**`) in one atomic commit, without invoking the heavy code pre-commit chain (`x-code-format -> x-code-lint -> compile`) that `x-git-commit` enforces. RULE-007 of EPIC-0049 requires every planning skill (`x-epic-create`, `x-epic-decompose`, `x-epic-map`, `x-epic-orchestrate`, `x-story-create`, `x-story-plan`, `x-task-plan`) to delegate to this skill at end-of-run so the working tree stays clean and every plan mutation has a Git audit trail.

## Usage

```
/x-planning-commit --epic-id 0049 --paths plans/epic-0049/epic-0049.md --subject "add epic scaffold"
/x-planning-commit --epic-id 0049 --paths plans/epic-0049/epic-0049.md,plans/epic-0049/IMPLEMENTATION-MAP.md --subject "add planning artifacts (22 stories)"
/x-planning-commit --epic-id 0049 --paths plans/epic-0049/epic-0049.md --subject "preview" --dry-run
/x-planning-commit --epic-id 0049 --paths plans/epic-0049/epic-0049.md --subject "chore bump" --scope chore
```

## Workflow

1. **Validate args** -- `--epic-id` matches `^\d{4}$`; `--subject` ≤ 72 chars.
2. **Validate paths** -- each `--paths` entry exists AND is under `plans/**` or `.claude/templates/**` (whitelist).
3. **Dry-run branch** -- if `--dry-run`, print the message that would be created and exit 0 without staging.
4. **Stage** -- `git add <paths...>` (explicit list, never `git add .`).
5. **No-op branch** -- if `git diff --cached --quiet` returns 0, un-stage and exit 0 with `noOp=true`.
6. **Commit** -- `git commit -m "<scope>(epic-<NNNN>): <subject>\n\n<body>"` **without** the code pre-commit chain.
7. **Report** -- JSON line with `commitSha`, `filesCommitted`, `noOp`, `dryRun`.

## Commit Format

```
<scope>(epic-<NNNN>): <subject>

[optional body]
```

Example:

```
docs(epic-0049): add planning artifacts (22 stories)

Generated by /x-epic-decompose.
```

## Whitelist

| Prefix | Purpose |
|--------|---------|
| `plans/` | Epic / story / task plans, reports, execution-state, implementation map |
| `.claude/templates/` | Planning skill templates |

Any other prefix (e.g., `src/`, `pom.xml`, `.github/`) is REJECTED with `PATH_NOT_WHITELISTED`. This guarantees `x-planning-commit` cannot accidentally be used to commit source code and bypass the format/lint/compile chain.

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success (commit created, no-op, or dry-run) |
| 1 | `PATH_NOT_WHITELISTED` -- path outside `plans/**` / `.claude/templates/**` |
| 2 | `PATH_NOT_EXISTS` -- path missing on disk |
| 3 | `INVALID_EPIC_ID` -- `--epic-id` not 4 digits |
| 4 | `COMMIT_FAILED` -- `git commit` returned non-zero |
| 5 | `SUBJECT_TOO_LONG` -- subject > 72 chars |

## Outputs

| Artifact | Description |
|----------|-------------|
| Git commit | Single atomic commit with the declared paths (when not no-op / dry-run) |
| JSON report | One line: `{"commitSha", "filesCommitted", "noOp", "dryRun"}` |
