# Skill: TDD Execution with Atomic Cycles

## Purpose

Executes systematic Red-Green-Refactor TDD cycles for {{PROJECT_NAME}} tasks. Reads the task plan generated by x-plan-task, runs each TDD cycle in Transformation Priority Premise (TPP) order, validates each phase, and delegates atomic commits to x-commit with TDD tags.

## Triggers

- `/x-tdd TASK-XXXX-YYYY-NNN` -- execute all TDD cycles for a task
- `/x-tdd TASK-XXXX-YYYY-NNN --from-cycle 3` -- resume from cycle 3
- `/x-tdd TASK-XXXX-YYYY-NNN --dry-run` -- preview cycles without execution

## Arguments

| Argument | Type | Required | Description |
|----------|------|----------|-------------|
| `<task-id>` | String | Yes | Task ID in format TASK-XXXX-YYYY-NNN |
| `--from-cycle` | Integer | No | Resume from specific cycle number |
| `--dry-run` | Flag | No | Preview TDD cycles without executing |

## TDD Cycle Phases

```
1. RED       -> Write failing test (compile + run -> FAIL expected)
2. GREEN     -> Write minimal code to pass (run -> PASS expected)
3. REFACTOR  -> Improve design without changing behavior (run -> PASS)
4. COMMIT    -> Delegate to x-commit with TDD tag
```

## Transformation Priority Premise (TPP) Order

Tests are ordered from simplest transformation to most complex:
`{} -> nil -> constant -> constant+ -> scalar -> collection -> ...`

## Error Handling

| Scenario | Action |
|----------|--------|
| Task plan not found | ABORT with hint to run x-plan-task first |
| RED phase test passes | ABORT -- test must fail in RED phase |
| GREEN phase test fails | RETRY with more implementation |
| REFACTOR changes behavior | ABORT -- tests must stay green |

## Integration Notes

- Reads task plans generated by x-plan-task
- Delegates commits to x-commit with TDD tags (RED, GREEN, REFACTOR)
- Used by x-dev-lifecycle as the implementation engine
- Validates TDD compliance for quality gates (RULE-005)
