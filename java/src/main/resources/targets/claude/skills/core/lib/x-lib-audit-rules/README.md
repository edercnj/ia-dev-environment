# x-lib-audit-rules

> Audits compliance of all project rules AND knowledge packs against source code. Launches parallel subagents (one per rule/knowledge-pack) for scanning, then aggregates into a unified report with severity classification and story suggestions.

| | |
|---|---|
| **Category** | Library (internal) |
| **Called by** | x-code-audit, x-dev-story-implement (standalone usage supported) |

> **Spec**: See [SKILL.md](./SKILL.md) for the complete execution specification.

## Purpose

Performs full codebase compliance scanning against project rules and knowledge pack patterns. Discovers all rules and knowledge packs, launches parallel subagents for scanning, deduplicates findings, and produces a severity-categorized report with optional story creation for remediation.

## Integration Points

| Caller | Context | Input | Output |
|--------|---------|-------|--------|
| x-code-audit | Periodic codebase review | Scope flag (`--scope all\|rules\|patterns`), optional rule filter (`--rules 01,02`), optional `--fix` | Compliance report with executive summary, per-rule/KP findings, severity counts, and suggested stories |
| standalone | Ad-hoc compliance check | Same arguments as above | Same report format |

## Procedure

1. **Discover** -- list all rules from `.claude/rules/` and knowledge packs from skill reference directories, filtered by scope arguments
2. **Scan** -- launch one parallel subagent per rule and one per knowledge pack in a single message; each scans source code for violations
3. **Aggregate** -- collect subagent outputs, deduplicate (rule findings take precedence over KP findings for same file:line), classify by severity (CRITICAL/HIGH/MEDIUM/LOW), and generate the consolidated report
4. **Create stories** (if `--fix`) -- group violations into actionable stories for remediation via x-dev-story-implement or x-dev-implement
