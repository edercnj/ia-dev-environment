# STORY-0032-0004: Consolidate Naming — x-dev-lifecycle to x-dev-story-implement

## Status: Pendente

## Parent Epic: EPIC-0032

## Depends On: — (no dependencies, Phase 1)

## Summary

Consolidate the duplicate skill naming by making `x-dev-story-implement` the canonical name. Remove `x-dev-lifecycle` as a standalone skill and update all references in `x-dev-epic-implement` and other skills that point to the old name.

## User Story

As a **developer or skill author**, I want a single canonical name for the story implementation lifecycle skill, so that there is no confusion about which skill to invoke and no risk of invoking a stale version.

## Background

Two nearly identical skills exist:
- `core/x-dev-lifecycle/SKILL.md` — The older name, referenced by x-dev-epic-implement's subagent prompt
- `core/x-dev-story-implement/SKILL.md` — Newer version with `AskUserQuestion` in allowed-tools

The x-dev-epic-implement subagent prompt tells the subagent:
```
Skill(skill: "x-dev-lifecycle", args: "{storyId}")
```

But `x-dev-story-implement` is the more descriptive name and the more up-to-date version. This story consolidates to `x-dev-story-implement`.

## Acceptance Criteria

### AC-1: x-dev-epic-implement references x-dev-story-implement

```gherkin
Given the x-dev-epic-implement SKILL.md subagent prompt template
When the subagent dispatch instruction is generated
Then it MUST reference "x-dev-story-implement" not "x-dev-lifecycle":
  Skill(skill: "x-dev-story-implement", args: "{storyId}")
And the fallback instruction MUST also reference the correct name
```

### AC-2: x-dev-lifecycle content merged into x-dev-story-implement

```gherkin
Given x-dev-lifecycle and x-dev-story-implement SKILL.md files
When comparing their content
Then x-dev-story-implement MUST contain all features from both versions
And x-dev-story-implement MUST have "AskUserQuestion" in allowed-tools (from newer version)
And x-dev-story-implement MUST have all reference files from x-dev-lifecycle
```

### AC-3: x-dev-lifecycle removed or deprecated

```gherkin
Given the consolidation is complete
When I search for x-dev-lifecycle directory
Then one of:
  a) The directory is removed entirely (preferred)
  b) The SKILL.md contains only a redirect: "This skill has been renamed to x-dev-story-implement. Use Skill(skill: 'x-dev-story-implement', args: '...')"
```

### AC-4: No stale references remain

```gherkin
Given all skill files in core/
When I search for "x-dev-lifecycle" string
Then ZERO matches are found in delegation instructions
And ZERO matches in allowed-tools or skill-reference tables
And the only acceptable matches are in changelog/migration notes
```

### AC-5: Reference files consolidated

```gherkin
Given x-dev-lifecycle has references/:
  - planning-phases.md
  - verification-phase.md
  - scope-assessment.md
  - openapi-generator.md
And x-dev-story-implement has references/:
  - planning-phases.md
  - verification-phase.md
  - scope-assessment.md
  - openapi-generator.md
When consolidated
Then x-dev-story-implement/references/ MUST contain the most up-to-date version of each file
And x-dev-lifecycle/references/ is removed (if directory is removed)
```

### AC-6: Golden files updated

```gherkin
Given the naming change
When golden files are regenerated for all profiles
Then golden files MUST NOT contain "x-dev-lifecycle" in skill references
And golden files MUST reference "x-dev-story-implement" where applicable
```
