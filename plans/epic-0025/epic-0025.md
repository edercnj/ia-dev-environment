# Épico: Platform Target Filter — Geração Seletiva por Plataforma de IA

**Autor:** Product Owner
**Data:** 2026-04-06
**Versão:** 1.0
**Status:** Em Refinamento

## 1. Visão Geral

**Chave Jira:** —

O comando `ia-dev-env generate` atualmente executa todos os 33 assemblers em sequência, produzindo artefatos para Claude Code (`.claude/`), GitHub Copilot (`.github/`), OpenAI Codex (`.codex/`, `.agents/`) e documentação (ROOT) simultaneamente. A maioria dos usuários utiliza apenas uma ferramenta de IA, resultando em diretórios desnecessários, tempo de geração desperdiçado e documentação que reflete plataformas irrelevantes. Este épico adiciona a flag `--platform` ao CLI e a seção `platform:` ao YAML config, permitindo gerar apenas os artefatos da(s) plataforma(s) selecionada(s). O default é `all` para garantir retrocompatibilidade total. As plataformas suportadas são: `claude-code`, `copilot`, `codex` e `all`.

## 4. Regras de Negócio Transversais (Source of Truth)

> Regras que se aplicam a múltiplas histórias. Cada história referencia as regras pelo ID. Alterações de regra propagam automaticamente para todas as histórias dependentes.

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-001]** | Retrocompatibilidade Total | O comportamento default (`--platform all` ou sem flag quando `platform` não está no YAML) DEVE reproduzir exatamente o mesmo conjunto de artefatos que a versão atual. Nenhum teste existente pode quebrar. Se `platforms` está vazio ou contém todos os valores, nenhum filtro é aplicado. |
| **[RULE-002]** | Ordem de Assemblers Preservada | A ordem dos 33 assemblers (RULE-005 do projeto) deve ser mantida. A filtragem apenas pula assemblers, nunca reordena. O método `buildAssemblers()` retorna a lista na mesma sequência, com os assemblers filtrados removidos. |
| **[RULE-003]** | Shared é Sempre Incluído | Os 13 assemblers de documentação e CI/CD (target ROOT) que não são específicos de nenhuma plataforma devem executar SEMPRE, independente da seleção: ConstitutionAssembler, DocsAssembler, GrpcDocsAssembler, RunbookAssembler, IncidentTemplatesAssembler, ReleaseChecklistAssembler, OperationalRunbookAssembler, SloSliTemplateAssembler, DocsContributingAssembler, DataMigrationPlanAssembler, CicdAssembler, EpicReportAssembler, DocsAdrAssembler. |
| **[RULE-004]** | CLI Tem Precedência sobre YAML | Se `--platform` é fornecido na CLI E `platform:` existe no YAML config, a flag CLI prevalece. A resolução ocorre em `GenerateCommand` antes de criar `PipelineOptions`. |
| **[RULE-005]** | Validação de Valores | Valores inválidos para `--platform` devem falhar com mensagem clara listando os valores aceitos (`claude-code`, `copilot`, `codex`, `all`). A validação ocorre via `ITypeConverter<Platform>` do Picocli, antes do pipeline. |
| **[RULE-006]** | Contagens Dinâmicas | O README.md e CLAUDE.md gerados devem refletir APENAS os artefatos da plataforma selecionada. A seção "Generation Summary" deve ser condicional. A tabela de mapeamento cross-platform só aparece quando mais de uma plataforma é selecionada. |
| **[RULE-007]** | Dry-Run Respeita Filtro | O modo `--dry-run` deve listar apenas os assemblers que seriam executados para a plataforma selecionada, produzindo a lista filtrada de arquivos. |
| **[RULE-008]** | Verbose Respeita Filtro | O modo `--verbose` deve exibir quais assemblers foram incluídos e quais foram filtrados, com a razão (plataforma). Formato: `INCLUDED: RulesAssembler (platform: claude-code)` / `SKIPPED: GithubSkillsAssembler (platform: copilot)`. |
| **[RULE-009]** | Composição de Plataformas | `--platform claude-code,copilot` deve executar assemblers de AMBAS as plataformas (união), mais os `shared`. A lógica de inclusão verifica se a interseção entre `descriptor.platforms` e `options.platforms` é não-vazia. |
| **[RULE-010]** | Enum Extensível | O design deve permitir adicionar novas plataformas no futuro (ex: `cursor`, `windsurf`) sem modificar lógica existente. Cada nova plataforma requer apenas: (1) novo valor no enum, (2) atribuição nos assemblers relevantes da factory, (3) novo kebab-case no converter CLI. |
