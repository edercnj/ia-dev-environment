## 3. Descrição

Como **mantenedor do `ia-dev-env`**, eu quero atualizar em massa todas as skills
consumidoras (`java/src/main/resources/targets/claude/skills/core/**/SKILL.md`)
que hoje citam `skills/{kp}/SKILL.md` para passar a citar
`knowledge/{kp}.md`, de modo que, após a migração da source-of-truth (STORY-0051-0002),
as skills resolvam corretamente seus KPs no novo diretório.

Esta é uma edição mecânica regex-based sobre ~30 `SKILL.md` em `core/`. A
auditoria final é um grep: `grep -rln "skills/\(architecture\|patterns\|security\|...\)" core/`
retorna zero. Roda em paralelo com STORY-0051-0004 (rules) porque os arquivos
tocados são disjuntos.

### 3.1 Inventário de consumidores (aproximado)

Executar antes da implementação:

```bash
grep -rln "skills/\(api-design\|architecture\|architecture-cqrs\|architecture-hexagonal\|architecture-patterns\|ci-cd-patterns\|coding-standards\|compliance\|data-management\|data-modeling\|database-patterns\|ddd-strategic\|disaster-recovery\|feature-flags\|finops\|infra-patterns\|infrastructure\|layer-templates\|observability\|owasp-asvs\|parallelism-heuristics\|patterns\|patterns-outbox\|pci-dss-requirements\|performance-engineering\|protocols\|release-management\|resilience\|security\|sre-practices\|stack-patterns\|testing\)/SKILL\.md" \
  java/src/main/resources/targets/claude/skills/core/ --include=SKILL.md
```

Resultado esperado: lista de ~30 arquivos. Salvar em
`plans/epic-0051/skill-consumer-inventory.txt`.

### 3.2 Substituição canônica

Para cada ocorrência em cada SKILL.md:

| Antes | Depois |
| :--- | :--- |
| `skills/architecture/SKILL.md` | `knowledge/architecture.md` |
| `skills/patterns/SKILL.md` | `knowledge/patterns/index.md` (pasta) OU `knowledge/patterns.md` (arquivo) — conforme decisão da STORY-0051-0002 |
| `skills/{kp}/references/foo.md` | `knowledge/{kp}/foo.md` |

A resolução arquivo-vs-pasta vem do inventário gerado na STORY-0051-0002.

### 3.3 Casos excluídos (não tocar)

- Ocorrências em seções `## Triggers` e `## Examples` de qualquer SKILL.md
  (exceção Rule 13) — mas essas seções referenciam comandos `/x-*`, não KPs;
  na prática não há ocorrência a preservar.
- Ocorrências em `skills/lib/**` — `lib/` hospeda utilitários e nunca
  referencia KPs (verificado previamente).
- Referências a `skills/{nome}/SKILL.md` onde `{nome}` **não** está no
  inventário de RULE-051-04 (ex.: `skills/x-test-tdd/SKILL.md` é uma skill
  real, não um KP — fica intocada).

## 3.5 Entrega de Valor

- **Valor Principal:** Todas as ~30 skills `x-*` resolvem seus KPs no novo
  path sem regressão funcional.
- **Métrica de Sucesso:** Grep invariante retorna 0; `mvn process-resources`
  verde; smoke test invocando ao menos 3 skills (ex.: `/x-arch-plan`,
  `/x-review-pr`, `/x-story-plan`) completa sem erro de path.
- **Impacto no Negócio:** Usuários do `ia-dev-env` (dev Claude Code) continuam
  tendo skills funcionais após a migração.
