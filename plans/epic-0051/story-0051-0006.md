## 3. Descrição

Como **mantenedor do `ia-dev-env`**, eu quero formalizar a decisão arquitetural
(ADR), remover o código legado de KP em `SkillsAssembler` /
`SkillsCopyHelper` / `KnowledgePackSelection`, e publicar entrada em
`CHANGELOG.md`, para encerrar o épico com uma superfície limpa e histórico
documental coerente.

Esta é a última story. Remove toda a lógica que tratava KPs como caso especial
do `SkillsAssembler`, publica o ADR na próxima numeração livre, atualiza
`CLAUDE.md` (root) removendo menções a "fake skills" / "knowledge-packs dentro
de skills/", e adiciona entrada `Changed` no `CHANGELOG.md`.

### 3.1 ADR

Criar em `adr/ADR-00XX-knowledge-packs-dedicated-dir.md` (próxima numeração
após o último ADR existente — escanear `adr/` no commit base).

Conteúdo (template canônico do projeto):

- **Context:** "fake skill" problem, RULE-051-07 contrato do diretório.
- **Decision:** Alternativa C — `.claude/knowledge/` dedicado.
- **Consequences:**
  - Positivas: semântica correta; Claude Code não escaneia; validação
    independente; alinhamento com `.claude/rules/`.
  - Negativas: breaking change no layout gerado (bump MINOR do `ia-dev-env`);
    projetos legados precisam regenerar.
- **Alternatives Considered:** A (status quo), B (subpasta `_kp/`),
  D (plugin MCP), E (references/ por skill) — registradas em
  `plans/epic-0051/spec-epic-0051.md`.
- **Related:** EPIC-0051, `plans/epic-0051/`.

### 3.2 Cleanup de `SkillsAssembler` e afins

Remover de `SkillsAssembler.java`:

- Qualquer método que itere sobre `knowledge-packs/` em `targets/`.
- Qualquer comentário / javadoc mencionando knowledge packs.

Remover de `SkillsCopyHelper.java`:

- Lógica específica para `infra-patterns` / `stack-patterns` que opera
  sob `knowledge-packs/`. Se `infra-patterns` e `stack-patterns` são KPs
  condicionais selecionados por stack, migrar a **seleção** para o novo
  `KnowledgeAssembler` (ou um helper `KnowledgePackSelection` reescrito).

Reescrever ou remover `KnowledgePackSelection.java`:

- Se a lógica de seleção por stack ainda é necessária para o novo fluxo,
  mantê-la mas adaptada à API do `KnowledgeAssembler`.
- Se a seleção se torna trivial (todos os KPs copiados incondicionalmente),
  remover completamente e documentar a simplificação no ADR.
