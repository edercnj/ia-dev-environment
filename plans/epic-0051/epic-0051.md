**Status:** Em Refinamento
**Épico ID:** EPIC-0051

## 1. Visão Geral

**Chave Jira:** —

Os Knowledge Packs (KPs) do projeto `ia-dev-environment` vivem hoje em
`.claude/skills/{nome}/SKILL.md`, lado a lado com skills invocáveis. Isso trata
KPs como "fake skills": eles só estão ali porque Claude Code escaneia
`.claude/skills/**/SKILL.md`; não são invocáveis, não têm triggers úteis e
violam o contrato mínimo de skill (ex.: `.claude/skills/patterns/SKILL.md` é
Markdown puro sem frontmatter algum). O resultado é ruído semântico (~32 pastas
"skills" que não são skills), validação frágil (cada assembler precisa
reconhecer o caso especial) e confusão em documentação.

Este épico move todos os KPs para um diretório dedicado
`.claude/knowledge/`, irmão de `skills/` e `rules/`, com arquivos `.md` puros
(frontmatter opcional). O assembler dedicado `KnowledgeAssembler.java` (espelha
`RulesAssembler`) substitui a lógica que hoje vive em `SkillsAssembler` /
`SkillsCopyHelper` / `KnowledgePackSelection`. As ~30 skills consumidoras em
`core/**/SKILL.md` que hoje fazem `Read("skills/{nome}/SKILL.md")` passam a
apontar para `knowledge/{nome}.md`; as 7 rules que citam KPs como "Full
reference" são atualizadas em massa.

Fora de escopo: alterar **conteúdo** de qualquer KP; introduzir camada de
compatibilidade por path antigo (migração big-bang dentro do epic); implementar
um MCP resource server (overengineering rejeitado); tocar na API pública de
qualquer skill `x-*`; migrar projetos legados já gerados por versões antigas
do `ia-dev-env` (usuário regenera via `ia-dev-env generate`).

## 4. Regras de Negócio Transversais (Source of Truth)

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-051-01]** | Source-of-truth única para KPs | A fonte canônica para Knowledge Packs é `java/src/main/resources/targets/claude/knowledge/`. O diretório antigo `…/skills/knowledge-packs/` é removido ao final do épico. Nenhum KP pode existir simultaneamente em ambos os locais — `KnowledgeMigrationInvariantTest` falha se detectar coexistência.<br>Aplicabilidade: todas as stories. Prioridade máxima — viola a razão de ser do épico se ignorada. |
| **[RULE-051-02]** | Assembler dedicado (`KnowledgeAssembler`) | Um novo `dev.iadev.application.assembler.KnowledgeAssembler` (≤ 250 linhas, Rule 03) copia verbatim de `targets/claude/knowledge/` para `.claude/knowledge/`. `SkillsAssembler`, `SkillsCopyHelper` e `KnowledgePackSelection` **não podem mais conhecer KPs** ao final do épico — qualquer referência residual é violação.<br>Aplicabilidade: STORY-0051-0001, STORY-0051-0006. |
| **[RULE-051-03]** | Path canônico `knowledge/{nome}.md` | Toda skill, rule ou documentação que hoje aponta para `skills/{nome}/SKILL.md` (onde `{nome}` é um KP listado em RULE-051-04) passa a apontar para `knowledge/{nome}.md`. KPs com bundled references viram `knowledge/{nome}/` (pasta) com `index.md` + auxiliares; a decisão por KP é registrada na STORY-0051-0002.<br>Aplicabilidade: STORY-0051-0003, STORY-0051-0004. |
| **[RULE-051-04]** | Inventário canônico de KPs | A lista autoritativa de KPs migrados é derivada de `ls java/src/main/resources/targets/claude/skills/knowledge-packs/` no commit base do épico. Cobre ao menos: `architecture`, `architecture-cqrs`, `architecture-hexagonal`, `architecture-patterns`, `api-design`, `ci-cd-patterns`, `coding-standards`, `compliance`, `data-management`, `data-modeling`, `database-patterns`, `ddd-strategic`, `disaster-recovery`, `feature-flags`, `finops`, `infra-patterns`, `infrastructure`, `layer-templates`, `observability`, `owasp-asvs`, `parallelism-heuristics`, `patterns-outbox`, `pci-dss-requirements`, `performance-engineering`, `protocols`, `release-management`, `resilience`, `security`, `sre-practices`, `stack-patterns`, `testing`. | **[RULE-051-05]** | Migração big-bang por PR de épico | Não há camada de compatibilidade por path antigo. Branches abertas durante o épico que referenciem `skills/{kp}/` passam a falhar no rebase — aceitável porque EPIC-0051 é infra. A PR final do épico é a migração atômica; merges parciais em `develop` são proibidos (ver seção 6 de `spec-epic-0051.md`).<br>Aplicabilidade: todas as stories; fiscalizada pelo processo de review. |
| **[RULE-051-06]** | Hotspot declarado em RULE-004 | `KnowledgeAssembler.java` entra no catálogo de hotspots de EPIC-0041 (Rule 04 / `plans/epic-0041/`). Stories 0051-0001 e 0051-0006 declaram-no no footprint. `/x-parallel-eval --scope=epic` rodado antes da PR final não pode reportar colisão não mitigada.<br>Aplicabilidade: STORY-0051-0001, STORY-0051-0006. |
| **[RULE-051-07]** | Contrato do novo diretório | `.claude/knowledge/` contém arquivos `.md` em kebab-case. Frontmatter é opcional; quando presente aceita apenas `name`, `description` e `tags` — **não** aceita `user-invocable`, `allowed-tools` ou `argument-hint` (não é skill). `KnowledgeAssembler` reprova (erro explícito, não warning) qualquer KP fonte que declare um desses campos.<br>Aplicabilidade: STORY-0051-0001, STORY-0051-0002. |
| **[RULE-051-08]** | Goldens regenerados em uma única story | Toda a regeneração de `src/test/resources/golden/` é concentrada na STORY-0051-0005 para evitar merges de goldens divergentes. Stories anteriores declaram `regen: src/test/resources/golden/**` no footprint e aceitam `/x-parallel-eval` degradar a execução para serial, se necessário.<br>Aplicabilidade: STORY-0051-0005; referenciada por todas. |
