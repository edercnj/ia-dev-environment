## 3. Descrição

Como **mantenedor do `ia-dev-env`**, eu quero atualizar as 7 rules em
`java/src/main/resources/targets/claude/rules/` que citam `skills/{kp}/SKILL.md`
como "Full reference" para passar a citar `knowledge/{kp}.md`, garantindo
coerência documental entre source-of-truth e artefatos gerados.

Roda em paralelo com STORY-0051-0003 — arquivos tocados são disjuntos
(`rules/` vs `skills/core/`). `/x-parallel-eval` confirma ausência de colisão.

### 3.1 Inventário das rules afetadas

Pré-confirmado no spec (§3 do epic):

| Rule | Referência atual |
| :--- | :--- |
| `03-coding-standards.md` | `skills/coding-standards/SKILL.md`, `skills/testing/SKILL.md` |
| `04-architecture-summary.md` | `skills/architecture/SKILL.md`, `skills/layer-templates/SKILL.md` |
| `05-quality-gates.md` | `skills/testing/SKILL.md` |
| `06-security-baseline.md` | `skills/security/SKILL.md` |
| `07-operations-baseline.md` | `skills/observability/SKILL.md` |
| `08-release-process.md` | `skills/protocols/SKILL.md` |
| `09-branching-model.md` | `skills/protocols/SKILL.md` |

Total: 7 rules, ~10 ocorrências.

### 3.2 Substituição canônica

| Antes | Depois |
| :--- | :--- |
| `` `skills/testing/SKILL.md` `` | `` `knowledge/testing.md` `` |
| `` `skills/layer-templates/SKILL.md` `` | `` `knowledge/layer-templates.md` `` |
| etc. | etc. |

A resolução arquivo-vs-pasta segue a mesma decisão da STORY-0051-0002 /
STORY-0051-0003.

### 3.3 Fora de escopo

- Rule 13 cita `/x-pr-fix`, `/x-pr-watch-ci` etc. — **não mexer**.
- Rule 20 (interactive gates) e Rule 21 (CI-Watch) não citam KPs — **não mexer**.
- Rule 02 (`02-domain.md`) é template com placeholders `{PLACEHOLDER}` — **não mexer**.

## 3.5 Entrega de Valor

- **Valor Principal:** Rules apontam para o path correto, preservando a
  navegabilidade humana e a integridade do contrato documental.
- **Métrica de Sucesso:** Grep `skills/\({lista}\)/SKILL.md` em `rules/` retorna 0;
  `mvn process-resources` verde; `.claude/rules/*.md` gerado reflete as alterações.
- **Impacto no Negócio:** Desenvolvedores (usuários finais do `ia-dev-env`)
  continuam conseguindo seguir os ponteiros "Full reference" das rules.
