## 3. Descrição

Como **mantenedor do `ia-dev-env`**, eu quero mover todos os ~32 KPs de
`targets/claude/skills/knowledge-packs/{nome}/SKILL.md` para
`targets/claude/knowledge/{nome}.md`, normalizando o formato (kebab-case,
frontmatter opcional e conformante com RULE-051-07), para que o
`KnowledgeAssembler` entregue na STORY-0051-0001 possa gerar `.claude/knowledge/`
no próximo `mvn process-resources`.

Esta é a story de migração mecânica mais volumosa do épico. Três decisões
técnicas registradas aqui:

1. **KPs com `references/` bundled:** viram `targets/claude/knowledge/{nome}/`
   (pasta) com `index.md` (conteúdo do antigo SKILL.md) + os arquivos de
   `references/*.md` como irmãos. Alternativa de inlining rejeitada para
   preservar separação `quick-reference` vs `deep-dive`.
2. **Limpeza de frontmatter:** `user-invocable: false`, `allowed-tools`,
   `argument-hint`, `context-budget` removidos (contrato novo, RULE-051-07).
   Mantidos: `name`, `description`. `tags` adicionado quando fizer sentido.
3. **Persistência da lista:** evidência da lista real (ls no commit base)
   salva em `plans/epic-0051/kp-inventory.txt`.

### 3.1 Processo de migração

Para cada `{nome}` em `ls targets/claude/skills/knowledge-packs/`:

1. Se `{nome}/references/` **não existe**:
   - Ler `{nome}/SKILL.md`.
   - Remover campos proibidos do frontmatter.
   - Escrever em `targets/claude/knowledge/{nome}.md`.
2. Se `{nome}/references/` **existe**:
   - Criar dir `targets/claude/knowledge/{nome}/`.
   - `SKILL.md` → `knowledge/{nome}/index.md` (frontmatter limpo).
   - `references/*.md` → `knowledge/{nome}/*.md` (verbatim).
3. Remover `targets/claude/skills/knowledge-packs/{nome}/` após verificar
   cópia (diff byte-a-byte do corpo, ignorando frontmatter).

### 3.2 `KnowledgePackSelection` — handling

O arquivo `java/src/main/java/dev/iadev/application/assembler/KnowledgePackSelection.java`
hoje filtra KPs por stack. A lógica de seleção é preservada, mas passa a operar
sobre `targets/claude/knowledge/` via `KnowledgeAssembler`. A remoção completa
da integração com `SkillsAssembler` é feita na STORY-0051-0006 (cleanup) para
manter a migração isolada de refactors colaterais.

## 3.5 Entrega de Valor

- **Valor Principal:** Arquivos dos ~32 KPs residem no novo path com formato
  normalizado; `KnowledgeAssembler` passa a gerar saída real em `.claude/knowledge/`.
- **Métrica de Sucesso:** `ls targets/claude/skills/knowledge-packs/` retorna
  zero entradas ao final; `ls targets/claude/knowledge/` lista ao menos os 32
  KPs de RULE-051-04; `mvn process-resources` gera `.claude/knowledge/` com
  mesma quantidade de arquivos top-level do que o inventário.
- **Impacto no Negócio:** Desbloqueia stories 0003 e 0004 (retrofits em
  paralelo).
