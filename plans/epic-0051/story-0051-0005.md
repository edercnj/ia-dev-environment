## 3. Descrição

Como **mantenedor do `ia-dev-env`**, eu quero concentrar a regeneração de todos
os arquivos em `src/test/resources/golden/` em uma única story, após as
migrações das stories 0051-0001 a 0051-0004, garantindo um diff de goldens
limpo e reviewável e um smoke test end-to-end que prove a migração funciona.

Stories anteriores acumulam mudanças que afetam goldens (novo assembler, novo
dir `.claude/knowledge/`, ausência de `.claude/skills/{kp}/`, rules editadas,
skills editadas). Regeneração única evita conflitos em merges intermediários e
produz um commit atômico revisável como unidade ("todos os goldens do épico").

### 3.1 Procedimento de regeneração

Seguir o runbook canônico (memory `reference_golden_regen_command.md`):

```bash
cd java
mvn process-resources       # pré-requisito obrigatório
mvn -Dtest=GoldenFileRegenerator test
git status src/test/resources/golden/
```

Esperado:

- `.claude/skills/{kp}/SKILL.md` × ~32 — **deletados**.
- `.claude/knowledge/*.md` ou `.claude/knowledge/{kp}/index.md` × ~32 —
  **criados**.
- `.claude/skills/core/**/SKILL.md` — com os deltas do retrofit (STORY-0051-0003).
- `.claude/rules/0[3-9]-*.md` — com os deltas do retrofit (STORY-0051-0004).
- `.claude/README.md` — atualizado automaticamente pelo assembler (seção
  "Knowledge Packs" refletindo o novo layout).

### 3.2 Smoke test `KnowledgePackMigrationSmokeTest`

Cobre a invariante de ponta a ponta:

1. `mvn process-resources` gera `.claude/` completo.
2. Para cada `{kp}` no `kp-inventory.txt`:
   - `.claude/knowledge/{kp}.md` OU `.claude/knowledge/{kp}/index.md` existe.
   - Frontmatter não declara `user-invocable`, `allowed-tools`, etc.
3. `.claude/skills/{kp}/SKILL.md` **não** existe para nenhum KP.
4. Para 3 skills consumidoras amostradas (`x-arch-plan`, `x-review-pr`,
   `x-story-plan`), o SKILL.md gerado em `.claude/skills/` referencia apenas
   paths `knowledge/*.md` (não `skills/{kp}/SKILL.md`).
5. Rule 03, 04, 08 geradas em `.claude/rules/` referenciam `knowledge/*.md`.

### 3.3 Auditorias legadas

Rodar e validar verde:

- `mvn test` completo.
- `scripts/audit-interactive-gates.sh --baseline`.
- `scripts/audit-rule-20.sh`.
- Rule 13 grep (seção "Audit Command" de `13-skill-invocation-protocol.md`).

Nenhuma dessas auditorias deve regir — se alguma regredir, é bug em story
anterior e deve ser consertado aqui.

## 3.5 Entrega de Valor

- **Valor Principal:** Build 100% verde com goldens atualizados; prova
  automatizada de que a migração funciona end-to-end.
- **Métrica de Sucesso:** `mvn test` verde; `KnowledgePackMigrationSmokeTest`
  passa; `git diff --stat src/test/resources/golden/` mostra um único commit
  atômico com o delta esperado.
- **Impacto no Negócio:** Confiança para merge — a PR do épico chega à review
  final com sinal verde automático.
