## 1. Visão Geral

**Chave Jira:** —

Este épico endereça vulnerabilidades comprovadamente exploráveis no protocolo de execução do framework `ia-dev-environment`, diagnosticadas durante o pós-mortem do EPIC-0053. A skill `x-pr-watch-ci` foi silenciosamente pulada em 6 task-PRs e no PR final #619 sem que qualquer camada de enforcement da Rule 24 detectasse a omissão. Análise exaustiva do catálogo identificou 14 gaps análogos em orquestradoras principais, todos causados por: (a) tabela "Mandatory Evidence Artifacts" da Rule 24 incompleta com apenas 5 entradas de 11 sub-skills auditáveis; (b) invocações em prosa sem marker `MANDATORY — NON-NEGOTIABLE` em ao menos 8 pontos críticos; (c) infraestrutura de enforcement (Camada 3 CI audit) inexistente. O épico realiza expansão aditiva da Rule 24, cria a Rule 45 (CI-Watch Integrity), implementa o script de auditoria da Camada 3, e promove smoke tests críticos para o ciclo local. **Fora de escopo:** refatoração profunda de skills, alterações em ADR-0010/ADR-0012, e adição de código Java runtime (Rule 14).

## 3. Definições de Qualidade Globais

### Global Definition of Ready (DoR)

- [ ] Story tem contexto técnico claro e referências às rules aplicáveis identificadas
- [ ] Dependências estão mapeadas e todas as histórias predecessoras estão `Concluída`
- [ ] Artefatos-alvo identificados (SKILL.md, scripts bash, regras) com paths absolutos
- [ ] Critérios de aceite Gherkin escritos e revisados antes do início
- [ ] Ambiente de desenvolvimento configurado e `mvn verify` passando no branch base

### Global Definition of Done (DoD)

- **Cobertura:** ≥ 95% Line, ≥ 90% Branch (absolute gate — Rule 05).
- **Testes Automatizados:** Cada história DEVE ter pelo menos 1 teste automatizado validando o critério de aceite principal. Tipos obrigatórios variam por história (ver DoD Local).
- **Smoke Tests:** Obrigatório — `mvn verify` incluindo smoke tests deve passar antes do PR.
- **Relatório de Cobertura:** JaCoCo em formato XML+HTML; disponível no artefato CI.
- **Documentação:** Source of truth atualizada (`java/src/main/resources/targets/claude/`); golden files regenerados via `mvn process-resources` + `GoldenFileRegenerator`.
- **Persistência:** N/A — este épico não introduz persistência de dados de negócio.
- **Performance:** Scripts bash devem completar em < 5s para repositórios com até 100 epics.
- **TDD Compliance:** Commits mostram padrão test-first. Refactoring explícito após green. Testes incrementais (simples a complexos via TPP — Transformation Priority Premise).
- **Double-Loop TDD:** Acceptance tests derivados de cenários Gherkin (outer loop). Unit tests guiados por TPP (inner loop).
- **Rule 21:** Story PR targeta `epic/0057` com auto-merge habilitado.
- **Rule 24 Retroativo:** Story produz evidência completa: `review-story-XXXX.md`, `techlead-review-story-XXXX.md`, `verify-envelope-STORY-ID.json`, `story-completion-report-STORY-ID.md`.

## 4. Regras de Negócio Transversais (Source of Truth)

> Regras que se aplicam a múltiplas histórias. Cada história referencia as regras pelo ID.

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-001]** | Sub-skills declaradas em SKILL.md são tool calls obrigatórias | Qualquer `Skill(skill: "...", ...)` declarado no corpo de uma SKILL.md é uma chamada de ferramenta (tool call), nunca prosa ou descrição conceitual. O LLM NÃO DEVE sumarizar, simular, ou pular sub-skills declaradas. Único bypass legítimo: flags `--skip-*` explícitas passadas pelo caller. Violação: PROTOCOL_VIOLATION silencioso. |
| **[RULE-002]** | Tabela "Mandatory Evidence Artifacts" é fonte da verdade para Camada 3 | A tabela da Rule 24 §32–42 define quais sub-skills produzem artefatos de evidência persistente. Toda sub-skill que produz arquivo auditável DEVE estar na tabela com path e camada de enforcement. Ausência implica cegueira das Camadas 3 e 4. |
| **[RULE-003]** | Enforcement via scripts Bash — sem código Java runtime (Rule 14) | Scripts de auditoria (`scripts/audit-*.sh`) são implementados em Bash. Nenhuma classe Java runtime é introduzida neste épico. Rule 14 (Project Scope Guard) se aplica integralmente: código Java só é permitido se servir ao pipeline de geração (`ia-dev-env generate`/`validate`). |
| **[RULE-004]** | Markers `MANDATORY — NON-NEGOTIABLE` obrigatórios em blocos de invocação | Blocos `Skill(skill: "...", ...)` em orchestrators DEVEM ser precedidos de frase explícita `**MANDATORY TOOL CALL — NON-NEGOTIABLE (Rule 24)**`. Invocação em prosa (sem o marker) é violação de Rule 24 §Camada-1. |
| **[RULE-005]** | Rule 21 — Story PRs targetam epic/0057; gate final para develop é manual | Todas as story PRs deste épico targetam `epic/0057` (auto-merge habilitado). O PR final `epic/0057 → develop` é gate manual (human reviewer). Nenhuma story dá merge direto em `develop`. |
| **[RULE-006]** | Rule 22 — Skill Visibility: internal skills referenciadas pelo nome canônico | Skills `x-internal-*` são invocadas apenas via `Skill(skill: "x-internal-...", ...)` por outras skills. Nenhuma skill interna aparece em docs de usuário ou é invocada por CI scripts diretamente. |
