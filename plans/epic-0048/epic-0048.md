# Épico: Java-Only Generator + Correção de Bugs A (pastas vazias) e B (CLAUDE.md raiz)

**Autor:** Eder Celeste Nunes Junior
**Data:** 2026-04-16
**Versão:** 1.0
**Status:** Em Refinamento
**Épico ID:** EPIC-0048

## 1. Visão Geral

**Chave Jira:** —

O gerador `ia-dev-env` neste repositório (`<repo-root>`) suporta hoje 6 linguagens de programação (java, python, go, kotlin, typescript, rust) com suporte parcial a csharp/.NET. Na prática 100% do uso é Java e a manutenção multi-linguagem gera custo constante: ~2.835 arquivos golden não-Java em `java/src/test/resources/golden/`, 25+ smoke tests parametrizados por 17 perfis via `SmokeProfiles#profiles`, duplicação de agents/hooks/rules/knowledge-packs por linguagem e débito técnico acumulado (`csharp-dotnet` leftover em `StackMapping.java` sem perfil/golden correspondente).

Simultaneamente, dois bugs de geração foram confirmados pelo autor em execução real da CLI: **Bug A** — `ia-dev-env generate` cria diretórios como `.github/`, `.codex/`, `.cursor/` vazios no projeto gerado (raiz provável: `CopyHelpers.copyDirectory#preVisitDirectory` chama `Files.createDirectories` antes de saber se haverá conteúdo útil); **Bug B** — `FileCategorizer.isRootFile` (`java/src/main/java/dev/iadev/cli/FileCategorizer.java:88`) reconhece `CLAUDE.md` como arquivo raiz, mas nenhum assembler o produz (não existe `ClaudeMdAssembler`, nem template `shared/templates/CLAUDE.md`). Resultado: projetos gerados ficam sem o sumário executivo auto-loaded que é contrato padrão de projetos Claude-Code.

Este épico entrega três mudanças em **13 stories** sequenciadas em ordem topológica (~85h): (1) **Escopo Java-only** — remover python, go, kotlin, typescript, rust, csharp de todos os mapeamentos, templates, skills, rules, goldens, smoke tests e YAMLs (bancos/mensageria/arquitetura/interface types/compliance **permanecem intactos** por serem ortogonais); (2) **Fix Bug A** — invariante `OutputDirectoryIntegrityTest` (RED-first) + fix estrutural em `CopyHelpers` + `pruneEmptyDirs` pós-assembly em `AssemblerPipeline` + regeneração dos 9 goldens Java; (3) **Fix Bug B** — novo `ClaudeMdAssembler` dedicado (single-responsibility, decisão arquitetural confirmada em ADR-0048-B), template Pebble `shared/templates/CLAUDE.md`, registro em `AssemblerFactory` como último grupo `buildRootDocAssemblers`, atualização controlada dos 9 goldens Java com o novo arquivo.

Fora de escopo: introdução de novas linguagens; revamp do pipeline de assembly; telemetria; mudanças em rules de arquitetura/compliance/databases/mensageria; renomear classes/pacotes; alterar thresholds JaCoCo. Release **v4.0.0 (MAJOR, breaking)**: usuários que precisam de python/go/kotlin/typescript/rust pinam v3.x; branch `legacy/v3` mantida read-only; documentado em ADR-0048-A e release notes. Precondição dura: STORY-0048-0001 (investigação) é gate — nenhuma remoção inicia antes do inventário canônico + repro-bugs mergearem (RULE-048-08).

## 3. Definições de Qualidade Globais

### Global Definition of Ready (DoR)

- Baseline verde em `develop`: `mvn clean verify` passa com coverage ≥95% line / ≥90% branch antes da primeira story.
- Branch `feature/epic-0048-java-only-generator` criada a partir de `develop` atualizado.
- Tag `pre-epic-0048-java-only` criada em `develop` (rollback anchor) antes do merge da STORY-0048-0001.
- Branch `backup/pre-epic-0048` congelada (não-deletar).
- STORY-0048-0001 (investigação) concluída antes de qualquer remoção (RULE-048-08). Produz relatório em `plans/epic-0048/reports/` com inventário canônico + scripts repro.
- ADR-0048-A (Java-only scope) mergeada antes de STORY-0048-0003.
- ADR-0048-B (CLAUDE.md contract) mergeada antes de STORY-0048-0011.
- `planningSchemaVersion: "2.0"` declarado em `execution-state.json` deste épico.
- Source-of-truth respeitado (RULE-001): nenhuma edição em `.claude/**` do repo raiz — apenas `java/src/main/resources/targets/claude/**` + regeneração via `mvn process-resources`.

### Global Definition of Done (DoD)

- **Cobertura:** ≥ 95% Line / ≥ 90% Branch globais (JaCoCo, `java/pom.xml:293-342`). Nenhuma classe nova pode nascer abaixo do threshold. Validado em `mvn verify` de cada PR.
- **Testes Automatizados:** cada história DEVE ter ≥1 teste unitário novo/modificado validando o critério de aceite principal. Histórias de bug (STORY-0048-0009, 0011) têm teste RED-first commitado antes do fix.
- **Smoke Tests:** `mvn verify` verde após cada story. Ao final, `GoldenFileTest` roda exatamente 9 perfis Java e todos passam byte-for-byte; `SmokeProfiles.profileList()` retorna exatamente 9 entradas.
- **Golden Parity:** nenhum dos 9 goldens Java sofre mutação byte-a-byte — **exceto** (a) remoção de dirs vazios em STORY-0048-0009 (Bug A fix) e (b) adição de `CLAUDE.md` na raiz em STORY-0048-0011 (Bug B fix). Mudanças registradas como linhas explícitas nos PR bodies.
- **Zero Skipped Tests:** ao final, `grep -rn "@Disabled" java/src/test` e `grep -rn "skipped profile" java/src/test` retornam 0 hits novos. Testes afetados por linguagens removidas são deletados ou reescritos, nunca desabilitados.
- **Sem Código Morto:** `grep -rn "python\|kotlin\|typescript\|rust\|golang\|\"go\"\|csharp\|dotnet" java/src/main/java` retorna hits apenas em comentários histórico-explicativos ou mensagens de erro "not supported" — nunca em caminhos executáveis. Auditado em STORY-0048-0013.
- **Documentação:** README.md raiz, CLAUDE.md raiz e CHANGELOG.md atualizados; ADR-0048-A e ADR-0048-B criadas e linkadas; `.claude/` regenerado via `mvn process-resources`.
- **Persistência:** N/A (épico não introduz state files além do `execution-state.json` do próprio orquestrador).
- **Performance:** tempo de `mvn test` reduz ≥30% (proporcional à remoção de 8/17 perfis + smoke matrices). Baseline medido em STORY-0048-0001; delta reportado em `plans/epic-0048/reports/epic-execution-report.md` ao final.
- **TDD Compliance:** commits mostram pattern test-first (teste precede implementação no git log); refactoring explícito após green; testes são incrementais (simples → complexo via Transformation Priority Premise). Histórias de bug seguem RED-GREEN-REFACTOR documentado nos PRs.
- **Double-Loop TDD:** acceptance tests derivados de cenários Gherkin (outer loop — e.g., `Epic0048EndToEndTest`, `CliLanguageValidationTest`); unit tests guiados por TPP (inner loop — e.g., `CopyHelpersPruneEmptyDirsTest`).
- **Backward Compatibility:** `--language java` (explícito) e omissão de `--language` continuam funcionando. Valores `python|go|kotlin|typescript|rust|csharp` produzem erro claro (RULE-048-06), não NPE, não silent fallback.
- **Commits Atômicos:** 1 commit por task (Rule 18), Conventional Commits com escopo `task(TASK-0048-YYYY-NNN)`.
- **Rollback Plan:** documentado na spec (Seção 9); feature flags `--legacy-empty-dirs` e `--no-claude-md` disponíveis em v4.0.0 para mitigação.
