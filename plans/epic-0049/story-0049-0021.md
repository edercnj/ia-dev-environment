## 3. Descrição

Como **plataforma**, eu quero adicionar versionamento Git automático às 3 skills de criação/decomposição de épico (`x-epic-create`, `x-epic-decompose`, `x-epic-map`), aplicando o padrão uniforme: detect-context → ensure epic branch → fases originais → planning-commit. Working tree fica limpo após qualquer execução.

### 3.1 Padrão aplicado a cada skill

```
Step P1: Skill x-git-worktree detect-context
Step P2: Skill x-internal-epic-branch-ensure --epic-id {ID}
Step P3: ... fases originais (criação dos artifacts) ...
Step P4: Skill x-planning-commit --scope {chore|docs} --epic-id {ID} \
         --paths {paths} --subject "{subject}"
Step P5: Skill x-git-push --branch epic/{ID} (opcional)
```

### 3.2 Por skill

- **`x-epic-create`**: P3 escreve `epic-XXXX.md`. P4 commit com scope `chore(epic-XXXX): init epic specification`
- **`x-epic-decompose`**: orquestra `x-epic-create` + `x-story-create` + `x-epic-map`. Cada sub-skill commita individualmente OU `x-epic-decompose` faz commit consolidado P4 com `chore(epic-XXXX): full decomposition (epic + N stories + map)`
- **`x-epic-map`**: P3 escreve `IMPLEMENTATION-MAP.md`. P4 commit com `docs(epic-XXXX): update implementation map`

### 3.3 Skipping

- `--dry-run`: P4 e P5 viram no-op (logado)
- Diff vazio em P4: silent no-op (`x-planning-commit` retorna `commitSha=null`)
- Já dentro de worktree do épico: P2 detecta e retorna sem criar

## 3.5 Entrega de Valor

- **Valor Principal:** Audit trail completo do nascimento de um épico (artefatos commitados na branch `epic/XXXX`); working tree limpo após `x-epic-decompose`.
- **Métrica de Sucesso:** Após esta story, executar `x-epic-decompose 9999 --spec foo.md` deixa working tree limpo (zero arquivos untracked).
