## 3. Descrição

Como **plataforma**, eu quero refatorar `x-story-implement` como UseCase puro (orquestrador de tasks), simétrico ao `x-epic-implement`, aceitando `--target-branch`, `--auto-merge`, `--epic-id` e propagando para `x-task-implement`/`x-pr-create`. Reduz ~50% das linhas SKILL.md.

### 3.1 Novo esqueleto (~320 linhas)

| Fase | Sub-skills invocadas | Linhas SKILL.md |
| :--- | :--- | :--- |
| 0 — Args, Context & Resume | `x-internal-args-normalize`, `x-internal-story-load-context`, `x-internal-story-resume` (se --resume) | ~80 |
| 1 — Plan | `x-internal-story-build-plan` | ~50 |
| 2 — Task Execution Loop | invoca `x-task-implement` por task; ao fim invoca `x-pr-create` | ~120 |
| 3 — Verify, Report & Cleanup | `x-internal-story-verify`, `x-internal-story-report`, `x-git-worktree remove` | ~70 |

### 3.2 Novos flags

| Flag | Default | Propagação |
| :--- | :--- | :--- |
| `--target-branch <branch>` | `develop` (legacy) | → `x-pr-create --target-branch` e `x-task-implement` |
| `--auto-merge <strategy>` | `none` (legacy) | → `x-pr-create --auto-merge` |
| `--epic-id <ID>` | — | → labels e logs |

### 3.3 Backward compat

Sem flags novos, comportamento idêntico ao atual (target=develop, sem auto-merge).

## 3.5 Entrega de Valor

- **Valor Principal:** Aplica princípio thin orchestrator simétrico ao `x-epic-implement`; reduz ~50% das linhas SKILL.md; aceita `--target-branch`, `--auto-merge`, `--epic-id`.
- **Métrica de Sucesso:** (a) `x-story-implement` SKILL.md ≤ 350 linhas; (b) zero git/gh/mvn direto; (c) propagação OO funcional (story PR alvo configurável).
