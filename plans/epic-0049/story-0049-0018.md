## 3. Descrição

Como **plataforma**, eu quero refatorar `x-epic-implement` agressivamente para ser um thin orchestrator (~460 linhas SKILL.md, queda de 77% das atuais 1.997 linhas), mudando defaults para sequencial + auto-merge em `epic/XXXX`, delegando todas as responsabilidades inline para as 6 sub-skills criadas em S5-S10 e a estensão de `x-pr-create` (S16).

**Esta é a story crítica do épico.** Todo o resto serve a ela.

### 3.1 Mudança de defaults

| Aspecto | ANTES (EPIC-0042) | DEPOIS (EPIC-0049) |
| :--- | :--- | :--- |
| Paralelismo | default `true` (via `--sequential` para sequencial) | default `false` (via `--parallel` para paralelo) |
| Auto-merge target | `develop` | `epic/XXXX` |
| Auto-merge strategy | `merge` (sem flag) | `merge` (configurável via `--auto-merge-strategy`) |
| PR final | N/A (cada story merge direto) | `epic/XXXX → develop` (manual gate) |

- Flag explícito `--legacy-flow` força legacy (Phase 2 e Phase 5 viram no-op; stories mergem direto em develop)

- **Valor Principal:** Reduz `x-epic-implement` de 1.997 → ~460 linhas SKILL.md (queda 77%); muda defaults para sequencial + auto-merge em `epic/XXXX`; habilita reuso por outros orquestradores.
