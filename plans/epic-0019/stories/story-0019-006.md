## 3. Descricao

Como **developer**, eu quero que o changelog seja gerado apos os reviews e fixes, para que os fix commits da Phase 5 sejam capturados no CHANGELOG.md.

### Contexto

No x-dev-lifecycle, Phase 3 (Documentation) gera o changelog ANTES de Phase 4 (Specialist Review) e Phase 7 (Tech Lead Review). Se reviews encontram issues e fixes sao aplicados em Phase 5, esses commits nao aparecem no changelog.

### 3.1 Split Phase 3

Dividir Phase 3 em duas partes:
- **Phase 3A** (antes dos reviews): Gerar docs estruturais (API docs, architecture docs, event catalogs, CLI reference)
- **Phase 3B** (apos Phase 7, antes de Phase 8): Gerar/regenerar o changelog entry

### 3.2 Atualizar Diagrama de Fluxo

Atualizar o diagrama de fluxo (linhas ~30-39) para refletir o split:

```
Phase 3A: Documentation (structural)   (orchestrator — inline)
Phase 4:  Review                        (invoke /x-review)
Phase 5-6: Fixes + PR                  (orchestrator — inline)
Phase 7:  Tech Lead Review             (invoke /x-review-pr)
Phase 3B: Documentation (changelog)    (orchestrator — inline)
Phase 8:  Verification                 (orchestrator — inline)
```

## 3.5 Entrega de Valor

- **Valor Principal:** CHANGELOG.md completo e preciso incluindo todos os fix commits
- **Metrica de Sucesso:** 100% dos commits (incluindo fixes de review) capturados no changelog
- **Impacto no Negocio:** Changelog confiavel para release notes e rastreabilidade
