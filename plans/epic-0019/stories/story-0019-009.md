## 3. Descricao

Como **architect**, eu quero que o epic orchestrator verifique consistencia entre stories apos cada fase, para que interfaces compartilhadas, naming e implementacoes nao divirjam.

### Contexto

Cada story e implementada em isolamento (RULE-001 clean context). Quando stories paralelas tocam interfaces compartilhadas (ex: mesmo port, mesmo DTO), podem criar definicoes conflitantes que so sao detectadas em runtime.

### 3.1 Cross-Story Consistency Gate

Adicionar gate apos o Integrity Gate, executado por fase. Subagent verifica:

1. **Interface Consistency:** Interfaces/ports declarados por cada story — conflitos de assinatura
2. **Naming Convention Consistency:** Classes, metodos, packages — padroes uniformes
3. **Duplicate Detection:** Classes/metodos com funcionalidade similar entre stories
4. **Shared Dependency Verification:** Componentes usados por multiplas stories — compatibilidade

### 3.2 Gate Behavior

- PASS: Continuar para proxima fase
- WARN: Log warnings, continuar
- FAIL: Pausar execucao, reportar ao usuario, requerer --resume apos fix manual

## 3.5 Entrega de Valor

- **Valor Principal:** Prevencao de inconsistencias entre stories paralelas detectadas antes do merge
- **Metrica de Sucesso:** Zero conflitos de interface nao detectados entre stories do mesmo epic
- **Impacto no Negocio:** Reducao de bugs de integracao em entregas multi-story

## 7. Criterios de Aceite (Gherkin)

```gherkin
@GK-1
Cenario: Stories sem conflitos de interface
  DADO 2 stories completadas na mesma fase que tocam interfaces diferentes
  QUANDO o Consistency Gate executa
  ENTAO status == PASS
  E nenhum finding reportado

@GK-2
Cenario: Stories com conflito de assinatura de interface
  DADO story-A define UserPort.findById(UUID) e story-B define UserPort.findById(Long)
  QUANDO o Consistency Gate executa
  ENTAO status == FAIL
  E findings contem "Interface conflict: UserPort.findById has incompatible signatures"

@GK-3
Cenario: Stories com naming inconsistente (WARNING)
  DADO story-A cria UserService e story-B cria user_handler (snake_case vs CamelCase)
  QUANDO o Consistency Gate executa
  ENTAO status == WARN
  E findings contem "Naming inconsistency: mixed conventions"

@GK-4
Cenario: Stories com implementacao duplicada
  DADO story-A cria DateUtils.formatISO() e story-B cria TimeHelper.toISOString() com mesma logica
  QUANDO o Consistency Gate executa

  E findings contem "Potential duplicate: DateUtils.formatISO and TimeHelper.toISOString"

@GK-5

  QUANDO o orchestrator processa o resultado
```
