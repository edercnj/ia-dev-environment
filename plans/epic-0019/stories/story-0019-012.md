## 3. Descricao

Como **developer**, eu quero que os placeholders remanescentes no x-dev-epic-implement sejam substituidos por implementacoes ou referencias, para que o skill file seja claro e sem ambiguidades.

### Contexto

O SKILL.md do x-dev-epic-implement contem 5+ placeholders que referenciam stories futuras. Alguns foram implementados mais adiante no arquivo mas os placeholders permanecem, criando confusao.

### 3.2 Progress Reporting

Apos cada story completar, emitir:

```
>>> Epic {epicId} Progress: Phase {phase} — {completed}/{total} stories
    ({percentage}% complete). Last: {storyId} = {status}. Elapsed: {elapsed}
```

### 3.3 Partial Execution Filter

Quando `--phase N` ativo: filtrar stories para apenas phase N.
Quando `--story ID` ativo: filtrar para apenas aquela story.
Filtros aplicados ANTES de dependency checking em `getExecutableStories()`.
