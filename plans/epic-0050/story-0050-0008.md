## 3. Descrição

Como **engenheiro de plataforma**, eu quero substituir o valor `Recommended Model: Adaptive` (presente em 8 dos 10 agents) e `Recommended Model: Opus` (do `product-owner`) por valores determinísticos, para eliminar a ambiguidade semântica e garantir que invocações `Agent(subagent_type: "<agent-name>")` resolvam o modelo correto.

Hoje, "Adaptive" é um placeholder sem contrato — na prática resolve para Opus (default do general-purpose). O `product-owner` declara Opus mas sua tarefa (validação de DoR, decomposição de spec) é Sonnet-adequada. Esta story resolve ambos gaps em 10 arquivos de agent.

### 3.1 Matriz de mudança

| Agent | Atual | Novo | Rationale |
| :--- | :--- | :--- | :--- |
| `architect` | **Opus** | **Opus** (mantém) | Design arquitetural profundo |
| `product-owner` | **Opus** | **Sonnet** (mudança) | Validação de DoR é Sonnet-adequada |
| `qa-engineer` | Adaptive | **Sonnet** | Checklist de qualidade |
| `security-engineer` | Adaptive | **Sonnet** | Análise STRIDE |
| `tech-lead` | Adaptive | **Sonnet** | Quality gates |
| `sre-engineer` | Adaptive | **Sonnet** | Ops review |
| `performance-engineer` | Adaptive | **Sonnet** | Perf review |
| `devops-engineer` | Adaptive | **Sonnet** | CI/CD review |
| `devsecops-engineer` | Adaptive | **Sonnet** | Pipeline security review |
| `java-developer` | Adaptive | **Sonnet** | Code implementation |

### 3.2 Mudança exata

Para cada arquivo em `java/src/main/resources/targets/claude/agents/*.md`:

1. Localizar seção `## Recommended Model` (geralmente linhas 10-20).
2. Substituir valor atual pelo novo conforme matriz.
3. Atualizar a 1-2 linhas de rationale para refletir o tier escolhido.
4. Garantir que o cabeçalho/metadata YAML (se presente) também reflete o novo valor.

### 3.3 Integração com Rule 23

A STORY-0050-0001 (Rule 23) define no "Agent metadata contract" a sintaxe determinística. Esta story aplica o contrato aos 10 agents existentes. O audit Check D (STORY-0050-0009) valida que não há mais "Adaptive".

## 3.5 Entrega de Valor

- **Valor Principal:** Elimina ambiguidade em 10 agents; 8 agents migram de "Adaptive" (placeholder) para "Sonnet" (determinístico); `product-owner` migra de Opus para Sonnet.
- **Métrica de Sucesso:** `grep -l "Adaptive" .claude/agents/` retorna 0 arquivos. `grep -l "Recommended Model: Sonnet" .claude/agents/` retorna 9 arquivos. Audit Check D passa.
- **Impacto no Negócio:** Quando Claude Code invoca um agent definido, agora sabe com certeza qual modelo usar — antes, caía em Opus silenciosamente.
