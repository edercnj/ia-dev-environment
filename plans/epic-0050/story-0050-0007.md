## 3. Descrição

Como **engenheiro de plataforma**, eu quero propagar o parâmetro `model:` nas ~30 chamadas `Skill(...)` dos orquestradores (`x-epic-implement`, `x-story-implement`, `x-review`, `x-task-implement`), para garantir que cada sub-skill rode no tier apropriado da matriz (especialmente `x-git-commit` em Haiku, não herdando Sonnet do orquestrador pai).

Essa é a terceira camada de enforcement da Rule 23 (complementar a frontmatter e Agent param). Sem ela, mesmo após STORY-0050-0002 e STORY-0050-0004, um `x-story-implement` (Sonnet) invocando `x-git-commit` (Haiku-eligible) poderia cascatear Sonnet para dentro do git commit — desperdício.

### 3.1 Call-sites a modificar

**`x-epic-implement/SKILL.md`:**
| Call-site | Model target |
| :--- | :--- |
| `Skill("x-story-implement", ...)` | `"sonnet"` (explícito mesmo herdando) |

**`x-story-implement/SKILL.md`:**
| Call-site | Model target |
| :--- | :--- |
| `Skill("x-test-tdd", ...)` | `"sonnet"` |
| `Skill("x-git-commit", ...)` | `"haiku"` |
| `Skill("x-task-implement", ...)` | `"sonnet"` |
| `Skill("x-git-worktree", ...)` (se existe) | `"haiku"` |

**`x-review/SKILL.md`** (~10 call-sites):
| Call-site | Model target |
| :--- | :--- |
| `Skill("x-review-qa", ...)` | `"sonnet"` |
| `Skill("x-review-perf", ...)` | `"sonnet"` |
| `Skill("x-review-devops", ...)` | `"sonnet"` |
| `Skill("x-review-pr", ...)` | `"sonnet"` |
| ...demais | `"sonnet"` |

**`x-task-implement/SKILL.md`:**
| Call-site | Model target |
| :--- | :--- |
| `Skill("x-git-commit", ...)` | `"haiku"` |
| `Skill("x-test-tdd", ...)` | `"sonnet"` |

### 3.2 Mudança exata

Adicionar `model: "<tier>"` como campo entre `skill:` e `args:` em cada `Skill(...)` call. Exemplo:

**Antes:**
```
Skill(skill: "x-git-commit", args: "-m 'feat: ...'")
```

**Depois:**
```
Skill(skill: "x-git-commit", model: "haiku", args: "-m 'feat: ...'")
```

### 3.3 Contagem total esperada

~30 call-sites modificados no total entre os 4 orquestradores. Número exato a ser refinado durante a implementação via `grep -c 'Skill(skill:' x-*/SKILL.md`.

## 3.5 Entrega de Valor

- **Valor Principal:** Captura ~10% da redução de tokens; fecha a terceira camada de enforcement (frontmatter ✓, Agent ✓, Skill ✓).
- **Métrica de Sucesso:** `grep -n 'Skill(skill: "x-' .claude/skills/x-{epic,story}-implement/SKILL.md .claude/skills/x-review/SKILL.md .claude/skills/x-task-implement/SKILL.md | grep -v 'model:'` retorna 0 linhas.
- **Impacto no Negócio:** Propaga o tier corretamente ao longo da cadeia inteira; cada story executada posteriormente consome ~740 tokens a menos graças a `x-git-commit` em Haiku.
