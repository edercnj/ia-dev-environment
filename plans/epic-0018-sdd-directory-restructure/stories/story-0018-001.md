## 3. DESCRIPTION

### State Before

O repositório possui apenas `docs/` como diretório de documentação, com subdiretórios
que misturam concerns (specs, stories, audits, adr, architecture, guides, runbook).
Não existem diretórios raiz dedicados para o fluxo SDD.

```
ia-dev-environment/
├── docs/
│   ├── adr/
│   ├── architecture/
│   ├── audits/
│   ├── guides/
│   ├── runbook/
│   ├── specs/
│   └── stories/
└── (demais diretórios do projeto)
```

### State After

Novos diretórios raiz existem, prontos para receber conteúdo migrado nas stories
subsequentes. Cada diretório contém um `.gitkeep` para garantir presença no git.

```
ia-dev-environment/
├── steering/                  # Contexto persistente do projeto
├── specs/                     # Especificações de input humano
│   └── _templates/            # Templates de especificação
├── plans/                     # Artefatos SDD gerados (epics, stories, maps)
│   └── epic-0018-sdd-directory-restructure/  # Este épico (já existe)
├── results/                   # Outputs de execução
│   ├── audits/                # Auditorias de codebase e dependências
│   ├── reviews/               # Reviews de código
│   ├── runbooks/              # Runbooks e guias operacionais
│   └── test-runs/             # Relatórios de execução de testes
├── contracts/                 # Contratos de API e schemas
│   ├── api/                   # Definições de API (OpenAPI, gRPC)
│   ├── schemas/               # JSON Schemas, Avro, Protobuf
│   └── events/                # Event schemas
├── adr/                       # Architecture Decision Records
├── docs/                      # (ainda existe — será removido em STORY-007)
└── (demais diretórios do projeto)
```

### O que esta story NAO faz

- Nao move nenhum arquivo existente (migração é das stories 002-005)
- Nao cria arquivos de conteúdo (steering files são da STORY-002)
- Nao modifica código Java, templates ou skills
- Nao edita README.md, CLAUDE.md ou qualquer documentação existente

## 7. CORRECT vs INCORRECT

### CORRETO

```bash
# Criar diretórios com mkdir -p e adicionar .gitkeep
mkdir -p steering && touch steering/.gitkeep
mkdir -p specs/_templates && touch specs/_templates/.gitkeep
mkdir -p results/{audits,reviews,runbooks,test-runs}
touch results/audits/.gitkeep results/reviews/.gitkeep results/runbooks/.gitkeep results/test-runs/.gitkeep
mkdir -p contracts/{api,schemas,events}
touch contracts/api/.gitkeep contracts/schemas/.gitkeep contracts/events/.gitkeep
mkdir -p adr && touch adr/.gitkeep
```

### INCORRETO

```bash
# NÃO mover ou tocar em docs/ nesta story
git mv docs/adr adr          # ERRADO — migração é STORY-005
rm -rf docs/                  # ERRADO — docs/ ainda tem conteúdo

# NÃO criar arquivos de conteúdo
echo "# Product" > steering/product.md   # ERRADO — conteúdo é STORY-002

# NÃO esquecer .gitkeep em diretórios vazios
mkdir -p contracts/api        # ERRADO sem .gitkeep — git ignora dirs vazios
```
