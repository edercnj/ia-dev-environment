## 7. CORRECT vs INCORRECT

### CORRETO

```bash
# Migrar com git mv para preservar histórico
git mv docs/architecture/service-architecture.md steering/service-architecture.md

# Criar steering files derivando de fontes existentes
# product.md: extrair seções "Identity" e "Purpose" de rules/01-project-identity.md
# tech-stack.md: extrair tabela "Technology Stack" de rules/01-project-identity.md
# Citar a fonte no header do arquivo: "Derived from: .claude/rules/01-project-identity.md"
```

### INCORRETO

```bash
# NÃO copiar ao invés de mover — perde rastreabilidade
cp docs/architecture/service-architecture.md steering/   # ERRADO — usar git mv

# NÃO inventar requisitos que não existem nas fontes
echo "## Microservices Architecture" > steering/architecture-principles.md  # ERRADO — projeto é library, não microservices

# NÃO duplicar conteúdo das rules — referenciar e resumir
cat .claude/rules/04-architecture-summary.md > steering/architecture-principles.md  # ERRADO — copiar verbatim é duplicação
```
