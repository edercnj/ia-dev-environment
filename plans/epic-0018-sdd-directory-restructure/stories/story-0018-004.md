## 7. CORRECT vs INCORRECT

### CORRETO

```bash
# Migrar tracked dirs com git mv para preservar histórico
for dir in codex-support epic-0003 epic-0004 epic-0005 epic-0006 epic-0007 \
           epic-0008 epic-0009 epic-0010 epic-0011 epic-0012 epic-0013 \
           epic-0014 github-structure; do
  git mv "docs/stories/$dir" "plans/$dir"
done

# Migrar untracked dirs com mv + git add
for dir in epic-0015 epic-0016 epic-0017; do
  mv "docs/stories/$dir" "plans/$dir"
done

# Verificar contagem
BEFORE=514
AFTER=$(find plans/ -type f -not -path "*/epic-0018-*" | wc -l)
[ "$AFTER" -ge "$BEFORE" ] && echo "OK: $AFTER >= $BEFORE"
```

### INCORRETO

```bash
# NÃO mover com cp — perde histórico e duplica dados
cp -r docs/stories/* plans/   # ERRADO

# NÃO ignorar untracked dirs — eles TÊM conteúdo
git mv docs/stories/epic-0015 plans/   # ERRADO — falha porque não está tracked

# NÃO renomear dirs durante a migração
git mv docs/stories/codex-support plans/epic-0002-codex-support  # ERRADO — manter nome original

# NÃO mover plans/ de epic-0018 para dentro de si mesmo
git mv docs/stories/* plans/   # PERIGOSO — verificar se não pega epic-0018
```
