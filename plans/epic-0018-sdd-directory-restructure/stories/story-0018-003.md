## 7. CORRECT vs INCORRECT

### CORRETO

```bash
# Usar git mv para preservar histórico
git mv docs/specs/SPEC-architectural-improvements-v1.md specs/
git mv docs/guides/worktree-parallelism-strategy.md results/runbooks/
git mv docs/runbook/deploy-runbook.md results/runbooks/

# Verificar migração
find specs/ results/runbooks/ -type f | wc -l  # Deve retornar 3
```

### INCORRETO

```bash
# NÃO usar cp + rm — perde histórico git
cp docs/specs/SPEC-architectural-improvements-v1.md specs/
rm docs/specs/SPEC-architectural-improvements-v1.md   # ERRADO — use git mv

# NÃO renomear arquivos durante migração — apenas mover
git mv docs/runbook/deploy-runbook.md results/runbooks/deployment-guide.md  # ERRADO — manter nome original

# NÃO remover docs/ inteiro — outras stories ainda precisam dele
rm -rf docs/   # ERRADO — docs/stories/, docs/adr/, docs/audits/ ainda existem
```
