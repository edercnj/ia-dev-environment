## 3. Descrição

Retrofit `x-pr-merge-train` (sequential PR merging orchestrator) para emitir `TaskCreate` por PR com chaining via `addBlocks`.

### 3.1 PR Loop Sequencial

```
Para cada PR in merge-set:
  TaskCreate(subject: "pr-train › PR-K — {title}")
  TaskUpdate(..., addBlocks: [previous-PR])
  Merge PR K
  TaskUpdate(subject: "pr-train › PR-K", status: completed)
```

Resultado: lista mostra "pr-train › PR-1 — feature X (completed)" → "PR-2 — feature Y (in_progress)" → "PR-3 — fix Z (pending)".

## 3.2 Merge Status Tracking

Subject pode incluir merge status:
```
"pr-train › PR-K › Waiting for CI"
"pr-train › PR-K › Checking conflicts"
"pr-train › PR-K › Merged"
```

## 3.5 Entrega de Valor

Operador consegue ver status de merge train sequencial (qual PR está em qual etapa).
