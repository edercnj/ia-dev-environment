## 3. Descrição

Como **implementador de orquestradores**, eu quero retrofit `x-task-implement` para **emitir `TaskCreate` por step e por TDD cycle**, garantindo que:

- Operador vê cada step (0–5) como task separada
- Operador vê cada ciclo de TDD (Red, Green, Refactor) como sub-task
- Phase gates PRE/POST bloqueiem transições
- Subject segue padrão hierárquico: `"TASK-0060-0001-003 › Step 2 › Cycle 1 › Red"`

### 3.1 Pattern por Step

```markdown
## Step N — <Description>

### N.0 — Gate PRE
Skill(skill: "x-internal-phase-gate", args: "--mode pre --step N")

### N.1 — TaskCreate
TaskCreate(subject: "TASK-XXXX › Step N — <desc>", ...)
TaskUpdate(taskId: stepTaskId, status: "in_progress")

### N.2 — Executar
... sub-skills ...

### N.3 — Gate POST
Skill(skill: "x-internal-phase-gate", args: "--mode post --step N")

### N.4 — TaskUpdate
TaskUpdate(taskId: stepTaskId, status: "completed")
```

### 3.2 Pattern por TDD Cycle

Step 2 (TDD cycles) itera sobre N test cases:

```markdown
## Step 2 — TDD Red-Green-Refactor Cycles

**Batch A — TaskCreate para cada cycle:**

  TaskCreate(subject: "TASK-XXXX › Step 2 › Cycle K › Red — UT-K")
  TaskCreate(subject: "TASK-XXXX › Step 2 › Cycle K › Green — UT-K")
  TaskCreate(subject: "TASK-XXXX › Step 2 › Cycle K › Refactor — UT-K")

**Para cada cycle K:**

  TaskUpdate(taskId: tddCycle[K].red, status: "in_progress")
  ... Red phase ...
  TaskUpdate(taskId: tddCycle[K].red, status: "completed")

  TaskUpdate(taskId: tddCycle[K].green, status: "in_progress")
  ... Green phase ...
  TaskUpdate(taskId: tddCycle[K].green, status: "completed")

  TaskUpdate(taskId: tddCycle[K].refactor, status: "in_progress")
  ... Refactor phase ...
  TaskUpdate(taskId: tddCycle[K].refactor, status: "completed")

**Gate POST:**

Skill(skill: "x-internal-phase-gate", args: "--mode wave --expected-tasks [red-ids] [green-ids] [refactor-ids]")
```

## 3.5 Entrega de Valor

- **Valor Principal:** Operador consegue ver exatamente qual ciclo de TDD de qual test case travou (ex: "Cycle 3 › Red — UT-3" está in_progress).
- **Métrica de Sucesso:** Lista de tasks mostra "TASK-0060-0001-003 › Step 2 › Cycle 5 › Green › in progress" quando implementando.
- **Impacto no Negócio:** Reduz debugging time de test failures de ~20min para < 2min.
