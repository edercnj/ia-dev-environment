# EPIC-0019: Quality Gates e Prevencao Total a Falhas no Workflow Autonomo de Desenvolvimento

> **Status**: DRAFT
> **Epic ID**: EPIC-0019
> **Feature Slug**: workflow-quality-gates
> **Criado em**: 2026-04-04

---

## 1. BUSINESS OBJECTIVE

O workflow de desenvolvimento autonomo via IA (`/x-dev-epic-implement` -> `/x-dev-lifecycle`)
possui lacunas criticas que comprometem a qualidade do codigo entregue. A principal falha e que
o prompt do subagent no epic orchestrator lista apenas 5 passos simplificados, omitindo as 9
fases completas do x-dev-lifecycle — incluindo Specialist Review (Phase 4), Tech Lead Review
(Phase 7) e Final Verification (Phase 8). Isso significa que stories podem ser marcadas como
SUCCESS sem que nenhum review tenha sido executado.

Alem disso, o workflow carece de:
- Aceitacao formal do Product Owner (validacao de valor de negocio)
- Validacao pos-hoc de Conventional Commits
- Report consolidado de execucao com metricas completas
- Consistencia cross-story em epics com multiplas stories paralelas
- Estrategia de rollback para falhas parciais
- Enforcement do Definition of Done como gate bloqueante

O objetivo e tornar o workflow 100% autonomo com prevencao total a falhas, garantindo que
todo quality gate seja enforced, todo review aconteca, e nenhum passo possa ser silenciosamente
ignorado.

## 2. SCOPE BOUNDARIES

### In Scope

- Corrigir o prompt do subagent em `x-dev-epic-implement` para invocar `/x-dev-lifecycle` completo
- Estender o contrato SubagentResult com campos de review, coverage e TDD
- Adicionar PO Acceptance Gate ao x-dev-lifecycle
- Adicionar validacao pos-hoc de Conventional Commits no Integrity Gate
- Adicionar Cross-Story Consistency Gate entre fases no epic orchestrator
- Implementar estrategia de rollback e retry com contexto de erro
- Converter DoD de relatorio para gate bloqueante
- Corrigir timing do changelog (apos reviews, nao antes)
- Adicionar validacao de qualidade ao architecture plan
- Adicionar validacao INVEST ao story decomposition
- Estender o execution report template com novas secoes
- Limpar placeholders remanescentes no x-dev-epic-implement

### Out of Scope

- Modificar o gerador Java (assemblers, templates) — sera atualizado em epic separado
- Modificar testes existentes em `tests/`
- Criar novos skills ou agentes
- Modificar a estrutura de diretorios do projeto

### Deferred

- Atualizar o gerador Java para produzir os skills corrigidos em novos projetos
- Atualizar golden files e smoke tests
- Adicionar metricas de observabilidade ao workflow
