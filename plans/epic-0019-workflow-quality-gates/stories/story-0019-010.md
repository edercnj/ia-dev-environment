## 3. Descricao

Como **architect**, eu quero que o architecture plan gerado pelo x-dev-architecture-plan tenha validacao automatica de qualidade, para que planos com violacoes de layer boundaries nao sejam aceitos.

### Contexto

O x-dev-architecture-plan gera planos mas nao ha validacao que o plano segue as regras de hexagonal architecture: dependency direction, domain purity, port/interface completeness.

### 3.1 Self-Check no Architecture Plan

Adicionar validacao ao final do x-dev-architecture-plan/SKILL.md:

1. **Dependency Direction Check:** Para cada relacao no Component Diagram, validar que nao viola a regra de dependency direction (domain nunca importa adapter)
2. **Layer Completeness Check:** Todo novo componente deve ter layer declarada
3. **Port/Interface Check:** Todo outbound adapter deve implementar um port do domain
4. **Circular Dependency Check:** Nenhum ciclo que cruza layer boundaries

### 3.2 Integracao com x-dev-lifecycle

Em Phase 1 Step 1A, apos invocar x-dev-architecture-plan, verificar resultado da validacao. Se FAIL, ABORT.

## 3.5 Entrega de Valor

- **Valor Principal:** Planos de arquitetura que respeitam hexagonal architecture desde a fase de design
- **Metrica de Sucesso:** Zero architecture plans com violacoes de layer boundaries
- **Impacto no Negocio:** Reducao de retrabalho por violacoes de arquitetura detectadas tardiamente
