## 3. Descricao

Como **tech lead**, eu quero que o Integrity Gate valide o formato Conventional Commits de todos os commits, para que commits mal formatados sejam detectados antes de avancar para a proxima fase.

### Contexto

O x-git-push define o formato Conventional Commits e o Integrity Gate verifica [TDD] suffixes (Step 4.5), mas nao valida o formato completo `type(scope): subject`. Um commit `wip: stuff [TDD]` passaria o check de TDD mas violaria Conventional Commits.

### 3.1 Step 4.6: Conventional Commits Validation

Adicionar Step 4.6 ao Integrity Gate (apos Step 4.5 TDD Compliance, antes Step 5 Smoke Gate):

Para cada commit no git log da fase atual:
1. Validar regex: `^(feat|fix|refactor|test|docs|build|chore|infra)(\([a-z0-9-]+\))?: .{1,72}$`
2. Tipo deve ser um dos 8 permitidos
3. Scope presente (WARNING se ausente, nao bloqueante)
4. Subject <= 72 chars, sem ponto final
5. Contar violacoes: `ccViolationCount`

Status:
- PASS: ccViolationCount == 0
- WARNING: 1-2 violacoes (nao bloqueante)
- FAIL: 3+ violacoes (bloqueante)

### 3.2 Adicionar ao Gate Result

Adicionar campo `conventionalCommits` ao gate result:

```json
{
  "conventionalCommits": {
    "status": "PASS | WARNING | FAIL",
    "ccViolationCount": 0,
    "violations": []
  }
}
```

## 7. Criterios de Aceite (Gherkin)

```gherkin
@GK-1
Cenario: Todos commits validos
  DADO uma fase com 10 commits todos no formato "feat(scope): subject [TDD]"
  QUANDO Step 4.6 valida os commits
  ENTAO ccViolationCount == 0
  E status == "PASS"

@GK-2
Cenario: Commits validos com scope ausente (WARNING)
  DADO uma fase com 10 commits, 1 no formato "feat: subject" (sem scope)
  QUANDO Step 4.6 valida os commits
  ENTAO ccViolationCount == 0 (scope ausente e WARNING, nao violacao)
  E status == "PASS" com warning "1 commit(s) without scope"

@GK-3
Cenario: 2 commits invalidos (WARNING, nao bloqueante)
  DADO uma fase com 10 commits, 2 com formato invalido "wip: stuff"
  QUANDO Step 4.6 valida os commits
  ENTAO ccViolationCount == 2
  E status == "WARNING"
  E a fase NAO e bloqueada

@GK-4
Cenario: 3+ commits invalidos (FAIL, bloqueante)
  DADO uma fase com 10 commits, 4 com formato invalido
  QUANDO Step 4.6 valida os commits
  ENTAO ccViolationCount == 4
  E status == "FAIL"
  E a fase e bloqueada
  E violations lista os 4 commits invalidos

@GK-5
Cenario: Subject com mais de 72 caracteres
  DADO um commit "feat(auth): this is a very long subject line that exceeds the maximum allowed seventy two character limit for conventional commits"
  QUANDO Step 4.6 valida o commit
  ENTAO o commit e contado como violacao
  E a violacao indica "subject exceeds 72 characters"
```
