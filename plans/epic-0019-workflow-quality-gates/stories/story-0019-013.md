## 3. Descricao

Como **tech lead**, eu quero que os status de stories, epics e implementation maps sejam atualizados apos implementacao, para que os arquivos markdown reflitam fielmente o estado real do desenvolvimento.

### Contexto

Os epics 0015, 0016 e 0018 foram implementados e mergeados na main, mas seus arquivos markdown permanecem com status stale:

| Epic | Arquivo | Status Atual | Status Real |
|---|---|---|---|
| EPIC-0015 | EPIC-0015.md | `Em Refinamento` | Concluido (PR #130, commit `48706e70`) |
| EPIC-0015 | story-0015-*.md (15 files) | `Pendente` | Concluido |
| EPIC-0015 | IMPLEMENTATION-MAP.md | Sem coluna Status | Todas as 15 stories concluidas |
| EPIC-0016 | EPIC-0016.md | `Pendente` | Parcialmente Concluido (11/15, PR #132) |
| EPIC-0016 | story-0016-*.md (11 completed) | `Pendente` | 11 Concluidas, 4 Pendentes |
| EPIC-0018 | EPIC-0018.md | `DRAFT` | Concluido (PR #131, commit `de53903b`) |
| EPIC-0018 | story-0018-*.md (7 files) | `DRAFT` | Concluido |
| EPIC-0018 | IMPLEMENTATION-MAP.md | Sem Status | Todas as 7 stories concluidas |

### 3.1 Parte Retroativa: Corrigir Status nos Epics Existentes

Para cada epic (0015, 0016, 0018):

1. **Atualizar EPIC-XXXX.md:**
   - Alterar o campo `Status` para `Concluido` (ou `Parcialmente Concluido` para 0016)
   - Adicionar referencia ao PR e commit SHA de merge

2. **Atualizar story-XXXX-YYYY.md:**
   - Alterar o campo `**Status:**` de `Pendente`/`DRAFT` para `Concluida`
   - Adicionar commit SHA da implementacao (extrair de execution-state.json quando disponivel, ou de git log)
   - Para stories nao concluidas do EPIC-0016 (0011, 0012, 0014, 0015): manter `Pendente`

3. **Atualizar IMPLEMENTATION-MAP.md:**
   - Adicionar ou atualizar coluna `Status` na dependency matrix
   - Preencher `Concluida` para stories implementadas, `Pendente` para as restantes

### 3.2 Parte Preventiva: Garantir Sincronizacao de Status no Workflow

Verificar e reforcar no `x-dev-epic-implement/SKILL.md` e `x-dev-lifecycle/SKILL.md`:

1. **Section 1.6b (Markdown Status Sync) do x-dev-epic-implement:** Ja existe e define a logica de sincronizacao. Verificar se esta sendo efetivamente executada pelo orchestrator apos cada story. Adicionar log explicito: `"STATUS SYNC: Updated {file} status to {newStatus}"` para rastreabilidade.

2. **Phase 8 do x-dev-lifecycle:** Ja tem items 2 (update IMPLEMENTATION-MAP) e 3 (update Story File). Adicionar verificacao pos-update: apos escrever o arquivo, re-ler e confirmar que o status foi alterado. Se nao, emitir WARNING.

3. **Adicionar ao DoD (Phase 8):**
   - `- [ ] Story markdown file Status field updated to Concluida`
   - `- [ ] IMPLEMENTATION-MAP Status column updated for this story`
   - `- [ ] Epic Status field updated if all stories complete`

Estas verificacoes ja estao parcialmente definidas nos items 6.5-6.6 do DoD existente, mas a execucao nao esta sendo enforced. O reforco e adicionar logging e re-leitura de confirmacao.

## 3.5 Entrega de Valor
