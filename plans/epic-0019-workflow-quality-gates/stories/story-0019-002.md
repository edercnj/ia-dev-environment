## 3. Descricao

Como **tech lead**, eu quero que o contrato SubagentResult inclua campos obrigatorios de review, coverage e TDD, para que o orchestrator possa validar que todas as fases foram executadas.

### Contexto

O SubagentResult atual (Section 1.5) tem apenas 4 campos: `status`, `commitSha`, `findingsCount`, `summary`. Nao ha como verificar se reviews foram executados, qual foi o score, ou qual a cobertura obtida.

### 3.1 Estender SubagentResult

Adicionar campos obrigatorios ao contrato (Section 1.5, linhas ~599-612):

```json
{
  "status": "SUCCESS | FAILED | PARTIAL",
  "commitSha": "<sha>",
  "findingsCount": 0,
  "summary": "<descricao>",
  "reviewsExecuted": {
    "specialist": true,
    "techLead": true
  },
  "reviewScores": {
    "specialist": "56/56",
    "techLead": "45/45"
  },
  "coverageLine": 97.5,
  "coverageBranch": 93.2,
  "tddCycles": 12
}
```

### 3.2 Validacao Enforced na Section 1.5

Na validacao do SubagentResult (RULE-008), adicionar:
- Se `status == SUCCESS` e `--skip-review` NAO esta ativo:
  - `reviewsExecuted.specialist` DEVE ser `true`, senao marca FAILED
  - `reviewsExecuted.techLead` DEVE ser `true`, senao marca FAILED
- Se `coverageLine < 95` ou `coverageBranch < 90`: marca FAILED
- Se `tddCycles == 0`: marca FAILED
