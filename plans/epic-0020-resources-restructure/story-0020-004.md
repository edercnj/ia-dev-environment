## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que os templates Codex estejam sob `targets/codex/templates/`, para que a estrutura de targets seja uniforme entre os tres assistentes de IA.

### Contexto

O Codex possui apenas uma pasta `codex-templates/` contendo templates Nunjucks/Pebble (`.njk`) para gerar `AGENTS.md`, `config.toml`, `requirements.toml` e `agents-override.md`. Os assemblers Codex usam paths de classpath como `"codex-templates/config.toml.njk"` para renderizar templates via template engine.

### 3.1 Movimentacao de diretorios

| Origem | Destino |
| :--- | :--- |
| `codex-templates/` | `targets/codex/templates/` |

### 3.2 Assemblers a atualizar

- `CodexConfigAssembler` — template path `"codex-templates/config.toml.njk"` → `"targets/codex/templates/config.toml.njk"`
- `CodexAgentsMdAssembler` — template path `"codex-templates/agents-md.md.njk"` → `"targets/codex/templates/agents-md.md.njk"`
- `CodexOverrideAssembler` — template path `"codex-templates/agents-override.md.njk"` → `"targets/codex/templates/agents-override.md.njk"`
- `CodexRequirementsAssembler` — template path `"codex-templates/requirements.toml.njk"` → `"targets/codex/templates/requirements.toml.njk"`

### 3.3 Template engine classpath

Os templates `.njk` sao carregados via classpath pela template engine (Pebble). A mudanca de path e um search-and-replace nas constantes dos assemblers. O Pebble resolve templates relativamente ao classpath root, entao o path completo `"targets/codex/templates/..."` funciona diretamente.

Templates `.njk` dentro de `codex-templates/sections/` que usam `{% include %}` com paths relativos precisam ser verificados — se usam paths absolutos (classpath-relative), devem ser atualizados.

## 3.5 Entrega de Valor

- **Valor Principal:** Targets Codex sob namespace consistente com Claude e GitHub Copilot
- **Metrica de Sucesso:** Output `AGENTS.md`, `config.toml`, `requirements.toml` identicos ao pre-migracao
- **Impacto no Negocio:** Estrutura uniforme entre targets, facilitando manutencao e extensao
