## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que o codigo legado do `ResourceResolver` (metodos deprecated, constantes de path obsoletas) seja removido e a documentacao atualizada, para que o codebase reflita a nova estrutura sem dead code.

### Contexto

Apos as stories 001-007, o metodo legado `resolveResourcesRoot(probe, depth)` nao e mais necessario — todos os assemblers usam `resolveResourceDir`. Constantes como `AGENTS_TEMPLATES_DIR`, `SKILLS_TEMPLATES_DIR`, etc. que apontavam para paths antigos tambem sao obsoletas. A documentacao (CLAUDE.md, README.md) ainda referencia a estrutura antiga.

### 3.1 Remocoes de codigo

- Remover ou deprecar `resolveResourcesRoot(String probe, int depth)` se nao ha callers
- Remover constantes de path obsoletas dos assemblers (ex: `AGENTS_TEMPLATES_DIR = "agents-templates"`)
- Remover imports nao utilizados resultantes das remocoes

### 3.2 Atualizacoes de documentacao

- Atualizar `CLAUDE.md` (raiz) com nova estrutura de recursos
- Atualizar `README.md` do projeto Java com nova estrutura
- Atualizar `resource-config.json` (native-image) se paths antigos estiverem listados
- Verificar e atualizar `docs/` com qualquer referencia a paths antigos

## 3.5 Entrega de Valor

- **Valor Principal:** Codebase limpo sem dead code, documentacao refletindo a realidade
- **Metrica de Sucesso:** Zero referencias a paths antigos no codigo e documentacao
- **Impacto no Negocio:** Onboarding simplificado — documentacao e codigo contam a mesma historia
