## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que toda a base de conhecimento compartilhada (languages, frameworks, databases, patterns, protocols, security, infrastructure, cloud-providers) esteja sob `knowledge/`, para que a organizacao reflita as secoes do YAML de configuracao e seja facil localizar qualquer recurso de conhecimento.

### Contexto

Atualmente, 9 pastas de conhecimento target-agnostic estao misturadas no top-level com pastas target-specific. Elas sao referenciadas por multiplos assemblers (RulesAssembler, PatternsAssembler, ProtocolsAssembler, etc.) e contem markdown de referencia sobre linguagens, frameworks, bancos de dados, padroes arquiteturais e protocolos.

### 3.1 Movimentacao de diretorios

| Origem | Destino | Secao YAML |
| :--- | :--- | :--- |
| `core/` | `knowledge/core/` | (core principles) |
| `languages/` | `knowledge/languages/` | `language` |
| `frameworks/` | `knowledge/frameworks/` | `framework` |
| `databases/` | `knowledge/databases/` | `data` |
| `patterns/` | `knowledge/patterns/` | `architecture` |
| `protocols/` | `knowledge/protocols/` | `interfaces` |
| `security/` | `knowledge/security/` | `security` |
| `infrastructure/` | `knowledge/infrastructure/` | `infrastructure` |
| `cloud-providers/` | `knowledge/cloud-providers/` | `cloud` |

### 3.2 Assemblers a atualizar

- `CoreRulesWriter` — refs `core/`, `databases/`, `security/` → prefixar com `knowledge/`
- `LanguageKpWriter` — ref `languages/{lang}` → `knowledge/languages/{lang}`
- `FrameworkKpWriter` — ref `frameworks/{fw}` → `knowledge/frameworks/{fw}`
- `RulesConditionals` — refs `databases/`, `security/` → prefixar com `knowledge/`
- `RulesInfraConditionals` — refs `cloud-providers/`, `infrastructure/` → prefixar com `knowledge/`
- `PatternsAssembler` — probe `"patterns"` → `"knowledge/patterns"`
- `ProtocolsAssembler` — probe `"protocols"` → `"knowledge/protocols"`
- `GithubMcpAssembler` — probe `"core"` → `"knowledge/core"`

### 3.3 Coordenacao com story-0020-002

O `RulesAssembler` usa `resourcesDir` (raiz dos recursos) para resolver tanto rules quanto knowledge dirs. Apos story-0020-002, a raiz deve continuar acessivel. Nesta story, os paths relativos mudam de `"databases"` para `"knowledge/databases"` dentro dos writers. Se story-0020-002 ja foi concluida, os writers ja estarao usando a raiz correta; apenas os sufixos de path mudam.

## 3.5 Entrega de Valor

- **Valor Principal:** Base de conhecimento alinhada com secoes YAML, facilitando localizacao e manutencao
- **Metrica de Sucesso:** 9 diretorios reorganizados, todos os assemblers resolvendo paths corretos
- **Impacto no Negocio:** Developer experience melhorada: ao adicionar conhecimento sobre um novo framework, o desenvolvedor sabe exatamente onde colocar (`knowledge/frameworks/`)
