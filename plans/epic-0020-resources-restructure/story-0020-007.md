## 3. Descricao

Como **engenheiro de qualidade**, eu quero validar que a reestruturacao de recursos nao causou nenhuma regressao, para que possamos garantir que o output gerado permanece identico em todos os perfis de configuracao.

### Contexto

O projeto possui golden files (arquivos de referencia) para cada perfil de configuracao. Apos mover todos os recursos (stories 002-006), e necessario regenerar os golden files e confirmar que o output e byte-for-byte identico. Qualquer divergencia indica um path incorreto ou um assembler nao atualizado.

### 3.1 Perfis a validar

Todos os perfis existentes devem ser validados:
- go-gin
- java-quarkus
- java-spring
- java-spring-cqrs-es
- java-spring-event-driven
- java-spring-hexagonal
- java-picocli-cli
- kotlin-ktor
- python-click-cli
- python-fastapi
- rust-axum
- typescript-commander-cli
- typescript-nestjs

### 3.2 Processo de validacao

1. Capturar output pre-migracao (se nao capturado antes das stories 002-006)
2. Regenerar golden files com `GoldenFileRegenerator` ou equivalente
3. Comparar byte-for-byte com golden files existentes
4. Rodar suite completa de testes (`mvn test`)
5. Verificar cobertura >= 95% line, >= 90% branch
6. Corrigir qualquer divergencia encontrada

### 3.3 Divergencias esperadas

Nenhuma. A reestruturacao e puramente interna — o output gerado nao deve mudar.

## 3.5 Entrega de Valor

- **Valor Principal:** Garantia formal de zero regressao, confianca para merge na main
- **Metrica de Sucesso:** 1,384+ testes passando, cobertura nos thresholds, golden files identicos
- **Impacto no Negocio:** Risco zero de quebra em projetos existentes que usam o gerador
