## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero um metodo `resolveResourceDir` no `ResourceResolver` que retorne diretamente o diretorio resolvido sem aritmetica de profundidade, para que os assemblers possam referenciar paths aninhados sem risco de erro no calculo de `depth`.

### Contexto

O metodo atual `resolveResourcesRoot(String probe)` localiza um diretorio no classpath e navega N niveis acima para encontrar a raiz de recursos. Com a nova estrutura aninhada (`targets/claude/agents/` = 3 niveis), cada assembler precisaria calcular corretamente a profundidade — fonte frequente de bugs. O novo metodo elimina essa complexidade.

### 3.1 Novo metodo resolveResourceDir

Adicionar ao `ResourceResolver`:

```java
public static Path resolveResourceDir(String relativePath)
```

O metodo recebe um caminho relativo (ex: `"targets/claude/agents"`) e retorna o `Path` absoluto para esse diretorio. Internamente, resolve a raiz dos recursos via classpath e appenda o caminho relativo. Nao usa aritmetica de `depth`.

### 3.2 Compatibilidade

O metodo existente `resolveResourcesRoot(String probe, int depth)` permanece inalterado. Nenhum assembler e modificado nesta story — apenas o novo metodo e adicionado com testes.

## 3.5 Entrega de Valor

- **Valor Principal:** API de resolucao de paths simplificada que previne bugs de calculo de profundidade
- **Metrica de Sucesso:** Novo metodo coberto com testes para paths de 1, 2 e 3 niveis de profundidade
- **Impacto no Negocio:** Habilitacao segura de todas as stories subsequentes de migracao
