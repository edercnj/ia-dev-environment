# Épico: Enforcement de Reviews Obrigatórias em x-story-implement (EPIC-0053)

**Autor:** Eder Celeste Nunes Junior
**Data:** 2026-04-19
**Versão:** 1.0.0
**Status:** Em Refinamento

## 1. Visão Geral

**Chave Jira:** —

Subagentes executando `x-story-implement` têm silenciosamente omitido as steps de review
(Step 3.4 Specialist Review e Step 3.6 Tech Lead Review) mesmo sem o flag
`--skip-verification` ser passado. A evidência foi observada no EPIC-0042 (story-0042-0001:
`reviewsExecuted: {specialist: false, techLead: false}`). O problema raiz é a ausência de
linguagem explicitamente obrigatória nessas steps — o texto atual não deixa claro que
reviews são não-negociáveis por padrão.

Este épico grava a obrigatoriedade diretamente no source da skill (`x-story-implement/SKILL.md`),
adiciona um error code nomeado (`PROTOCOL_VIOLATION`) para rastrear violações, e cria um teste
de golden file que valida a presença dos markers no output regenerado. A mudança é
**documentação-only**: nenhum código Java de produção é alterado.

**Fora de escopo:** implementação de um flag `--skip-review` funcional (documentado como
RESERVED); alterações em outras skills; mudanças de comportamento em runtime.

## 4. Regras de Negócio Transversais (Source of Truth)

> Regras que se aplicam a múltiplas histórias. Cada história referencia as regras pelo ID.

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-001]** | Mandatory Review Execution | Step 3.4 (Specialist Review via `x-review`) e Step 3.6 (Tech Lead Review via `x-review-pr`) são **NON-NEGOTIABLE** por padrão. Devem executar em toda story a menos que `--skip-verification` seja explicitamente passado. Um subagente que omite qualquer uma dessas steps sem o flag está em violação de protocolo. |
| **[RULE-002]** | Protocol Violation Logging | Toda violação de protocol (omissão de review sem flag) DEVE ser reportada emitindo um error code nomeado: `"PROTOCOL_VIOLATION: Step 3.4 skipped without --skip-verification"` ou `"PROTOCOL_VIOLATION: Step 3.6 skipped without --skip-verification"`. O code auxiliar é `"REVIEW_SKIPPED_WITHOUT_FLAG"` referenciado na Review Policy global. |
| **[RULE-003]** | Source-of-Truth SKILL.md | Toda mudança de comportamento ou linguagem obrigatória da skill `x-story-implement` DEVE ser feita no source file `java/src/main/resources/targets/claude/skills/core/dev/x-story-implement/SKILL.md`. O output `.claude/skills/x-story-implement/SKILL.md` é gerado via `mvn process-resources` e NUNCA editado manualmente. |
