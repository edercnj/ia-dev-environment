# STORY-0033-0002: Add TaskCreate/TaskUpdate for Execution Visibility (Level 2)

## Status: Pendente

## Parent Epic: EPIC-0033

## Depends On: STORY-0033-0001

## Summary

Add `TaskCreate` and `TaskUpdate` tool invocations to `x-dev-epic-implement` and `x-dev-story-implement` skills to provide Level 2 visibility in the Claude Code task list. Level 2 means: 1 task per story (epic level), 1 task per phase + 1 per task-execution (story level).

## User Story

As a **developer running skill orchestration**, I want to see real-time progress in the Claude Code task list showing which phase and task is currently executing, so that I can track progress and detect stalled executions without reading log output.

## Background

Currently, no skill uses `TaskCreate`/`TaskUpdate`. All tracking is via custom `execution-state.json` files that are invisible in the Claude Code UI. During 30+ minute executions, the user sees an empty task list with no indication of progress.

The Level 2 granularity creates ~10-15 tasks per story:
- 1 task for Phase 0 (Preparation)
- 1 task for Phase 1 (Planning)
- N tasks for Phase 2 (1 per task in execution loop)
- 1 task for Phase 3 (Verification)

At epic level: 1 task per story dispatch.

| Duplicate tracking (execution-state.json + TaskCreate) | TaskCreate is for UI visibility only; execution-state.json remains source of truth for resume/checkpoint |
