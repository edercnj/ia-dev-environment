# STORY-0033-0003: Add Task Tracking to Planning Subagent Prompts

## Status: Pendente

## Parent Epic: EPIC-0033

## Depends On: STORY-0033-0002

## Summary

Add `TaskCreate`/`TaskUpdate` instructions to the text-based prompts sent to planning subagents in Phase 1 (1B-1F). These subagents are currently black boxes — the user cannot see their progress or know which ones have completed.

## User Story

As a **developer waiting for planning to complete**, I want to see individual planning subagent progress in the Claude Code task list, so that I know which planning artifacts are ready and which are still being generated.

## Background

Phase 1 of x-dev-story-implement launches up to 5 planning subagents in a SINGLE message (parallel):
- 1B: Implementation Plan (Senior Architect subagent)
- 1B: Test Plan (via Skill x-test-plan)
- 1C: Task Decomposition (via Skill x-lib-task-decomposer)
- 1D: Event Schema Design (subagent, conditional)
- 1E: Security Assessment (via Skill x-threat-model, fallback to subagent)
- 1F: Compliance Assessment (subagent, conditional)

Subagents receive text prompts but don't create any tasks. The user sees "Phase 1: Planning" as a single task (from STORY-0033-0002) but can't see individual subagent progress.

## Acceptance Criteria

### AC-1: Planning subagent prompts include TaskCreate instruction

```gherkin
DADO o arquivo de referência planning-phases.md
QUANDO um prompt de subagente de planejamento é definido (Steps 1B, 1D, 1E fallback, 1F)
ENTÃO o prompt DEVE incluir uma instrução TaskCreate no início:
  "Before starting, create a task: TaskCreate(description: 'Planning: {artifact-type} — Story {storyId}')"
E o prompt DEVE incluir uma instrução TaskUpdate no final:
  "When done, update: TaskUpdate(id: N, status: 'completed')"
```

### AC-2: Skill-invoked planners also create tasks

```gherkin
DADO invocações da Phase 1 que usam Skill tool diretamente (1B test plan, 1C task decomposition, 1E threat model)
QUANDO o orquestrador as invoca
ENTÃO o orquestrador (x-dev-story-implement) DEVE criar um TaskCreate ANTES da invocação do Skill
E TaskUpdate APÓS a invocação do Skill completar
E o skill invocado NÃO precisa criar tasks (o orquestrador gerencia)
```

### AC-3: Task list shows individual planning progress

```gherkin
DADO que Phase 1 está executando com 5 subagentes paralelos
QUANDO o usuário visualiza a task list do Claude Code
ENTÃO ele DEVE ver tasks individuais como:
  "[completed] Planning: Architecture Plan — Story 0033-0001"
  "[in_progress] Planning: Implementation Plan — Story 0033-0001"
  "[completed] Planning: Test Plan — Story 0033-0001"
  "[in_progress] Planning: Task Decomposition — Story 0033-0001"
  "[completed] Planning: Security Assessment — Story 0033-0001"
```

### AC-4: Skipped planners do not create tasks

```gherkin
DADO uma fase de planejamento que é pulada (reuso de artefato, condição não atendida)
QUANDO o orquestrador avalia a condição de skip
ENTÃO NENHUM TaskCreate é emitido para o planner pulado
E uma mensagem de log é emitida: "Skipping {artifact-type}: {reason}"
```

## Data Contracts

### TaskCreate Description Pattern for Planning

```
# Orchestrator-managed (Skill invocations):
"Planning: Test Plan — Story {storyId}"
"Planning: Task Decomposition — Story {storyId}"
"Planning: Security Assessment — Story {storyId}"

# Subagent-managed (text prompt planners):
"Planning: Implementation Plan — Story {storyId}"
"Planning: Event Schema — Story {storyId}"
"Planning: Compliance Assessment — Story {storyId}"
```

### Subagent Prompt Addition Template

```markdown
> **FIRST ACTION:** Create a task for tracking:
>   TaskCreate(description: "Planning: {artifact-type} — Story {storyId}")
>
> ... (existing prompt content) ...
>
> **LAST ACTION:** Update the task:
>   TaskUpdate(id: <task-id-from-create>, status: "completed")
```

## Risks

| Risk | Mitigation |
|------|------------|
| Subagents may not have TaskCreate tool | general-purpose subagents have all tools by default |
| Parallel subagents may create tasks out of order | Task order in UI is non-deterministic for parallel launches; this is acceptable |
| Extra context consumption in subagent prompts | TaskCreate/TaskUpdate instructions add ~100 tokens per subagent — negligible |
