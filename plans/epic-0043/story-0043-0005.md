## 3. Descrição

Como **desenvolvedor executando `/x-review-pr`**, eu quero que após esgotar os retries automáticos de auto-remediation (quando TL retorna NO-GO e os 2 retry-loops não convergem), a skill me apresente um menu estruturado em vez de terminar silenciosamente, para que eu possa escolher entre tentar mais uma rodada de remediation, fazer fix manual via `x-pr-fix`, ou abortar o review.

Hoje quando o Tech Lead Review devolve NO-GO, `x-review-pr` dispara auto-remediation agents (para `TEST_FAILURE`, `COVERAGE_GAP`, `CODE_QUALITY`) com até 2 retries. Se nenhum retry converge, a skill termina retornando NO-GO sem ação adicional. O operador descobre pelo log e precisa decidir manualmente o que fazer — sem nenhum atalho no fluxo. Esta história adiciona um gate ao fim do retry loop com as **3 opções canônicas da Rule 002**: PROCEED (dispara mais 2 retries de auto-remediation — semântica específica da skill, documentada em `description`) / FIX-PR (invoca `x-pr-fix` no PR atual, loop-back ao menu) / ABORT (encerra com NO-GO final). Não altera o comportamento de auto-remediation dentro dos retries — apenas captura o fim da cadeia.

### 3.1 Localização da Mudança

- Arquivo primário: `java/src/main/resources/targets/claude/skills/core/review/x-review-pr/SKILL.md`
- Ponto de inserção: no fim da seção de NO-GO auto-remediation (~linhas 249–308), imediatamente antes do return/exit atual
- **Nuance:** skill hoje não tem state file persistente do review. Esta story introduz arquivo opcional `plans/review/<pr-number>/state.json` com schema Rule 20 §5.1 (apenas para resume via `--resume-review <pr>`).

### 3.2 Comportamento Após Retrofit

- Retry auto-remediation esgotado (2 retries sem convergir) → menu com EXATAMENTE 3 opções (slot canônico + `description` skill-specific):
  - **PROCEED** (slot 1 canônico) — `label: "Retry remediation (+2 loops)"`; `description: "Dispara novo par de retries de auto-remediation agents; ao fim, reapresenta este menu"`. A "ação de avançar" nesta skill é re-disparar a remediation; o nome `PROCEED` é invariante (RULE-002) e a semântica concreta vive na `description`.
  - **FIX-PR** (slot 2 canônico) — `Skill(skill: "x-pr-fix", args: "<PR>")`; loop-back ao menu após retorno
  - **ABORT** (slot 3 canônico) — encerra com decisão final NO-GO + code `REVIEW_REMEDIATION_EXHAUSTED`
- `--non-interactive` → comportamento atual (exit com NO-GO sem menu)
- 3 invocações consecutivas de PROCEED ou FIX-PR sem convergir → gate encerra automaticamente com `REVIEW_FIX_LOOP_EXCEEDED` e orientação textual para intervenção manual (nenhuma 4ª opção, per RULE-002)

### 3.3 Backward Compatibility

- Skill hoje é stateless entre invocações; o novo state file é opt-in (apenas escrito quando operador escolhe FIX-PR no menu)
- Chamadas existentes sem interação humana (pipelines orquestrados) passam `--non-interactive` e funcionam como antes

## 3.5 Entrega de Valor

- **Valor Principal:** NO-GO não vira dead-end. Operador tem 3 ações acionáveis dentro da skill.
- **Métrica de Sucesso:** 100% dos NO-GO com retry exhausted em invocações manuais resultam em menu em vez de exit silencioso.
- **Impacto no Negócio:** Reduz custo de contexto-switch em reviews que precisam fix manual (dev não precisa sair da skill, abrir outro terminal, rodar x-pr-fix, decidir próximo passo).

```mermaid
stateDiagram-v2
    [*] --> TL_REVIEW
    TL_REVIEW --> GO_EXIT: decisão = GO
    TL_REVIEW --> AUTO_REMEDIATE: decisão = NO-GO
    AUTO_REMEDIATE --> RETRY_LOOP: até 2 retries

    MENU --> FIX_PR: slot 2
    MENU --> ABORT: slot 3

    PROCEED --> AUTO_REMEDIATE: +2 retries
    AUTO_REMEDIATE --> GUARD_P: return
    GUARD_P --> MENU: attempts < 3
    GUARD_P --> AUTO_ABORT: attempts == 3 (REVIEW_FIX_LOOP_EXCEEDED)

    FIX_PR --> XFIX: Skill(skill: "x-pr-fix", args: "<PR>")
    XFIX --> GUARD_F: return
    GUARD_F --> MENU: attempts < 3
    GUARD_F --> AUTO_ABORT: attempts == 3 (REVIEW_FIX_LOOP_EXCEEDED)

    GO_EXIT --> [*]: decisão final GO
    NO_GO_EXIT --> [*]: decisão final NO-GO (sem menu)
    ABORT --> [*]: REVIEW_REMEDIATION_EXHAUSTED
    AUTO_ABORT --> [*]: encerra com orientação manual
```
