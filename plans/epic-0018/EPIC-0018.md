# EPIC-0018: Reestruturação de Diretórios e Adoção Formal de SDD

> **Status**: Concluído
> **Epic ID**: EPIC-0018
> **Feature Slug**: sdd-directory-restructure
> **Criado em**: 2026-04-04

---

## 1. BUSINESS OBJECTIVE

O repositório ia-dev-environment cresceu com uma estrutura de diretórios que
mistura concerns: specs (input humano), plans (artefatos gerados pelo agente) e
results (outputs de execução) convivem dentro de docs/ sem separação clara.
Isso dificulta saber o que é fonte de verdade versus o que é derivado, e impede
que o tooling SDD (skills /x-story-epic-full, /x-dev-lifecycle, etc.) deposite
outputs em locais previsíveis e rastreáveis.

O objetivo é reorganizar a estrutura de diretórios para refletir o fluxo SDD
de forma que qualquer desenvolvedor ou agente de IA entenda imediatamente onde
cada tipo de artefato vive, sem ambiguidade.

---

## 2. SCOPE BOUNDARIES

### In Scope

- Criar nova estrutura de diretórios: `steering/`, `specs/`, `plans/`, `results/`, `contracts/`, `adr/`
- Migrar conteúdo existente de `docs/` para os diretórios corretos:
  - `docs/specs/` -> `specs/`
  - `docs/stories/` -> `plans/`
  - `docs/adr/` -> `adr/`
  - `docs/audits/` -> `results/audits/`
  - `docs/architecture/` -> `steering/`
  - `docs/guides/` + `docs/runbook/` -> `results/runbooks/`
- Atualizar paths de output nos assemblers Java (`java/src/main/java/dev/iadev/assembler/`)
- Atualizar paths hardcoded nos skill templates (`java/src/main/resources/skills-templates/`, `java/src/main/resources/github-skills-templates/`)
- Atualizar utilitários de suporte (`AssemblerTarget`, `FileCategorizer`, `OverwriteDetector`)
- Atualizar `CLAUDE.md` raiz e `README.md` para refletir a nova estrutura
- Criar arquivos de `steering/` iniciais para o projeto ia-dev-environment

### Out of Scope

- Modificar a lógica de geração de conteúdo das skills (apenas paths)
- Modificar skills geradas em `.claude/skills/`, `.github/skills/`, `.codex/skills/` (são output, não fonte de verdade)
- Modificar `.claude/agents/`, `.github/agents/` ou qualquer configuração de agentes
- Criar novos skills ou agentes
- Implementar qualquer feature nova além da reorganização estrutural
- Alterar testes existentes em `tests/` (exceto se referenciam paths de output)

### Deferred

- Regenerar output do CLI após mudança de templates (requer release e re-execução em projetos downstream)
- Atualizar smoke tests que validam estrutura de diretórios gerada (depende de decisão sobre backwards-compatibility)
- Migrar referências em `AGENTS.md` e `.codex/` (gerados — serão atualizados na próxima regeneração)
