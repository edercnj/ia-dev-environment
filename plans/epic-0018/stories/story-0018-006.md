## 3. DESCRIPTION

### State Before

O gerador Java produz artefatos com paths de output que apontam para `docs/` e subdiretórios.
Esses paths estão hardcoded em três locais:

1. **Assemblers Java** (10 classes) — definem `OUTPUT_SUBDIR` como constantes string
2. **Skill templates** (47 arquivos) — contêm instruções com paths como `docs/stories/epic-XXXX/`
3. **Classes de suporte** (3 classes) — `AssemblerTarget`, `FileCategorizer`, `OverwriteDetector`

| DocsAdrAssembler | ADR_OUTPUT_SUBDIR | `"docs/adr"` | ~67 |

| `docs/plans/` | `plans/` | skill templates |

Classes de suporte atualizadas:

| Classe | Mudança |
|---|---|
| AssemblerTarget | Reavaliar enum DOCS — split ou remap para novos targets |
| FileCategorizer | Atualizar detecção de paths `docs/` para novos prefixos |
| OverwriteDetector | Atualizar ARTIFACT_DIRS com novos diretórios |

`mvn verify` passa sem erros após todas as mudanças.

### O que esta story NAO faz

- Não modifica skills geradas em `.claude/skills/`, `.github/skills/`, `.codex/skills/` deste repositório
- Não modifica a lógica de geração (assembler pipeline, template engine, copy helpers)
- Não altera a estrutura de arquivos no repositório raiz (migração já feita nas stories anteriores)
- Não cria novos assemblers — apenas atualiza paths nos existentes
- Não altera testes existentes exceto se referenciam paths de output

### Possibilidade de Decomposição

Se durante a implementação esta story se revelar muito grande para uma sessão única,
pode ser decomposta em:

- **006a**: Assemblers Java (10 classes + 3 suporte) — mudanças pequenas e pontuais
- **006b**: Skill templates — skills-templates/ (24 arquivos, 144 ocorrências)
- **006c**: GitHub skill templates — github-skills-templates/ (23 arquivos, 120 ocorrências)

A ordem seria 006a -> 006b -> 006c, com `mvn verify` após cada sub-story.
