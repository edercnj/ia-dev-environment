## 3. Descrição

Como **engenheiro de plataforma**, eu quero que a análise de conflitos do Phase 0.5 emita apenas warnings sem bloquear execução paralela, garantindo que o throughput de stories não seja reduzido por análises preventivas.

Atualmente, Phase 0.5 (Sections 0.5.1-0.5.5) detecta overlaps de arquivos entre stories de uma mesma fase e particiona as stories em "Parallel Batch" e "Sequential Queue". Stories com `code-overlap-high` ou `unpredictable` são forçadas a executar sequencialmente. Com per-story PRs, conflitos são resolvidos no nível do PR (rebase antes de merge, resolução via GitHub UI), não no nível de worktree merge. A análise de overlap passa a ser apenas informativa.

### 3.1 Alteração de Section 0.5.4 (Generate Adjusted Execution Plan)

- Remover particionamento em "Parallel Batch" e "Sequential Queue"
- Todas as stories ficam no batch paralelo, independente do nível de overlap
- O output muda de plano de execução ajustado para relatório de warnings
- Stories com `code-overlap-high`: emitir WARNING, não demover para sequencial
- Stories `unpredictable`: emitir WARNING, não demover para sequencial

### 3.2 Alteração de Section 0.5.5 (Integration with Core Loop)

- Remover lógica de particionamento que o core loop consome
- O core loop trata todas as stories como paralelo-elegíveis
- Se preflight analysis existe, warnings são logados mas não afetam dispatch
- Adicionar nota: "Com per-story PRs, conflitos de overlap são resolvidos no nível do PR via rebase/merge"

### 3.3 Formato do relatório advisory

- O arquivo `preflight-analysis-phase-N.md` continua sendo gerado
- Tabela de overlaps continua com classificações (config-only, code-overlap-low, code-overlap-high)
- Seção "Adjusted Execution Plan" muda para "Advisory Warnings"
- Lista de warnings com recomendação: "Considere rebasing manual se PRs conflitarem"

## 3.5 Entrega de Valor

- **Valor Principal:** Maximização do throughput de stories — análise de conflitos não bloqueia mais execução paralela, acelerando a implementação de épicos grandes
- **Métrica de Sucesso:** Todas as stories de uma fase são despachadas em paralelo independente do nível de overlap, com warnings logados para visibilidade
- **Impacto no Negócio:** Redução do tempo total de implementação de épicos, com manutenção da visibilidade de riscos de conflito
