## 3. Descrição

Como **engenheiro de plataforma**, eu quero que o SKILL.md esteja completamente auto-consistente e pronto para uso após todas as mudanças das stories anteriores, garantindo que todas as seções, referências e exemplos refletem o modelo per-story PR.

Esta story é a finalização do épico. Seu objetivo é consolidar todas as mudanças feitas nas stories 0001-0007, verificar consistência interna do SKILL.md, atualizar a Phase 3 (Verification), o `--dry-run` output, e garantir que nenhuma referência órfã à branch épica, mega-PR, ou rebase-before-merge permanece no documento.

### 3.1 Atualização de Phase 3 (Verification)

- Section 3.1 (Epic-Level Test Suite): rodar na `main` após todos os PRs merged — ajustar referência
- Section 3.2 (DoD Checklist): atualizar itens:
  - Remover: "PR created or failure documented"
  - Adicionar: "All story PRs merged to main or documented as FAILED/BLOCKED"
  - Atualizar: referências de tech lead review para per-story reviews
- Section 3.3 (Final Status Determination): basear em PR merge status, não em branch épica
- Section 3.4 (Completion Output): substituir URL de mega-PR por tabela de PRs de stories

### 3.2 Atualização do --dry-run output

- Phase 0, Step 8: o output do dry-run deve mostrar:
  - Modelo: "per-story PR" (ou "single-pr (legacy)" com --single-pr)
  - Para cada story: branch estimada, fase, dependências, PR targeting main
  - Warnings de overlap (advisory, não blocking)

### 3.3 Verificação de consistência interna

- Grep por "epic branch", "feat/epic-{epicId}-full-implementation", "1.4b", "1.4c", "REBASING" — devem existir APENAS dentro do guard `--single-pr`
- Verificar que todas as referências cruzadas entre seções são válidas
- Verificar que o schema do execution-state.json está completo e consistente
- Verificar que a tabela de flags inclui todas as novas flags (--single-pr, --auto-merge)

### 3.4 Atualização do argument-hint no frontmatter

- Adicionar `--single-pr` e `--auto-merge` ao argument-hint
- Formato: `[EPIC-ID] [--phase N] [--story story-XXXX-YYYY] [--skip-review [--reason "justification"]] [--dry-run] [--resume] [--sequential] [--skip-smoke-gate] [--auto-merge] [--single-pr]`

### 3.5 Atualização de "Integration Notes" (se existir)

- Verificar se existe seção de integration notes no final do SKILL.md
- Atualizar todas as referências para refletir modelo per-story PR
- Remover referências a branch épica, rebase-before-merge, conflict resolution subagent

## 3.5 Entrega de Valor

- **Valor Principal:** SKILL.md auto-consistente e pronto para uso, sem referências órfãs ao modelo antigo — permite que engenheiros usem o skill imediatamente após o épico ser implementado
- **Métrica de Sucesso:** Zero referências órfãs a branch épica, Sections 1.4b/1.4c, ou status REBASING fora do guard --single-pr. Todas as seções referenciadas existem.
- **Impacto no Negócio:** Skill funcional e documentado, permitindo adoção imediata do modelo per-story PR pela equipe
