## 3. Descricao

Como desenvolvedor do ia-dev-environment, eu quero que o checklist do Database Engineer no x-review seja expandido de 8 items (/16 pontos) para 20 items (/40 pontos), incluindo items categoria-especificos, para que o review de banco cubra naming conventions, encryption, partitioning e padroes especificos de cada tipo de banco.

### 3.1 Contexto

O skill x-review em `targets/claude/skills/core/x-review/SKILL.md` possui atualmente um checklist de 8 items para o Database Engineer. A expansao adiciona items universais e items condicionais por categoria de banco.

### 3.2 Items Universais (aplicaveis a todos os bancos)

Alem dos 8 items originais, adicionar:

1. Naming conventions compliance (tabelas, colunas, indices seguem convencao do banco)
2. Soft delete pattern (quando aplicavel, uso de `deleted_at` ou equivalente)
3. Temporal audit trail (campos `created_at`, `updated_at` em todas as entidades)
4. Encryption-at-rest para colunas sensiveis (PII, credentials)
5. FK indexing (todo foreign key possui indice correspondente, para SQL)
6. Partitioning evaluation para tabelas grandes (> 1M rows estimado)
7. Connection pool monitoring metrics (pool size, wait time, timeout config)
8. Dead tuple/compaction monitoring (VACUUM para SQL, compaction para NoSQL)

### 3.3 Items Condicionais por Categoria

Items que aparecem apenas quando o banco pertence a determinada categoria:

- **NoSQL (MongoDB, Cassandra, EventStoreDB):** Schema validation enforcement (JSON Schema para MongoDB, schema registry)
- **Graph (Neo4j, Neptune):** Graph traversal depth limits (prevencao de queries sem bound)
- **Time-Series (InfluxDB, TimescaleDB):** Cardinality management (limites de tags/labels, series explosion prevention)
- **Distributed/NewSQL (YugaByteDB, CockroachDB, TiDB):** Shard key selection review (hot spots, data distribution uniformity)

### 3.4 Pontuacao

- Cada item vale 2 pontos (mantendo padrao existente)
- 20 items universais + condicionais = pontuacao maxima de /40 pontos
- Items condicionais substituem items universais quando aplicavel (total permanece 20)

### 3.5 Preservacao dos Items Originais

Os 8 items originais do checklist DEVEM permanecer na lista expandida. A expansao e aditiva, nunca substitutiva dos items pre-existentes.

## 3.5 Entrega de Valor

- **Valor Principal:** Review de banco cobre 20 items incluindo padroes categoria-especificos
- **Metrica de Sucesso:** Checklist expandido de 8 para 20 items com pontuacao maxima de /40
- **Impacto no Negocio:** Reviews de banco detectam problemas de naming, encryption e partitioning que antes passavam despercebidos
