## 3. Descricao

Como desenvolvedor do ia-dev-environment, eu quero knowledge files para YugaByteDB, CockroachDB e TiDB seguindo o padrao de 3 arquivos por banco, para que projetos usando bancos SQL distribuidos recebam padroes de sharding, consensus e transacoes globais.

### 3.1 Contexto

Bancos NewSQL utilizam o padrao de arquivo SQL (`types-and-conventions.md`) porque sao compativeis com SQL. O arquivo de principios (`newsql-principles.md`) cobre: fundamentos de SQL distribuido, consensus Raft/Paxos, sincronizacao de relogio (hybrid logical clocks), transacoes globais, posicionamento CAP como sistemas CP e niveis de compatibilidade SQL.

### 3.3 Conteudo Esperado do newsql-principles.md

- Fundamentos de SQL distribuido (distributed query execution, data distribution)
- Consensus Raft/Paxos (leader election, log replication, quorum)
- Sincronizacao de relogio (hybrid logical clocks, clock skew handling)
- Transacoes globais (distributed transactions, serializable isolation)
- Posicionamento CAP como sistemas CP (consistency over availability)
- Niveis de compatibilidade SQL (PostgreSQL-wire vs MySQL-wire)

## 3.5 Entrega de Valor

- **Valor Principal:** Projetos com bancos distribuidos recebem orientacao de sharding e consensus
- **Metrica de Sucesso:** 10 knowledge files criados (1 principios + 9 banco-especificos) + 3 settings files
- **Impacto no Negocio:** Equipes usando YugaByteDB, CockroachDB ou TiDB recebem padroes de SQL distribuido, evitando erros comuns de distributed transactions e clock synchronization
