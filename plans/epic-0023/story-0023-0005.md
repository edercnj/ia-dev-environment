## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero knowledge files para ClickHouse e Apache Druid seguindo o padrao de 3 arquivos por banco, para que projetos OLAP/analytics recebam padroes de modelagem colunar, ingestao e otimizacao analitica.

Bancos colunares operam sobre paradigmas fundamentalmente diferentes dos relacionais: armazenamento orientado a colunas em vez de linhas, compressao por tipo de dado, materialized views como mecanismo primario de pre-agregacao, e padroes de ingestao batch/real-time em vez de transacoes ACID tradicionais. Esta story cria o conteudo de referencia completo para que a IA possa orientar projetos que utilizam ClickHouse ou Apache Druid.

### 3.1 Arquivos Common (columnar/common/)

- `columnar-principles.md` -- Fundamentos de bancos colunares:
  - Column-oriented storage: como dados sao armazenados e lidos por coluna
  - OLAP vs OLTP tradeoffs: quando usar banco colunar vs relacional
  - Estrategias de compressao: LZ4, ZSTD, codec por tipo de dado
  - Materialized views: conceito e uso como mecanismo de pre-agregacao
  - Padroes de ingestao: batch vs real-time, micro-batch, streaming

### 3.2 Arquivos ClickHouse (columnar/clickhouse/)

- `types-and-conventions.md` -- Familia de engines MergeTree (MergeTree, ReplacingMergeTree, SummingMergeTree, AggregatingMergeTree, CollapsingMergeTree, VersionedCollapsingMergeTree), tipos de dados, particionamento por data, politicas TTL, convencoes de nomenclatura
- `migration-patterns.md` -- ALTER TABLE patterns permitidos e proibidos, zero-downtime schema changes, ferramenta clickhouse-migrations, estrategias de backfill, column type changes
- `query-optimization.md` -- Design de primary key (sparse index), skip indexes (minmax, set, bloom_filter, ngrambf), materialized views para pre-agregacao, otimizacao de queries distribuidas, EXPLAIN e system.query_log

### 3.3 Arquivos Druid (columnar/druid/)

- `types-and-conventions.md` -- Segments como unidade de armazenamento, dimensoes vs metricas, ingestion specs (native batch, Kafka, Kinesis), rollup patterns, data schemas (flat vs nested), granularidades
- `migration-patterns.md` -- Segment compaction (auto vs manual), evolucao de schema (adicao de dimensoes/metricas), re-ingestion de dados historicos, task management, supervisor patterns
- `query-optimization.md` -- Query granularity optimization, segment pruning via time filtering, lookup optimization para joins, tuning de broker/historical, cache configuration, query context parameters

### 3.4 Settings Files

- `targets/claude/settings/database-clickhouse.json` -- Permite `clickhouse-client` como ferramenta CLI
- `targets/claude/settings/database-druid.json` -- Permite `curl *druid*` para interacao via API REST

## 3.5 Entrega de Valor

- **Valor Principal:** Projetos OLAP recebem orientacao completa de modelagem colunar e otimizacao analitica
