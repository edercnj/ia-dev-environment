## 3. Descricao

Como desenvolvedor do ia-dev-environment, eu quero knowledge files para Elasticsearch e OpenSearch seguindo o padrao de 3 arquivos por banco, para que projetos com search engines recebam padroes de indexacao, mapping design e otimizacao de relevancia.

### 3.1 Contexto

Search engines utilizam `modeling-patterns.md` (nao `types-and-conventions.md`) pois sao NoSQL-style. O arquivo de principios (`search-principles.md`) cobre: fundamentos de inverted index, relevance scoring (TF-IDF, BM25), analysis pipeline (tokenizers, filters), mapping design, casos de uso search vs analytics, modelo de sharding e replicacao.

### 3.2 Arquivos a Criar

**Principios comuns:**
- `knowledge/databases/search/common/search-principles.md`

**Elasticsearch (3 arquivos):**
- `knowledge/databases/search/elasticsearch/modeling-patterns.md`
- `knowledge/databases/search/elasticsearch/migration-patterns.md`
- `knowledge/databases/search/elasticsearch/query-optimization.md`

**OpenSearch (3 arquivos):**
- `knowledge/databases/search/opensearch/modeling-patterns.md`
- `knowledge/databases/search/opensearch/migration-patterns.md`
- `knowledge/databases/search/opensearch/query-optimization.md`

**Settings files:**
- `targets/claude/settings/database-elasticsearch.json`
- `targets/claude/settings/database-opensearch.json`

### 3.3 Conteudo Esperado do search-principles.md

- Fundamentos de inverted index (term dictionary, postings list, doc values)
- Relevance scoring (TF-IDF, BM25, function_score, boosting)
- Analysis pipeline (analyzers, tokenizers, token filters, char filters)
- Mapping design (dynamic vs explicit mapping, field types, multi-fields)
- Casos de uso: search (full-text, fuzzy, autocomplete) vs analytics (aggregations, metrics)
- Modelo de sharding e replicacao (primary/replica, shard allocation, routing)

### 3.4 Diferenca entre Elasticsearch e OpenSearch

- Elasticsearch e OpenSearch compartilham a mesma base (fork a partir da v7.10)
- Divergencias: licenciamento, plugins proprietarios, roadmap de features
- Knowledge files devem cobrir diferencas de API onde aplicavel (ex: security plugin nativo no OpenSearch vs X-Pack no Elasticsearch)

## 3.5 Entrega de Valor

- **Valor Principal:** Projetos com search engines recebem orientacao de indexacao e relevancia
- **Metrica de Sucesso:** 7 knowledge files criados (1 principios + 6 banco-especificos) + 2 settings files
- **Impacto no Negocio:** Equipes usando Elasticsearch ou OpenSearch recebem padroes de mapping design e relevance tuning, evitando problemas de performance e resultados irrelevantes
