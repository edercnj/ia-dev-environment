## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero knowledge files para Neo4j e Amazon Neptune seguindo o padrao de 3 arquivos por banco, para que projetos usando graph databases recebam convencoes, padroes de migracao e otimizacao de queries especificos.

Graph databases sao fundamentalmente diferentes de bancos relacionais: operam sobre vertices (nodes) e arestas (relationships) em vez de tabelas, usam linguagens de travessia (Cypher, Gremlin, SPARQL) em vez de SQL, e possuem padroes de indexacao e otimizacao proprios. Esta story cria o conteudo de referencia completo para que a IA possa orientar projetos que utilizam Neo4j ou Amazon Neptune.

### 3.1 Arquivos Common (graph/common/)

- `graph-principles.md` -- Fundamentos de graph databases:
  - Property graph vs RDF: diferencias, quando usar cada um
  - Traversal vs query paradigms: travessia imperativa (Gremlin) vs declarativa (Cypher/SPARQL)
  - Posicionamento CAP: consistencia vs disponibilidade em graph DBs
  - Arvore de decisao: quando usar graph vs relacional (fan-out, profundidade de travessia, query patterns)

### 3.2 Arquivos Neo4j (graph/neo4j/)

- `modeling-patterns.md` -- Node/relationship design, convencoes Cypher, property graph patterns, estrategias de labels, anti-patterns de modelagem
- `migration-patterns.md` -- Evolucao de schema, gerenciamento de constraints, migracoes APOC, migracao de indices, versionamento de schema
- `query-optimization.md` -- Tipos de indice (B-tree, fulltext, range, point, text), PROFILE/EXPLAIN, otimizacao de travessia, tuning de memoria, cache warming

### 3.3 Arquivos Neptune (graph/neptune/)

- `modeling-patterns.md` -- Gremlin vs SPARQL: quando usar cada um, design de vertex/edge, constraints especificas do Neptune, limites de propriedades
- `migration-patterns.md` -- Bulk loader (S3), Neptune Streams para CDC, evolucao schema-free, estrategias de re-ingestion
- `query-optimization.md` -- Query hints, dimensionamento de instancias, caching, DFE (Data Flow Engine), query timeouts

### 3.4 Settings Files

- `targets/claude/settings/database-neo4j.json` -- Permite `cypher-shell` como ferramenta CLI
- `targets/claude/settings/database-neptune.json` -- Permite `aws neptune` como ferramenta CLI
