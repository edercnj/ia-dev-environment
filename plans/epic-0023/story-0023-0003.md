## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que o `version-matrix.md` inclua tabelas para Graph, Columnar, NewSQL, TimeSeries e Search databases, e que o `database-patterns` KP tenha routing para essas novas categorias, para que a IA saiba quais arquivos de referencia consultar para cada tipo de banco.

Atualmente o `version-matrix.md` contem apenas secoes para SQL databases (PostgreSQL, MySQL, MariaDB, Oracle, SQL Server) e NoSQL databases (MongoDB, Cassandra, Redis, DynamoDB, Couchbase). O `database-patterns` SKILL.md contem routing apenas para essas duas categorias. Esta story expande ambos os arquivos com 5 novas categorias.

### 3.1 version-matrix.md -- Novas Secoes

Adicionar 5 novas secoes ao final do arquivo, cada uma com tabela de versoes:

- **Graph Databases**: Neo4j (versao recomendada, driver, OGM), Amazon Neptune (engine version, SDK)
- **Columnar/OLAP Databases**: ClickHouse (versao, client, JDBC driver), Apache Druid (versao, client libraries)
- **NewSQL/Distributed Databases**: YugaByteDB (versao, JDBC driver), CockroachDB (versao, driver), TiDB (versao, MySQL-compatible driver)
- **TimeSeries Databases**: InfluxDB (versao 2.x, client libraries), TimescaleDB (versao, extensao PostgreSQL)
- **Search Engines**: Elasticsearch (versao 8.x, client), OpenSearch (versao 2.x, client)

### 3.2 database-patterns SKILL.md -- Routing para Novas Categorias

Adicionar apos a linha 36 (final das secoes existentes de routing) novas secoes de routing:

- **Graph Database Routing**: Quando database e neo4j ou neptune, consultar `graph/common/` + `graph/{banco}/`
- **Columnar Database Routing**: Quando database e clickhouse ou druid, consultar `columnar/common/` + `columnar/{banco}/`
- **NewSQL Database Routing**: Quando database e yugabytedb, cockroachdb ou tidb, consultar `newsql/common/` + `newsql/{banco}/`
- **TimeSeries Database Routing**: Quando database e influxdb ou timescaledb, consultar `timeseries/common/` + `timeseries/{banco}/`
- **Search Engine Routing**: Quando database e elasticsearch ou opensearch, consultar `search/common/` + `search/{banco}/`

### 3.3 Preservacao de Conteudo Existente

As secoes existentes de SQL e NoSQL em ambos os arquivos devem permanecer inalteradas. Nenhuma linha existente deve ser modificada ou removida.

## 3.5 Entrega de Valor

- **Valor Principal:** IA recebe routing atualizado para 5 novas categorias de banco, direcionando para os arquivos de referencia corretos
- **Metrica de Sucesso:** Cada nova categoria tem secao no version-matrix e routing no database-patterns SKILL.md
- **Impacto no Negocio:** Habilita a IA a fornecer orientacao correta para projetos usando bancos graph, columnar, newsql, timeseries ou search
