## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que o gerador suporte 7 categorias de banco (SQL, NoSQL, Graph, Columnar, NewSQL, TimeSeries, Search) em vez das 2 atuais (SQL, NoSQL), para que novos bancos possam ser adicionados sem modificar a logica de roteamento.

Atualmente o sistema reconhece apenas duas categorias: SQL (postgresql, mysql, mariadb, oracle, sqlserver) e NoSQL (mongodb, cassandra, redis, dynamodb, couchbase). Esta story expande para 7 categorias, criando a infraestrutura Java necessaria para rotear arquivos de knowledge corretamente.

### 3.1 RulesConditionals.java

Adicionar 5 novos Sets constantes:
- `GRAPH_DB_TYPES` = Set.of("neo4j", "neptune")
- `COLUMNAR_DB_TYPES` = Set.of("clickhouse", "druid")
- `NEWSQL_DB_TYPES` = Set.of("yugabytedb", "cockroachdb", "tidb")
- `TIMESERIES_DB_TYPES` = Set.of("influxdb", "timescaledb")
- `SEARCH_DB_TYPES` = Set.of("elasticsearch", "opensearch")

Estender o metodo `copyDbTypeFiles()` com else-if branches para cada nova categoria, copiando os diretorios `common/` + `{banco}/` correspondentes.

### 3.2 StackMapping.java

Alterar `DATABASE_SETTINGS_MAP` de `Map.of()` para `Map.ofEntries()` (Map.of() tem limite de 10 entradas) e adicionar 12 novas entradas:
- neo4j, neptune (graph)
- clickhouse, druid (columnar)
- yugabytedb, cockroachdb, tidb (newsql)
- influxdb, timescaledb (timeseries)
- elasticsearch, opensearch (search)

### 3.3 resource-config.json

Adicionar patterns de include para os novos subdiretorios sob `knowledge/databases/`:
- `knowledge/databases/graph/**`
- `knowledge/databases/columnar/**`
- `knowledge/databases/newsql/**`
- `knowledge/databases/timeseries/**`
- `knowledge/databases/search/**`

### 3.4 Estrutura de Diretorios

Criar estrutura vazia (com .gitkeep) para:
- `knowledge/databases/graph/{common,neo4j,neptune}/`
- `knowledge/databases/columnar/{common,clickhouse,druid}/`
- `knowledge/databases/newsql/{common,yugabytedb,cockroachdb,tidb}/`
- `knowledge/databases/timeseries/{common,influxdb,timescaledb}/`
- `knowledge/databases/search/{common,elasticsearch,opensearch}/`

## 3.5 Entrega de Valor

- **Valor Principal:** Gerador suporta 7 categorias de banco, habilitando expansao futura sem modificar logica de roteamento
- **Metrica de Sucesso:** Cada banco novo e roteado corretamente para sua categoria com `common/` + `{banco}/`
- **Impacto no Negocio:** Desbloqueia 6 stories de conteudo (story-0023-0004 a story-0023-0009) que dependem desta infraestrutura
