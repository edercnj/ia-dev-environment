## 3. Descrição

Como **desenvolvedor**, eu quero que os golden files de teste reflitam os novos nomes para que os testes de byte-for-byte passem.

Os golden files em `src/test/resources/golden/` são snapshots do output esperado do gerador. Com os renames, os diretórios de skills no output agora usam os novos nomes, mas os golden files ainda esperam os antigos. Os golden files precisam ser regenerados rodando o pipeline completo para cada perfil.

### 3.1 Perfis a Regenerar

17 perfis em `src/test/resources/golden/`:
- go-gin, java-quarkus, java-spring, java-spring-clickhouse, java-spring-kafka, java-spring-neo4j, kotlin-ktor, python-click-cli, python-fastapi, python-fastapi-kafka, rust-axum, typescript-nestjs, etc.

Cada perfil gera outputs para 3-4 plataformas:
- `.claude/` — skills, rules, agents, hooks, templates
- `.agents/` — skills
- `.codex/` — skills
- `.github/` (apenas platform-claude-code) — skills, agents, instructions

### 3.2 Estimativa de Impacto

- ~5000+ arquivos modificados (diretórios renomeados + conteúdo atualizado)
- O diff é grande mas mecânico — 100% auto-gerado

### 3.3 Procedimento

1. Deletar `src/test/resources/golden/` inteiro
2. Rodar `mvn test -Dtest=ByteForByteGoldenTest -DupdateGoldenFiles=true` (ou equivalente)
3. Commit dos novos golden files

### 3.4 Entrega de Valor

- **Valor Principal:** Testes de byte-for-byte passam
- **Métrica de Sucesso:** `mvn verify` 100% verde
- **Impacto no Negócio:** CI passa, merge possível
