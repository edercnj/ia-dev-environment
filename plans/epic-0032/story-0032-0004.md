## 3. Descrição

Como **desenvolvedor**, eu quero que nenhum SKILL.md contenha referências a nomes antigos para que os links entre skills estejam corretos.

Após renomear os diretórios (stories 0002 e 0003), o conteúdo dos SKILL.md ainda contém referências aos nomes antigos. Por exemplo, `x-dev-epic-implement/SKILL.md` contém `Skill(skill: "x-dev-lifecycle")` que deve ser atualizado para `Skill(skill: "x-dev-story-implement")`.

### 3.1 Escopo de Substituição

Todos os arquivos `.md` dentro de `java/src/main/resources/targets/` devem ser varridos:
- SKILL.md (body e description no frontmatter)
- README.md (de cada skill)
- references/*.md (arquivos de referência)
- Knowledge packs README.md
- GitHub Copilot skill .md files
- Templates .j2

### 3.2 Estratégia de Substituição

Dois passes necessários:
1. **Nomes longos** (sem risco de substring): `x-dev-lifecycle` → `x-dev-story-implement`, `x-fix-epic-pr-comments` → `x-pr-fix-epic-comments`, etc.
2. **Nomes curtos** (risco de substring): usar word-boundary (`perl -e` com lookbehind/lookahead) para `x-commit`, `x-tdd`, `x-format`, `x-lint`, `x-changelog`, `x-docs`, `x-worktree`, `x-pentest`

### 3.3 Validação

Após substituição, executar grep para cada nome antigo e confirmar zero ocorrências.

### 3.4 Entrega de Valor

- **Valor Principal:** Integridade referencial entre todos os skills
- **Métrica de Sucesso:** `grep -r "{nome_antigo}" resources/` retorna zero resultados para cada nome renomeado
- **Impacto no Negócio:** Skills que delegam para outros skills funcionam corretamente
