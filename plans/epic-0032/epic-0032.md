# Épico: Padronização de Nomenclatura de Skills

**Autor:** Eder Celeste Nunes Junior
**Data:** 2026-04-09
**Versão:** 1.0
**Status:** Pronto

## 1. Visão Geral

**Chave Jira:** —

Os nomes das skills do projeto `ia-dev-env` não seguem uma convenção consistente. Skills relacionadas ao mesmo domínio usam prefixos diferentes — por exemplo, `x-pr-create` e `x-fix-pr-comments` tratam de Pull Requests, mas têm estruturas de nome incompatíveis. O mesmo ocorre com `x-dev-epic-implement` (implementação de épicos) e `x-dev-lifecycle` (implementação de stories), quando o paralelo natural seria `x-dev-story-implement`.

Este épico padroniza **25 skills** para seguir a convenção `x-{categoria}-{ação}`, atualiza todas as referências cruzadas em SKILL.md, código Java, testes, golden files e documentação. Também documenta formalmente a regra de **source of truth**: toda alteração de skills deve ser feita em `java/src/main/resources/targets/`, nunca nos diretórios gerados (`.claude/`, `.github/`, `.codex/`, `.agents/`).

### Global Definition of Ready (DoR)

- Tabela de renames aprovada (seção 4, RULE-001)
- Inventário de todos os arquivos impactados por skill mapeado
- Regra de source of truth definida

### Global Definition of Done (DoD)

- **Cobertura:** ≥ 95% Line, ≥ 90% Branch
- **Testes Automatizados:** Todos os testes existentes passam sem regressão (zero novas falhas)
- **Smoke Tests:** Golden files regenerados e byte-for-byte com output do gerador
- **Relatório de Cobertura:** JaCoCo
- **Documentação:** CLAUDE.md, AGENTS.md e CHANGELOG.md atualizados com novos nomes
- **Persistência:** N/A
- **Performance:** N/A
- **TDD Compliance:** N/A (refactoring apenas — sem novo comportamento)
- **Double-Loop TDD:** N/A

## 4. Regras de Negócio Transversais (Source of Truth)

> Regras que se aplicam a múltiplas histórias. Cada história referencia as regras pelo ID.

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-001]** | Convenção de Nomenclatura | Todos os skills executáveis seguem o padrão `x-{categoria}-{ação}`. Knowledge packs não têm prefixo `x-`. Runners condicionais usam prefixo `run-`. |
| **[RULE-002]** | Source of Truth | A fonte verdade de skills é `java/src/main/resources/targets/`. Os diretórios `.claude/`, `.github/`, `.codex/`, `.agents/` e `src/test/resources/golden/` são saídas geradas e NUNCA devem ser editados diretamente. |
| **[RULE-003]** | Completude de Rename | Ao renomear um skill, TODOS os seguintes devem ser atualizados: (1) diretório em resources, (2) campo `name:` no frontmatter, (3) referências cruzadas em todos os SKILL.md, (4) SkillGroupRegistry.java, (5) SkillsSelection.java, (6) testes, (7) GitHub Copilot skills, (8) golden files. |
| **[RULE-004]** | Backward Compatibility | Nenhuma lógica de backward compatibility é necessária. Os nomes antigos são completamente substituídos. |
