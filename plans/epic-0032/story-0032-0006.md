## 3. Descrição

Como **desenvolvedor**, eu quero que todos os testes passem (zero falhas) para que o CI esteja verde e o merge seja possível.

Esta story tem dois escopos:

### 3.1 Atualizar Assertions de Testes com Nomes Antigos

Aproximadamente 30+ classes de teste referenciam nomes antigos de skills em:
- Strings quoted: `"x-dev-lifecycle"`, `"x-tdd"`, `"x-commit"`, etc.
- Paths: `".claude/skills/x-dev-lifecycle/SKILL.md"`
- DisplayNames: `@DisplayName("x-dev-lifecycle ...")`
- Javadoc comments referenciando nomes antigos

Estratégia:
1. `sed` para nomes quoted (`"x-nome-antigo"` → `"x-nome-novo"`)
2. `sed` para paths (`/x-nome-antigo/` → `/x-nome-novo/`)
3. `sed` global para nomes em DisplayName e comments

### 3.2 Corrigir 26 Falhas Pré-existentes

O branch `develop` já possui 26 testes falhando. Estes devem ser investigados e corrigidos:

| Classe de Teste | Falhas | Causa Provável |
| :--- | :--- | :--- |
| `SubagentContextIsolationTest$DevLifecycle` | 4 | SKILL.md de x-dev-story-implement não tem markers CONTEXT ISOLATION |
| `SubagentContextIsolationTest$EpicImplement` | 2 | x-dev-epic-implement não tem markers de roles |
| `SubagentContextIsolationTest$CrossCutting` | 1 | Depende dos anteriores |
| `OutputCompactionTest$ContextManagement` | 5 | SKILL.md não tem context management markers |
| `OutputCompactionTest$MultiProfile` | 6 | Geração multi-profile sem context mgmt |
| `OutputCompactionTest$PhaseReportSubagent` | 2 | Phase report sem markers esperados |
| `OutputCompactionTest$SelectiveCheckpoint` | 3 | Checkpoint sem markers esperados |
| `LazyKpLoadingTest$LifecyclePrompts` | 3 | Lifecycle SKILL.md sem lazy loading refs |

**Investigação necessária:** Ler cada teste, entender o que ele espera, verificar se o conteúdo do SKILL.md deveria conter o marker ou se o teste está incorreto.

### 3.3 Entrega de Valor

- **Valor Principal:** Suite de testes 100% verde
- **Métrica de Sucesso:** `mvn test` com zero falhas
- **Impacto no Negócio:** CI verde permite merge
