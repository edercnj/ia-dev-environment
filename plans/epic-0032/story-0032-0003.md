## 3. Descrição

Como **desenvolvedor**, eu quero que os 10 conditional skills tenham nomes padronizados para que a convenção `x-{categoria}-{ação}` seja uniforme em todo o projeto.

### 3.1 Skills a Renomear

| # | Diretório Atual | Novo Diretório | Categoria |
| :--- | :--- | :--- | :--- |
| 1 | `conditional/x-contract-lint` | `conditional/x-test-contract-lint` | test |
| 2 | `conditional/x-sast-scan` | `conditional/x-security-sast` | security |
| 3 | `conditional/x-dast-scan` | `conditional/x-security-dast` | security |
| 4 | `conditional/x-secret-scan` | `conditional/x-security-secret-scan` | security |
| 5 | `conditional/x-container-scan` | `conditional/x-security-container` | security |
| 6 | `conditional/x-infra-scan` | `conditional/x-security-infra` | security |
| 7 | `conditional/x-pentest` | `conditional/x-security-pentest` | security |
| 8 | `conditional/x-sonar-gate` | `conditional/x-security-sonar` | security |
| 9 | `conditional/setup-environment` | `conditional/x-setup-stack` | setup |
| 10 | `conditional/instrument-otel` | `conditional/x-obs-instrument` | observability |

### 3.2 Operações por Skill

Mesmo procedimento de story-0032-0002:
1. `git mv` do diretório
2. Atualizar campo `name:` no frontmatter
3. Renomear arquivo correspondente em `targets/github-copilot/skills/` (onde aplicável)

### 3.3 Entrega de Valor

- **Valor Principal:** 10 conditional skills com prefixo `x-security-*`, `x-test-*`, etc.
- **Métrica de Sucesso:** Zero diretórios conditional com prefixo incorreto
- **Impacto no Negócio:** Scans de segurança agrupados sob prefixo `x-security-*`
