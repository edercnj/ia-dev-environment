# Épico: Restauração do Escopo de Gerador do `ia-dev-env`

**Autor:** Platform Team
**Data:** 2026-04-18
**Versão:** 1.0
**Status:** Pronto

## 1. Visão Geral

**Chave Jira:** —

Devolver ao CLI `ia-dev-env` o seu propósito original: **copiar `.md` / `.sh` / `.yml` / `.json` de `java/src/main/resources/targets/claude/` para o projeto destino, resolvendo placeholders Pebble**, dirigido por perfis YAML em `shared/config-templates/`. Todo o código Java que cresceu para suportar workflows de runtime (orquestração de release, checkpoint de execução, análise de paralelismo, reportagem de progresso, emissão/análise de telemetria, smoke framework, linter CI) é removido. Sete skills `.md` que hoje dependem desse código Java são reescritas como LLM+bash (`jq`/`awk`/`git`/`gh`). Uma nova **Rule 21** passa a ser carregada em toda conversa, tornando o princípio "apenas gerador" uma invariante verificável.

## 3. Definições de Qualidade Globais

### Global Definition of Ready (DoR)

- História declara, em sua Section 2.1 (Inputs) e 2.2 (Outputs), contratos verificáveis por `grep`, `test -f`, ou execução de comando.
- História declara testabilidade (INDEPENDENT / REQUIRES_MOCK / COALESCED — RULE-15).
- História declara `## File Footprint` (RULE-004 / RULE-008) com `write:` / `read:` / `regen:` explícitos.
- Todas as stories bloqueadoras estão em status `Concluída`.
- Rule 21 está publicada em `resources/targets/claude/rules/21-generator-scope.md` (aplicável a partir de story-0052-0002 em diante).

### Global Definition of Done (DoD)

- **Cobertura:** ≥ 95% Line, ≥ 90% Branch (Rule 05) sobre o escopo Java remanescente (`cli`, `config`, `domain`, `application`, `template`, `infrastructure`, `exception`, `util`).
- **Testes Automatizados:** cada história tem pelo menos 1 teste automatizado validando seu critério de aceite principal. Stories de remoção Java validam via suite existente + smoke 18 stacks. Stories de reescrita de skill validam via golden file regenerado + execução dry-run da skill.
- **Smoke Tests:** Obrigatório para stories de aplicação final — story-0052-0008 e story-0052-0009 rodam `generate` em todas as 18 stacks.
- **Relatório de Cobertura:** JaCoCo em `java/target/site/jacoco/index.html`; CI falha se abaixo do threshold.
- **Documentação:** CLAUDE.md raiz atualizado; Rule 21 publicada; SKILL.md das 7 skills reescritas coerente com novo comportamento.
- **Persistência:** N/A (não há migração de schema de dados).
- **Performance:** `ia-dev-env generate --stack X -o tmp --force` conclui em ≤ 10s para qualquer stack, igual ao baseline pré-épico.
- **TDD Compliance:** Commits mostram padrão test-first. Refatoração explícita após green. Testes incrementais via TPP.
- **Double-Loop TDD:** Cenários Gherkin de cada story viram acceptance tests (outer loop). Unit tests guiados por TPP (inner loop).

## 4. Regras de Negócio Transversais (Source of Truth)

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-001]** | Escopo de código Java | O CLI `ia-dev-env` é **apenas um gerador**. Código Java vive APENAS em 8 pacotes: `cli`, `config`, `domain`, `application`, `template`, `infrastructure`, `exception`, `util`. Qualquer PR que adicionar código em `dev.iadev.release`, `dev.iadev.checkpoint`, `dev.iadev.parallelism`, `dev.iadev.progress`, `dev.iadev.telemetry`, `dev.iadev.smoke`, `dev.iadev.ci` ou pacotes novos análogos é rejeitado. Features novas = artefatos `.md`/`.sh`/`.yml` em `resources/targets/{platform}/` e `resources/shared/`. |
| **[RULE-002]** | Contrato do comando `generate` é invariante | As 9 flags existentes (`-c`, `-i`, `-o`, `-s`, `-v`, `--dry-run`, `-f`, `--overwrite-constitution`, `-p`) e as 18 stacks (`java-picocli-cli` ... `rust-axum`) permanecem com o mesmo comportamento. Stories que tocam a CLI DEVEM preservar o output byte-a-byte dos Assemblers categoria A contra os golden files existentes. |
| **[RULE-003]** | Skills coupladas passam a LLM+bash | Skills `.md` não podem invocar `java -cp` nem `java -jar` apontando para o JAR do `ia-dev-env`. A análise, orquestração e decisão vivem no LLM usando ferramentas padrão (`jq`, `awk`, `git`, `gh`, `grep`). |
| **[RULE-004]** | Hooks shell são preservados | Os scripts `telemetry-*.sh` e `post-compile-check.sh` em `resources/targets/claude/hooks/` são shell puro e continuam ativos. Apenas o **analisador** do NDJSON gerado por eles muda (Java → bash+LLM). |
| **[RULE-005]** | Ordem topológica imutável | A execução segue a ordem: guardrail (story-0001) → reescrita de skills (stories 0002–0004) → remoção de entry points (story-0005) → remoção de pacotes (stories 0006–0007) → ajuste de Assemblers (story-0008) → validação (story-0009). Violar essa ordem pode quebrar o próprio orchestrator `/x-epic-implement` enquanto ele é reescrito (auto-dogfooding). |
| **[RULE-006]** | Nenhuma feature nova | Este épico é 100% remoção / simplificação / reescrita-equivalente. Zero features novas. Zero `.java` novo. Novos arquivos permitidos: `.claude/rules/21-generator-scope.md`, SKILL.md reescritos (conteúdo reescrito, caminho inalterado), atualização do `CLAUDE.md` raiz. |
| **[RULE-007]** | Rule 21 é gate duradouro | A rule 21 é carregada em toda conversa. Violações futuras são detectadas em code review via grep deterministic (ver Section 6 do SPEC). Remover ou enfraquecer a Rule 21 exige um novo ADR. |
