# Épico: Rollout ADR-0012 — Slim-by-default para 8 orchestrators

**Autor:** Eder Celeste Nunes Junior
**Data:** 2026-04-21
**Versão:** 1.0
**Status:** Em Refinamento
**Épico ID:** EPIC-0054

## 1. Visão Geral

**Chave Jira:** —

EPIC-0047 estabeleceu o framework arquitetural de compressão de skill bodies via **ADR-0012 (flipped-orientation / slim-by-default)** e shippou `SkillSizeLinter` (CI guard-rail), ADR-0011 (shared snippets), diretório `_shared/`, baseline `audits/skill-size-baseline.txt`, e aplicou o pattern a 5 skills piloto (x-test-tdd, x-story-implement, x-git-commit, x-code-format, x-code-lint). O corpus caiu de 50.191 → 43.320 linhas (−13,69%) mas **não atingiu** o target aspiracional de −40% (≤30.115 linhas). O gap remanescente (~13.205 linhas) concentra-se em **8 skills orchestrator** ainda > 500 linhas, somando 10.521 linhas combinadas.

Este épico **aplica** ADR-0012 de forma deliberada aos 8 orchestrators restantes, acelerando um rollout que o ADR-0012 §5 prescreve como oportunístico. Escopo exato: `x-release` (2.811), `x-epic-implement` (2.377), `x-pr-fix-epic` (1.296), `x-story-plan` (1.199), `x-pr-merge-train` (873), `x-task-implement` (821), `x-security-pipeline` (576) e `x-git-worktree` (568). Cada SKILL.md alvo é reescrita no contrato canônico de 5 seções ADR-0012 (`## Triggers`, `## Parameters`, `## Output Contract`, `## Error Envelope`, `## Full Protocol`) com `references/full-protocol.md` recebendo o corpo verboso carved. Projeção pós-épico: 10.521 → ~2.000 linhas body (8 × ~250) + ~2.400 linhas em references, **−6.121 linhas no hot-path**, corpus total ~37.200 linhas, gap ao target fechado em ~54%.

Este épico **NÃO redesenha** ADR-0012 (pattern consolidado), **NÃO modifica** `SkillSizeLinter` (guard-rail consumido, não alterado), **NÃO introduz código Java novo** (somente edits `.md` + golden regeneration), e **NÃO cobre** os 15 knowledge-packs > 500 linhas (tratados em EPIC-0051 "Knowledge Packs fora de `.claude/skills/`" que está Em Refinamento com abordagem diferente: remover KPs do hot-path via reestruturação, não slim+references). Skills 250-500 linhas permanecem em WARN tier para migração oportunística. O HIGH follow-up de EPIC-0047 (SkillSizeLinter em `src/main/java/` viola Rule 14) é concern separado e não resolvido por este épico.

Fora de escopo explícito: alterações em ADR-0011/ADR-0012 (contratos estabelecidos), alterações em `SkillSizeLinter` (estável), compressão de KPs (EPIC-0051 paralelo), relocation de `SkillSizeLinter` para `src/test/java/` (follow-up independente).

## 4. Regras de Negócio Transversais (Source of Truth)

> Regras que se aplicam às 4 histórias deste épico. Cada história referencia por ID.

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-054-01]** | Contrato literal ADR-0012 (5 seções canônicas) | Cada slim SKILL.md resultante DEVE ter exatamente as 5 seções top-level na ordem: `## Triggers`, `## Parameters`, `## Output Contract`, `## Error Envelope`, `## Full Protocol`.<br>A seção `## Full Protocol` contém APENAS uma frase de hand-off + link relativo para `references/full-protocol.md` (ex: `See \`references/full-protocol.md\` for step-by-step execution.`).<br>Seções adicionais (Examples, Integration Notes) PROIBIDAS no slim body — movem para `references/`.<br>Frontmatter YAML (name, description, allowed-tools) preservado verbatim.<br>SEM invenção de padrão: `references/full-protocol.md` é o nome canônico; partial carves existentes (ex: x-release/references/auto-version-detection.md) preservam nome atual como arquivo companheiro. |
| **[RULE-054-02]** | Atualização mandatória de `audits/skill-size-baseline.txt` | Toda skill que cai ≤ 500 linhas body após o carve DEVE ter sua entrada REMOVIDA de `audits/skill-size-baseline.txt` no mesmo commit do carve.<br>Staleness test de EPIC-0046 (`SkillSizeLinterBaselineStalenessTest`) detecta entries dead-letter como regressão.<br>Skills que permanecem > 500 linhas por decisão arbitrada (not expected em este épico) mantêm entry com comentário atualizado. |
