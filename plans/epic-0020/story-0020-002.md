## 3. Descricao

Como **desenvolvedor do ia-dev-environment**, eu quero que todos os recursos especificos do Claude Code estejam agrupados sob `targets/claude/`, para que a navegacao no diretorio de recursos seja intuitiva e consistente com os demais targets.

### Contexto

Atualmente, os recursos Claude estao em 5 pastas top-level sem prefixo comum: `agents-templates/`, `skills-templates/`, `core-rules/`, `hooks-templates/`, `settings-templates/`. Apos esta story, estarao sob `targets/claude/` com nomes simplificados.

### 3.1 Movimentacao de diretorios

| Origem | Destino |
| :--- | :--- |
| `agents-templates/` | `targets/claude/agents/` |
| `skills-templates/` | `targets/claude/skills/` |
| `core-rules/` | `targets/claude/rules/` |
| `hooks-templates/` | `targets/claude/hooks/` |
| `settings-templates/` | `targets/claude/settings/` |

### 3.2 Assemblers a atualizar

- `RulesAssembler` — probe `"core-rules"` → `"targets/claude/rules"`
- `SkillsAssembler` — probe `"skills-templates"` → `"targets/claude/skills"`
- `AgentsAssembler` — probe `"agents-templates"` → `"targets/claude/agents"`
- `HooksAssembler` — probe `"hooks-templates"` → `"targets/claude/hooks"`
- `SettingsAssembler` — probe `"settings-templates"` → `"targets/claude/settings"`
- `CoreRulesWriter`, `SkillsCopyHelper`, `ChecklistInjector`, `AntiPatternsRuleWriter` — constantes internas de path

### 3.3 Subtlety: RulesAssembler dual-purpose

O `RulesAssembler` usa o `resourcesDir` (raiz dos recursos) para navegar tanto para suas proprias rules quanto para `languages/`, `frameworks/`, `databases/`, etc. Ao mudar o probe para `"targets/claude/rules"`, o `resolveResourcesRoot` deve continuar retornando a raiz dos recursos (nao o diretorio da rules). Alternativamente, usar `resolveResourceDir` para a rules dir e manter um segundo path para a raiz.

## 3.5 Entrega de Valor

- **Valor Principal:** Recursos Claude organizados sob namespace proprio, navegacao intuitiva para qualquer desenvolvedor
- **Metrica de Sucesso:** Todos os testes de assemblers Claude passando com paths novos
- **Impacto no Negocio:** Reducao de tempo de onboarding e manutencao ao localizar recursos Claude
