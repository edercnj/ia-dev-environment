## Planning Summary

Story 0034-0002 is the atomic removal of Codex (OpenAI Codex / TOML-based) target support from the `ia-dev-environment` generator. It is the second of three target-removal stories in epic-0034 and carries the LARGEST single-story golden-file deletion volume (2944 files). Inventory per baseline: 7 Java main classes, 6 test classes, 15 resource files under `targets/codex/`, 2944 golden fixtures under `.codex/` in 17 profiles, 18 shared YAMLs to clean, and 17 dependent test files that require atomic adjustments. Multi-agent planning surfaced three reconciliations against the story text: (1) `AssemblerFactory.buildCodexAssemblers()` contains an exception — `DocsAdrAssembler` (shared, not Codex) — that must be relocated before the method is deleted; (2) actual YAML count is 18 (not 17 as stated in story §3.7); (3) 17 dependent test files reference `Platform.CODEX` / `AssemblerTarget.CODEX` / `hasCodex` and must be bundled into the enum-edit task atomically. Agent consensus aligns with the story's existing §8 task decomposition of 5 tasks — the multi-agent plan expands it to 6 atomic tasks (splitting YAML cleanup from final verification, matching the successful pattern of story 0034-0001). The task DoD was augmented with security/quality gates from the Security Engineer and Tech Lead agents. Execution of story 0034-0002 depends only on story 0034-0001 being merged (execution level); planning proceeded independently.

## Architecture Assessment

Affected layers: `domain.model` (Platform enum removes CODEX), `application.assembler` (AssemblerTarget enum removes CODEX entry but preserves CODEX_AGENTS, AssemblerFactory factory, 7 Codex assembler classes, 6 Codex test classes, PlatformContextBuilder), `cli` (PlatformConverter verified, GenerateCommand option description, FileCategorizer branch), `util` (OverwriteDetector ARTIFACT_DIRS). Dependency direction respected: delete Codex production classes FIRST (task 001) to remove inbound references to Platform.CODEX; delete test classes SECOND (task 002); edit enums and shared hygiene classes + 17 dependent test files THIRD (task 003) once no code still imports the removed constants; delete resources FOURTH (task 004); delete golden files FIFTH (task 005, parallelizable with task 004); final gate with YAML cleanup + build verify + PR SIXTH (task 006). Post-edit `AssemblerFactory.buildAllAssemblers()` returns 19 assemblers instead of 26 (delta -7). No architectural layering violations. RULE-004 (`resources/shared/templates/` protected) is not touched by this story. Class diagram: `Platform{CLAUDE_CODE,CODEX,SHARED}` (post-0001) → `Platform{CLAUDE_CODE,SHARED}` (post-0002). `AssemblerTarget{ROOT,CLAUDE,CODEX,CODEX_AGENTS}` (post-0001) → `AssemblerTarget{ROOT,CLAUDE,CODEX_AGENTS}` (post-0002, CODEX_AGENTS deliberately preserved for story 0034-0003). All 7 shared class line counts remain under Clean Code's 250-line limit post-edit. **Critical architectural gotcha:** `AssemblerFactory.buildCodexAssemblers()` historically registered `DocsAdrAssembler` as a shared descriptor inside the Codex builder — the refactor in TASK-001 must RELOCATE that descriptor to a surviving builder (`buildCicdAssemblers` recommended) before deleting the method, or DocsAdrAssembler becomes orphaned.

## Test Strategy Summary

This is a deletion story, so the TDD inner loop is inverted from the traditional RED→GREEN→REFACTOR flow. Instead, the QA strategy is anchored on: (1) RULE-006 pre-delete green confirmation (story-0001 post-merge baseline; 6 Codex test classes must be passing before TASK-002 deletes them); (2) compile-verified GREEN cycles after each task commit; (3) existing smoke tests `CliModesSmokeTest`, `PlatformDirectorySmokeTest`, `AssemblerRegressionSmokeTest` acting as regression detectors. Six acceptance tests are mapped to the six Gherkin scenarios in story §7, covering TPP levels nil→constant→scalar→conditional→boundary. **AT-1** (build verde, scalar) validated by `mvn clean verify`. **AT-2** (CLI rejects `--platform codex`, conditional) validated by CLI smoke. **AT-3** (CLI works for `claude-code`, constant) validated by CLI smoke. **AT-4** (`.codex/` golden files completely removed, boundary invariant) validated by `find ... -name '.codex' -type d | wc -l` returning 0 + absence of `config.toml` files. **AT-5** (grep sanity, scalar) validated in TASK-006. **AT-6** (degenerate — PlatformDirectorySmokeTest + CliModesSmokeTest pass, nil) validated by targeted `mvn test`. No new unit tests are written — the 6 existing Codex tests are DELETED atomically with their corresponding production code (RULE-006 atomicity). **Unique to this story:** 17 dependent test files (PlatformTest, ProjectConfigTest, PipelineOptionsTest, AssemblerFactoryBuildAllTest, AssemblerPipelineTest, SummaryTableBuilderPlatformTest, AssemblerFactoryPlatformTest, PlatformContextBuilderTest, PlatformFilterTest, SummaryRowFilterTest, PlatformPrecedenceResolverTest, BuildPlatformSetTest, PlatformVerboseFormatterTest, ContextBuilderPlatformTest, PlatformPipelineIntegrationTest, AssemblerRegressionSmokeTest, PlatformDirectorySmokeTest) require atomic surgical adjustments in TASK-003 to reflect the reduced enum sets. `AssemblerRegressionSmokeTest` may additionally require `expected-artifacts.json` regeneration — deferred to TASK-006. Projected coverage post-deletion: ≥ 95% line / ≥ 90% branch (within 2pp of post-story-0001 baseline, ideally identical to baseline since deleted production and deleted tests are proportional).

## Implementation Approach

Tech Lead consolidates agent proposals and upholds a 6-atomic-task decomposition (story §8 proposes 5; the 6th task was split from the story's TASK-005 to separate YAML cleanup from final verify, matching the successful pattern of story 0034-0001). Each task produces one Conventional Commit, with build-green-between-tasks as the core invariant (RULE-001). Task 003 carries the breaking-change footer because it is the commit that removes `Platform.CODEX` and the CLI contract value. Task 006 is the final PR creation gate and carries the full verification burden: `mvn clean verify`, coverage ≥ 95%/90% (degradation ≤ 2pp vs. post-story-0001 baseline ~95.69%/90.69%), 6 acceptance test validations, grep sanity, commit history check, PR body assembly with before/after metrics table, and conditional `expected-artifacts.json` regeneration. Tasks 004 and 005 are parallelizable (touch disjoint paths: `src/main/resources/` vs `src/test/resources/golden/`). **Critical decision:** TASK-001 must RELOCATE `DocsAdrAssembler` (currently registered inside `buildCodexAssemblers()`) into a surviving builder BEFORE deleting the method. This is a non-obvious architectural cleanup embedded in the deletion task — documented in the task 001 commit body. **Critical decision 2:** TASK-002 (delete 6 tests) may need to be coalesced with TASK-003 (enum edits + 17 dependent test updates) into a single atomic commit if `mvn test-compile` fails after 002 alone. The story still produces 6 logical tasks, but the git history may show 5 or 6 commits depending on the coalescence decision made at execution time. **Escalation notes** captured in the task breakdown document six reconciliation items (DocsAdrAssembler relocation requirement, 17 dependent test files, YAML count delta 17 vs 18, CODEX_AGENTS preservation requirement, PlatformContextBuilder method-signature simplification, manifest regeneration strategy).
