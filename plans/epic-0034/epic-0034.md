# Épico: Remoção de Targets Não-Claude do Gerador

**Autor:** Tech Lead / Platform Team
**Data:** 2026-04-10
**Versão:** 1.0
**Status:** Concluído (pending final PR merges)

## 1. Visão Geral

**Chave Jira:** —

Simplificação estratégica do gerador `ia-dev-environment` (Java 21 / Maven / picocli) para suportar exclusivamente Claude Code como target, removendo o suporte legado a GitHub Copilot (`.github/`), Codex (`.codex/`) e ao target genérico "agents" (`.agents/`). A remoção envolve ~650-700 arquivos afetados entre classes Java, resources, testes unitários, golden files e documentação. O escopo INCLUI deleção de **18 classes Java** (8 GitHub/Copilot + 7 Codex + 2 Agents + 1 `ReadmeGithubCounter`), 2 diretórios de resources, **29 classes de teste + 2 fixtures** e ~8.273 golden files, além de higienização de 10+ classes compartilhadas. O escopo EXCLUI explicitamente `.github/workflows/` (CI/CD GitHub Actions, ortogonal ao Copilot) e templates em `resources/shared/templates/` (usados pelo Claude). Ao final, o gerador terá um único `Platform.CLAUDE_CODE`, um único `AssemblerTarget.CLAUDE`, build verde, coverage mantida (≥ 95% line / ≥ 90% branch) e redução de ~87% no volume de arquivos gerados por profile.

**BREAKING CHANGE:** Esta mudança remove valores previamente aceitos do CLI (`--platform copilot|codex|agents`), sem período de deprecation. Conforme Rule 08 (Release Process), o próximo release deve ter bump MAJOR. Ver Seção 6 — Comunicação de Breaking Change.

## 3. Definições de Qualidade Globais

### Global Definition of Ready (DoR)

- Especificação técnica (`spec-remove-non-claude-targets.md`) revisada e aprovada
- Inventário de arquivos a deletar/editar validado (ver Seção 2 da spec)
- Branch `feature/epic-0034-remove-non-claude-targets` criada a partir de `develop` atualizado
- Build e testes no estado base (`develop`) verdes como baseline
- Contagem de arquivos gerados por profile registrada como baseline (~9.500 para `java-spring`)
- Coverage base registrada (line/branch) para comparação pós-remoção
- Lista explícita de diretórios PROTEGIDOS confirmada (`resources/shared/templates/`, `.github/workflows/`)
- `ExpectedArtifactsGenerator` identificado como ferramenta de regeneração de manifests

### Global Definition of Done (DoD)

- **Cobertura:** ≥ 95% line, ≥ 90% branch (Rule 05). Degradação máxima permitida: 2pp vs. baseline pré-épico.
- **Testes Automatizados:** Todos os testes remanescentes passando. Cada story DEVE ter pelo menos 1 teste automatizado (smoke ou unit) validando o critério de aceite principal. Remoção de testes DEVE ser proporcional à remoção do código que eles cobriam.
- **Smoke Tests:** Obrigatório. `PlatformDirectorySmokeTest`, `AssemblerRegressionSmokeTest` e `CliModesSmokeTest` devem passar após edição.
- **Relatório de Cobertura:** JaCoCo report gerado via `mvn clean verify` e anexado ao PR de cada story.
- **Documentação:** `CLAUDE.md` raiz, `.claude/rules/*.md` e `README.md` atualizados refletindo o novo escopo Claude-only.
- **Persistência:** Não aplicável (gerador stateless). Manifest `expected-artifacts.json` regenerado via `ExpectedArtifactsGenerator`.
- **Performance:** Tempo de `mvn clean verify` não pode aumentar. Esperado: redução de tempo por eliminação de ~29 classes de teste e ~8.273 comparações de golden files.
- **TDD Compliance:** Commits seguem padrão test-first onde aplicável (remoção de teste acompanha remoção proporcional de código). Refactoring explícito após green. Deleções são incrementais (uma plataforma por story via TPP: Copilot → Codex → Agents → Shared → Docs).
- **Double-Loop TDD:** Cenários Gherkin (outer loop) validam build verde e CLI funcional. Smoke tests (inner loop) validam estrutura de output. Nenhum teste removido sem que estivesse passando no baseline.
- **Build Verde:** Cada story termina com `mvn clean verify` verde ANTES do merge. Breakage intermediário entre stories é proibido.
- **Grep Sanity Check:** Ao final do épico, `grep -r "GithubInstructionsAssembler\|CodexConfigAssembler\|AgentsAssembler" java/src/main/java` retorna zero ocorrências.

## 6. Comunicação de Breaking Change

Esta mudança remove valores previamente aceitos do flag `--platform` (`copilot`, `codex`, `agents`) sem deprecation window. Conforme Rule 08 (Release Process) e semântica SemVer:

- **Bump requerido:** MAJOR (breaking change no contrato público da CLI)
- **CHANGELOG obrigatório em story-0034-0005:**
  - Seção `[Unreleased] > Removed`: lista `Platform.COPILOT`, `Platform.CODEX`, `AssemblerTarget.GITHUB`, `AssemblerTarget.CODEX`, `AssemblerTarget.CODEX_AGENTS`
  - Seção `[Unreleased] > Changed`: nota "CLI `--platform` aceita apenas `claude-code`. Valores anteriores (`copilot`, `codex`, `agents`, `all`) rejeitados com erro."
  - Seção `[Unreleased] > Migration`: instrução para usuários com scripts automatizados atualizarem para `--platform claude-code` ou remover o flag (default passa a ser `claude-code`).
- **Justificativa para remoção abrupta (sem deprecation):** o gerador está em pre-1.0 e tem adoção interna restrita ao platform team. Manter compatibility shims para targets mortos geraria manutenção desnecessária pela vida útil restante do pre-1.0.
