## 3. Descrição

Como **proprietário do roadmap**, eu quero rodar `/x-parallel-eval --scope=epic` retroativamente nos 5 épicos pendentes (0036, 0037, 0038, 0039, 0040) e gerar **patches `.diff`** que mostrem como cada implementation map mudaria, para revisar e aplicar manualmente o que fizer sentido — sem editar os maps automaticamente.

Como nenhum dos 5 épicos está em execução (todos "Em Refinamento"), aplicar a análise antes do start evita a primeira leva de conflitos esperados. Os patches são salvos em `plans/epic-0041/migrations/` para revisão isolada.

### 3.1 Procedimento

Para cada épico ∈ {0036, 0037, 0038, 0039, 0040}:

1. Re-gerar todos os task plans com `/x-task-plan --force` (story-0041-0002 já habilita File Footprint)
2. Re-gerar todos os story plans com `/x-story-plan --force` (story-0041-0003)
3. Rodar `/x-parallel-eval --scope=epic --epic plans/epic-XXXX --out plans/epic-0041/migrations/parallel-eval-XXXX.md`
4. Re-gerar `implementation-map-XXXX.md` em arquivo temporário
5. Diff contra o map original → salvar como `plans/epic-0041/migrations/implementation-map-XXXX.diff`
6. Para cada conflito detectado, registrar entrada em `plans/epic-0041/migrations/RECOMMENDATIONS.md`

## 3.5 Entrega de Valor

- **Valor Principal:** Visibilidade antecipada de conflitos potenciais nos próximos 5 épicos; lista priorizada de ajustes recomendados antes de iniciar execução.
