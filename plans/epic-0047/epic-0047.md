**Versão:** 1.0

**Épico ID:** EPIC-0047

## 1. Visão Geral

**Chave Jira:** —

O corpus de SKILL.md fontes (`java/src/main/resources/targets/claude/skills/`) está na ordem de **~45,9k linhas em 125 skills** (baseline aproximada, coletada em 2026-04-16 com `wc -l`), com forte duplicação inter-skill (`## Global Output Policy` em 74 skills, `## Error Handling` em 76, `## Integration Notes` em 47) e re-inlining de conteúdo que deveria estar em `references/` (efeito colateral do merge-conflict EPIC-0030 vs EPIC-0031 em v2.2.1). Cada invocação `Skill(skill: "...")` re-injeta o body inteiro no contexto do LLM — o top 6 (aprox.: x-release ~2,5k, x-epic-implement ~2,0k, x-story-implement ~1,3k, x-pr-fix-epic ~1,3k, click-cli-patterns ~1,2k, x-story-plan ~1,0k) consome ~9,4k linhas combinadas a cada invocação. Antes da execução de cada story, os valores exatos devem ser revalidados no repo com `wc -l`.

EPIC-0030 (Context Window Resilience) shippou a infraestrutura (rule `references/` pattern, `ContextBudget.fromLineCount()`, RULE-004 Output Compaction, RULE-005 Subagent Isolation) mas a meta de −50% nunca foi atingida porque (a) story 0030-0002 carving regrediu no merge-conflict e nunca foi re-aplicada, e (b) story 0030-0006 introduziu uma seção `## Slim Mode` append-only que o runtime do Claude Code não pode honrar (load é all-or-nothing por arquivo). Bucket A do plano `mellow-mixing-rainbow.md` (Sprints prévios, target v3.7.0) ataca os ganhos rápidos via deleção de boilerplate e re-extração das 5 skills regredidas; este épico (Bucket B) entrega o **framework estrutural** que torna essa compressão sustentável e enforçada por CI.

Este épico entrega: (1) diretório `targets/claude/skills/_shared/` com snippets cross-cutting reusáveis (`_shared/error-handling-pre-commit.md`, `_shared/tdd-tags-glossary.md`, `_shared/exit-codes-common.md`) e mecanismo de inclusão decidido por ADR-0006; (2) retirada formal do pattern `## Slim Mode` quebrado, **flipped orientation**: SKILL.md curta como default + `references/full-protocol.md` on-demand, com ADR-0007 documentando a decisão arquitetural; (3) CI lint Maven que falha qualquer SKILL.md > 500 linhas sem `references/` companion (preventivo para novas skills); (4) sweep de compressão nos 5 maiores knowledge packs (click-cli, k8s-helm, axum, terraform, dotnet — código de exemplo migra para references). Target: **−40% no corpus total** (~46k → ~28k linhas) preservando comportamento RULE-005 e paridade byte-a-byte de goldens.

Fora de escopo: runtime lazy-load no momento do `Skill()` call (bloqueado por Rule 13 — args não carregam slim flag); chunking inter-arquivo (Bucket C do plano); enforcement do `context-budget` field (separado em EPIC-0048, parking lot); compressão de template-vars `{{COMPILE_COMMAND}}` etc. (separado em EPIC-0049, parking lot). Precondição dura: Bucket A (8 PRs em chore-* off develop) deve estar mergeado em `develop` e Sprint 2 (medição de delta com `/cost` em workflows reais) deve ter rodado ANTES de STORY-0047-0002 (Slim Mode rebuild) entrar em execução — sem dado de baseline a decisão de orientation flip vira chute.

## 4. Regras de Negócio Transversais (Source of Truth)

| ID | Título | Descrição |
| :--- | :--- | :--- |
| **[RULE-047-01]** | `_shared/` é diretório irmão de `core/`/`conditional/`/`knowledge-packs/` | Localizado em `java/src/main/resources/targets/claude/skills/_shared/`. Não é uma 11ª categoria de skills (não aparece em ADR-0003). É infraestrutura de assembly: contém arquivos `.md` que skills referenciam via link relativo (Markdown anchor) ou via mecanismo de include resolvido em `mvn process-resources`. Skills em `_shared/` NÃO têm frontmatter de skill (não são invocáveis via `Skill()`). |
| **[RULE-047-02]** | Carve-out preserva referência cruzada explícita | Toda extração de seção da SKILL.md para `references/<arquivo>.md` ou para `_shared/<arquivo>.md` substitui a seção original por um bloco de 2 linhas: `## <Heading original>\n\nSee \`references/<arquivo>.md\`.` (ou `\`_shared/<arquivo>.md\` via include). LLM mantém capacidade de localizar o conteúdo durante invocação se necessário (lazy-link). |
| **[RULE-047-03]** | Slim como default; full como on-demand | Após STORY-0047-0002, cada SKILL.md afetada (ex-Slim-Mode) tem **a versão slim como body principal**. O conteúdo verboso vai para `references/full-protocol.md` (ou nome equivalente por skill). Acabou o pattern "append slim section ao final" — isso é arquiteturalmente quebrado (runtime carrega o arquivo inteiro independente do que o texto diz). ADR-0007 documenta. |
| **[RULE-047-04]** | Limite duro de 500 linhas por SKILL.md | CI lint (STORY-0047-0003) falha qualquer SKILL.md > 500 linhas que não tenha `references/` sibling não-vazio. Skills entre 250-500 linhas geram WARNING (não-bloqueante). Skills < 250 linhas passam silenciosamente. Threshold revisitado anualmente conforme corpus evolui. |
| **[RULE-047-05]** | Knowledge packs seguem mesmo regime | KPs em `knowledge-packs/` não são `core/` mas também respeitam RULE-047-04. Code samples > 30 linhas DEVEM ir para `references/examples-<lang>.md`. SKILL.md do KP fica como narrativa + índice de exemplos. |
| **[RULE-047-06]** | Atomic, Reversible Commits | Cada story produz 1 commit atômico por task (Rule 18). Mensagens Conventional Commits (Rule 08): `feat(task-0047-YYYY-NNN):` para snippets/lint, `docs(task-0047-YYYY-NNN):` para ADRs/regras, `refactor(task-0047-YYYY-NNN):` para carve-outs de SKILL.md, `test(task-0047-YYYY-NNN):` para goldens/smoke. |
| **[RULE-047-07]** | Medição obrigatória ao final do épico | Antes do back-merge da última story, executar `mvn test -Dtest=SkillCorpusSizeAudit` e capturar delta vs baseline pré-épico. Resultado documentado no CHANGELOG entry e na seção "Histórico de Medições" deste épico. Se delta < −20% (vs target −40%), abrir issue para investigar gap. |
